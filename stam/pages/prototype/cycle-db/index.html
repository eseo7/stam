STAM 요구사항 누락 검증표 (Prototype) 고객 요구사항이 기능정의, 메뉴/화면, WBS, 화면설계서, 테스트 시나리오까지 빠짐없이 이어졌는지 확인하는 실험 화면 진입 허브. 이번 단계(PR-1)는 브라우저 로컬 IndexedDB 만 사용하며 Firebase / Firestore / AI / 파일 업로드는 사용하지 않습니다. 중앙 DB 완성이 아니라 한 사이클의 데이터·연결 구조 검증이 목적입니다. Prototype Local IndexedDB 실험 화면 No Firebase
이 화면의 목적 이 표의 목적은 게시판 연결 자체가 아니라 요구사항 누락 방지입니다. 고객이 말한 요구사항이 기능정의, 화면, 작업, 설계, 테스트까지 이어졌는지 확인합니다. 데이터는 중앙 DB가 아니라 현재 브라우저의 IndexedDB에 저장됩니다.
▸ 검증 화면 Local

아래 화면에서 샘플 요구사항과 산출물 연결을 만들고(시드 생성), 요구사항별 산출물 누락 여부를 확인합니다. 데이터는 이 브라우저의 IndexedDB(stam-prototype-cycle-db)에 저장되어 새로고침 후에도 유지됩니다.

▸ 이 프로토타입의 기준 (Guardrails) Guardrails
  • 로컬 IndexedDB 기반 — Firebase SDK / Firestore 미사용 (PR-1 범위).
  • 화면은 CycleRepo 인터페이스만 호출 — 저장소 어댑터는 교체 가능(후속 Firestore).
  • 모든 artifact 기본 reviewStatus = Review Needed (Draft 기준 유지).
  • Missing 은 저장하지 않고, 기대 연결 대비 실제 link 를 계산해 표시한다.
  • 기존 제품 화면 / nav-data / shell / topbar 를 변경하지 않는 독립 prototype 경로다.
  • AI 자동 생성 / 자동 분석 / 자동 확정은 이번 범위가 아니다.
