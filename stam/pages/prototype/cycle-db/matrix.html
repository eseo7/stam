이 화면은 무엇을 보는 화면인가요? 이 화면은 요구사항별 산출물 연결 상태를 확인하는 실험 화면입니다. 게시판이 연결됐는지를 보는 것이 아니라, 고객 요구사항이 납품 산출물까지 빠짐없이 내려갔는지 확인합니다. REQ-001은 대부분의 산출물이 연결된 예시이고, REQ-002는 산출물 연결이 누락된 위험 항목 예시입니다. 시드 생성 → 새로고침 후 유지 확인 → 초기화 순서로 테스트하세요. 데이터는 중앙 DB가 아니라 현재 브라우저의 IndexedDB에 저장됩니다.
▸ 검증 순서 Local
  • ① 시드 생성: 샘플 요구사항과 산출물 연결 데이터를 만듭니다.
  • ② 새로고침: 데이터가 브라우저 로컬 DB에 유지되는지 확인합니다.
  • ③ 초기화: 로컬 DB에 저장된 prototype 데이터를 삭제합니다.

행은 요구사항이고, 열은 요구사항에서 파생되어야 할 산출물 단계입니다. 연결됨은 해당 산출물이 존재한다는 뜻입니다. 연결 누락은 아직 해당 산출물이 만들어지지 않았거나 연결되지 않았다는 뜻입니다. 검토 필요/초안은 연결은 됐지만 아직 확정 전이라는 뜻입니다.

연결됨(Linked) 검토 필요(Review Needed) 초안(Draft) 연결 누락(Missing) — 연결 누락은 저장하지 않고 기대 연결 대비 계산한 결과입니다.

  • REQ-001 로그인 2차 인증: 기능정의, 화면, WBS, 화면설계서, 테스트까지 대부분 연결된 예시입니다.
  • REQ-002 권한 관리: 고객 요구사항은 있지만 아직 산출물 연결이 누락된 위험 항목 예시입니다.
