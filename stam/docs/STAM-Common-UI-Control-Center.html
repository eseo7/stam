6. Legacy / Special Board 정책

WBS — Special Board (기준 참고 화면)
WBS는 공통 Board Filter의 설계 기준 화면입니다. 동시에 Gantt 뷰 / Focus View / 자체 Filter / 자체 Drawer 등 복잡한 특수 기능이 있어 즉시 공통화 대상이 아닙니다.
  • WBS 파일 (wbs.html, stam.wbs.css) 수정 금지
  • 신규 Board Filter/UI 설계 시 참고 읽기 허용
  • WBS 자체 공통화는 별도 작업으로 별도 검토 후 진행
화면설계서 (Screen Specification) — Legacy Board
화면설계서는 현재 Legacy Board로 분류됩니다. 다음 단계에서 Board Toolbar / Board Filter부터 점진 공통화를 진행할 후보입니다.
  • 현재는 화면 코드 수정 금지
  • 공통화 작업 시 Requirements 기준을 참고
  • Toolbar → Filter → Drawer 순으로 점진 적용 추천
신규 일반 목록형 보드 기준
WBS / 화면설계서가 아닌 신규 일반 목록형 보드는 Requirements / Menu Screen List 기준을 따릅니다. Section 4 표준 템플릿을 사용하세요.

우선순위 순서로 정리된 공통화 후보 목록입니다. 시점은 확정이 아닌 추천 기준입니다.

# 항목 이유 위험도 추천 시점
1 Table Row Selection 상태 class 정렬 4개 보드 모두 PARTIAL — 선택 상태 class 불일치 해소 LOW 즉시
2 화면설계서 Board Toolbar / Board Filter 공통화 Legacy 해소 1순위 — Requirements 기준 점진 적용 MID Table Row 정렬 후
3 Board Page Template 문서화 신규 보드 온보딩 속도 향상 — Section 4 기반 HTML 템플릿 파일 생성 LOW 2번과 병행 가능
4 Drawer Policy 확장 width 토큰 / tab 구조 / 전체보기 전환 정책 보완 LOW 신규 보드 추가 전
5 Form Controls alias 확대 화면별 커스텀 input 스타일 제거 — stam-input 통합 MID 화면설계서 공통화와 병행
6 Chips / Badges / Status 상태 chip/badge가 각 화면마다 파편화 LOW Form Controls 이후
7 Toast / Confirm 저장 완료/오류 피드백 UI 없음 — CRUD 구현 전 필요 MID CRUD 구현 전
8 Empty State 목록 데이터 없을 때 빈 화면 처리 미정의 LOW CRUD 구현 전
9 Summary Cards / Stat Strip 보드 상단 요약 수치 카드 공통화 LOW 대시보드 화면 추가 시
10 Tabs Drawer 내 탭 / 화면 내 탭 구조 공통화 MID 화면설계서 Drawer 개선 시
