STAM Implementation Readiness Dashboard v1 Readiness Dashboard

본 문서는 실제 CSS · JS 구현 착수 직전 단계에서 STAM의 준비 상태를 한 화면에서 확인하기 위한 정적 HTML 대시보드이다. 어떤 기준 문서가 완료되어 있고, 어떤 Gate가 남아 있으며, 첫 구현 후보가 무엇인지 시각적으로 정리한다. 본 문서로는 stam/css/**, stam/js/**, stam/pages/**, stam/index.html, stam/assets/**를 절대 수정하지 않는다.

01 Overview — 문서 목적 · 현재 단계 목적

STAM은 컴포넌트 가이드 v1 7종, 토큰 Readiness Check, Token Inventory v1, 6종 Token Mapping v1(Button · Form · Layer · Drawer · Dialog · Table), Decision Needed Register v1, P0 Decision Approval Sheet v1까지 완료된 상태이다. 본 단계는 “기준 문서는 모두 만들어졌으나, 아직 CSS · JS 구현은 시작되지 않은” 게이트 직전 상태이다. 본 Dashboard는 PM · 디자인 · 개발 · Cursor가 같은 기준을 보고 다음 구현 단위를 선택할 수 있도록 준비 상태를 통합한다.

이 문서가 다루는 것
  • 완료된 기준 문서 / 남은 Gate / 첫 구현 후보를 한 화면에 시각화
  • Document Status Board — 문서별 Status · Purpose · Next Action
  • Component Readiness Matrix — 컴포넌트별 위험 · 구현 허용 · 권장 순서
  • Gate Progress — Token Readiness → P0 Approval → First Implementation 단계
  • P0 Decision Snapshot — 결재 후보 10건 상태 요약
  • First Implementation Recommendation — Button CSS vs Drawer Layout 비교
  • Cursor Handoff Preview — 위임 전 조건 / 금지 사항
이 문서가 다루지 않는 것
  • stam/css/** 토큰 추가 / 값 변경 / 컴포넌트 클래스 작성
  • stam/js/** 컴포넌트 동작 / 라우팅 / 상태 관리 구현
  • stam/pages/**, stam/index.html, stam/assets/** 변경
  • P0 Approval Sheet 본문 · Decision Register 본문의 결재 결과 갱신
  • Firebase / GitHub Actions / package*.json / _ds/** / uploads/** 변경
  • Production / live 배포 의사결정
현재 단계 — 실제 구현 전 준비 상태 대시보드 본 문서는 “기준 문서가 모두 만들어졌는지, 첫 구현으로 진입해도 안전한지”를 한 페이지에서 보여준다. 본 문서로는 CSS · JS · 제품 화면을 절대 수정하지 않으며, 결재 / 구현 / 배포 의사결정은 P0 Approval Sheet · Decision Register · Cursor Handoff 프로토콜에서 별도로 처리한다.

08 Cursor Handoff Preview — 위임 전 조건 Handoff

Cursor에게 첫 구현을 위임하기 전 충족해야 할 조건과, 위임 메시지에 반드시 포함해야 할 규칙을 정리한다. 본 문서 자체는 위임 메시지가 아니다.

진입 조건 (Prerequisite)
  • P0 Approval Sheet v1 본문이 모두 Approved · Cursor 반영 완료
  • Decision Register v1 P0 10건 Status가 Approved로 동기화
  • repo working tree clean — 작업 시작 전 git status 확인
  • 수정 허용 파일을 위임 메시지에 명시 — 누적 / 암묵 명시 금지
구현 범위 (Scope)
  • 첫 구현은 Button CSS만 — Form / Drawer / Dialog / Table 동시 수정 금지
  • 관련 토큰은 stam/css/stam.tokens.css에서만 추가 / 갱신
  • 컴포넌트 클래스는 stam/css/stam.components.css에서만 작성
  • JS · 페이지 · 자산은 손대지 않음 — 이번 단계는 CSS 토큰 검증 단계
QA / 배포
  • Staging — stam-design-staging.web.app 배포 후 QA 진행
  • QA 4종 — URL · 콘솔 · 라이트/다크 · 가로 스크롤 PASS 확인
  • Production / live 배포 금지 — staging까지만
  • React dist 배포 금지 — 정적 HTML 기준 유지
금지 (Forbidden)
  • stam/css/** 중 위임 범위 외 파일 변경 금지
  • stam/js/** · stam/pages/** · stam/index.html · stam/assets/** 변경 금지
  • .firebaserc · firebase.json · .github/workflows/** 변경 금지
  • package.json · package-lock.json · _ds/** · uploads/** · *.zip · .env* 변경 금지
핸드오프 메시지 템플릿 “이번 작업: Button CSS 제한 구현. 수정 허용: stam/css/stam.tokens.css · stam/css/stam.components.css. 금지: 그 외 모든 파일. 사전 확인: P0 Approval Sheet v1 모두 Approved인가, Decision Register v1 동기화 완료인가. staging URL / 콘솔 / 라이트·다크 / 가로 스크롤 QA 4종 패스 후 보고. Production 배포 금지 · React dist 배포 금지.”
09 Risk / Hold Notes — 구현 전 주의 Risk

첫 구현 진입 전 인지해야 할 위험과 보류 항목을 정리한다. 본 항목들은 Cursor 위임 메시지에 동반 명시해야 한다.

Form — Select / Date Picker
  • JS 공통화(initStamFormControls) 확정 전에는 단독 구현 금지
  • native UI 회귀 위험 — staging QA 회귀 항목 유지 필요
  • 관련 항목 — P0-05 · P0-06
Drawer / Full Detail
  • 레이아웃 + 상태 전환이 함께 얽힘 — Layout(CSS)과 State(JS) 분리 진행
  • v1은 내부 상태 전환만 — URL routing 금지(P0-04)
  • 관련 항목 — P0-02 · P0-03 · P0-04
Table — 후순위
  • 전체 게시판에 영향 · 회귀 파급이 가장 큼
  • page-level h-scroll · 컬럼 schema(P0-07 · P0-08) 확정 후 진입
  • Backend API Boundary 정합화 필요
Layer — Dialog / Drawer / Dropdown 충돌
  • Layer 토큰은 Dialog / Drawer / Dropdown 충돌과 직접 관련
  • z-index 단계를 단일 토큰 표로 확정한 다음에만 컴포넌트 진입
  • 관련 항목 — P0-01 · P0-09
Global Loading / Toast 정책
  • Global Loading 최상위 / Toast는 Dialog 위 / Loading 아래
  • 전역 차단형 액션 진행 중 Toast 노출 보류 정책 필요
  • 관련 항목 — P0-01 · P0-09
동시 구현 금지
  • Button · Drawer · Form · Dialog · Table 중 1개만 진행
  • 각 단계 staging QA 통과 후 다음 단계 진입
