Board Builder 위치 확정

F5 하위

IA 위치

F. 관리/설정 → F5 게시판 설정 → 게시판 만들기 / Board Builder에 둔다. Board Builder는 산출물 게시판을 실제 구현하기 전 필드·목록·필터·미리보기 기준을 잡는 관리자 Preview다.

실제 repo 경로

stam/pages/boards-v2/board-builder.html
staging URL: /pages/boards-v2/board-builder.html

현재 Board Builder는 독립 Preview이며 App Shell/Left Nav/nav-data에 붙지 않는다. 입력 결과는 localStorage 기준 Preview이고, Firestore/API/fetch 호출이 없는 상태를 유지한다.
05

D. 협업/검토/승인 — 산출물 공통 레이어

산출물 공통 레이어

D그룹은 요구사항, WBS, 화면설계, 정책, 테스트 등 각 산출물에 붙는 검토/승인/댓글/변경이력 기능의 공통 레이어다.

전체 모아보기

개별 산출물 상세 안에도 표시되지만, D그룹 메뉴에서는 검토 요청·승인 요청·댓글·멘션·변경이력을 한곳에 모아 본다.

권한 단위

역할/담당/상태에 따라 문서·레코드·버튼·액션·필드·컴포넌트 단위 권한을 함께 고려한다.

06

메뉴 / 화면 추가 원칙

실제 repo 경로 기준메뉴, 링크, 화면 설계는 실제 존재하거나 생성 예정으로 승인된 경로만 사용한다.
nav-data SSOT 기준그룹명, 메뉴명, 순서, 카운트, href는 stam/js/stam.nav-data.js를 기준으로 맞춘다.
임의 경로 금지예상 파일명이나 가상의 route를 문서에 확정 경로처럼 쓰지 않는다. 미정이면 href 미정으로 표시한다.
권한 설계 병행화면/문서/버튼/액션/필드/컴포넌트 단위 권한과 승인 상태를 함께 검토한다.
모바일/좁은 화면 고려목록형 산출물은 PC 테이블과 모바일 카드/전체보기 흐름을 함께 설계한다.
제품 파일 신중 처리App Shell, Left Nav, Topbar, Firebase, API, 저장 로직은 별도 승인 작업으로 분리한다.
