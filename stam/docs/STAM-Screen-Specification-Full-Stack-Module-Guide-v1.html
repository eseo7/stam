01HeroPolicy
STAM Screen Specification Full Stack Module Guide v1 Candidate Full Stack Scope v1 Policy / Domain Module Guide

STAM 화면설계서 기능을 단일 화면 CRUD가 아닌 Full Stack Module로 정의하는 정책 Guide입니다. 문서 관리, 템플릿, 에디터(Authoring / Wireframe / Flowchart), 콜아웃, 검토/승인, 댓글, 버전관리, 저장/충돌, 권한, 로그, 내보내기를 포함합니다.

구현 코드 / DB 스키마 / 기술 엔진을 확정하지 않습니다. 모든 DB / 기술 항목은 Candidate로 표기합니다. 저장/충돌 정책은 Save & Conflict Handling Policy Guide v1에, 변경이력/복구 정책은 Change History / Recovery Policy Guide v1에 위임합니다.

02Purpose목적
  • STAM 화면설계서 기능의 전체 모듈 범위를 Full Stack Scope v1 기준으로 정의한다.
  • 단일 화면 CRUD를 넘어 에디터, 문서 관리, 템플릿, 검토/승인, 버전관리, 저장/충돌, 권한, 로그, 내보내기를 포함하는 모듈 경계를 명확히 한다.
  • 구현 순서는 분리할 수 있지만, 범위 제외처럼 표현하지 않는다. Full Stack Scope v1에 포함하되 구현 순서를 분리한다.
  • 저장/충돌 정책과 변경이력/복구 정책의 위임 기준을 명시하여 Guide 간 역할 경계를 확립한다.
  • 구현 코드 / DB 스키마 / 기술 선택을 확정하지 않으며, 후속 기능 요청 또는 Sprint Planning에서 확정한다.
03Scope범위
포함 — Full Stack Scope v1이 정의하는 것
  • 문서 관리 (생성/조회/수정/삭제/상태 전환)
  • 템플릿 관리 (기본 / 사용자 정의)
  • Authoring 에디터 범위
  • Wireframe 에디터 범위
  • Flowchart 에디터 범위
  • 콜아웃 & 설명 매핑 정책
  • 검토 / 승인 워크플로 정책
  • 댓글 / 협업 정책
  • 버전관리 / 변경이력 정책 (위임 기준)
  • 저장 / 충돌 처리 정책 (위임 기준)
  • 권한 / 역할 정책 기준
  • 로그 / 감사 이벤트 후보
  • 내보내기 정책 후보
미포함 — 이 문서가 확정하지 않는 것
  • 정식 DB 테이블 / 컬럼 / ERD 확정
  • 기술 엔진 확정
  • 실시간 동시 편집 구현 보장
  • 단일 액션 되돌림 보장 / 전면 복원 보장
  • PowerPoint / Figma급 범용 그리기 도구 직접 구현
  • 저장/충돌 처리 상세 정책 확정 (→ Save & Conflict Guide에 위임)
  • 변경이력/복구 상세 정책 확정 (→ Change History Guide에 위임)
  • 권한 전용 상세 정책 확정 (→ Permission Policy Guide에 위임)
  • 감사 로그 전용 상세 정책 (→ Audit Log Policy Guide에 위임)
  • 구현 API 확정
04Full Stack Scope v1 PrinciplesRule
  1. 1축소형 범위 명칭을 사용하지 않는다. 화면설계서 기능은 Full Stack Scope v1 기준으로 정의한다.
  2. 2기능 범위를 축소하지 않는다. Full Stack Scope v1에 포함하되 구현 순서를 분리한다.
  3. 3구현 순서 분리는 허용하지만, 범위 제외처럼 표현하지 않는다. "Full Stack Scope v1에 포함하되 구현 순서를 분리한다"는 표현을 사용한다.
  4. 4"나중에 붙인다"는 표현을 사용하지 않는다.
  5. 5DB 테이블 / 컬럼 / ERD는 확정하지 않는다. 후보로만 표기한다.
  6. 6기술 엔진은 후보로만 표기한다. 확정 언어를 사용하지 않는다.
  7. 7PowerPoint / Figma급 범용 그리기 도구를 직접 구현한다고 약속하지 않는다. STAM 도메인 위에 검증된 편집 엔진 후보를 결합하는 방향으로 표현한다.
  8. 8제품 차원의 단일 액션 되돌림 보장 / 전면 복원 보장을 약속하지 않는다.
  9. 9에디터 내부 임시 편집 이력은 후보로 표현할 수 있다. 단, Product-level recovery promise is not included 취지를 반드시 함께 표시한다.
  10. 10저장/충돌 정책은 Save & Conflict Handling Policy Guide v1에 위임한다.
  11. 11변경이력/복구 정책은 Change History / Recovery Policy Guide v1에 위임한다.
  12. 12모든 정책은 화면 / 업무 / 권한 / 데이터 민감도에 따라 달라질 수 있다. 절대 규칙으로 표현하지 않는다.
  13. 13구현 확정은 하지 않으며, 후속 기능 요청 또는 Sprint Planning에서 확정한다.
05What This Guide Does Not PromiseRule
✕PowerPoint / Figma급 범용 그리기 도구 직접 구현을 약속하지 않는다.
✕실시간 동시 편집 기능 구현을 약속하지 않는다.
✕제품 차원의 단일 액션 되돌림 보장 / 전면 복원 보장을 약속하지 않는다.
✕모든 충돌을 자동 병합한다고 약속하지 않는다. (→ Save & Conflict Guide에 위임)
✕DB 테이블 / 컬럼 / ERD 확정을 약속하지 않는다. 모두 Candidate.
✕기술 엔진 확정을 약속하지 않는다. 모두 Candidate.
✕저장/충돌 상세 정책을 이 문서에서 확정하지 않는다. Save & Conflict Guide에 위임.
✕변경이력/복구 상세 정책을 이 문서에서 확정하지 않는다. Change History Guide에 위임.
범위 한계 본 문서는 Full Stack Scope v1 정책 경계 문서이며, 구현 상세는 후속 기능 요청 또는 Sprint Planning에서 확정한다.

07Document Management ScopePolicy
CRUD / 메타데이터
  • 화면설계서 문서 생성 / 조회 / 수정 / 삭제 후보
  • 메타데이터 후보: 제목, 상태, 버전, 소유자, 프로젝트 연결, 생성일, 수정일
  • 문서 목록 조회, 검색, 필터링 후보
  • 문서 복사 / 복제 후보
  • 문서 잠금 / 잠금 해제 후보 (편집 충돌 방지 후보)
상태 전환 후보
  • Draft → In Review → Approved → Archived 후보
  • Rejected 상태 후보 (검토 반려)
  • 승인 이후 문서 편집 제한 정책은 화면/업무 기준에 따라 달라질 수 있다
  • 상태 전환 이력은 변경이력 후보 — Change History Guide에 위임
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
08Template Management ScopePolicy
  • 템플릿 생성 / 조회 / 수정 / 삭제 후보
  • 기본 제공 템플릿 / 사용자 정의 템플릿 구분 후보
  • 템플릿에서 문서 생성 후보 — 템플릿 콘텐츠를 복사하여 신규 문서 생성
  • 템플릿 공유 범위 후보: 개인 / 팀 / 전체 공개
  • 템플릿 권한 정책은 Permission / Role Policy와 연동
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
09Authoring Editor ScopePolicy
편집 기능 후보
  • 텍스트 블록 편집 (서식 포함) 후보
  • 섹션 / 페이지 구조 편집 후보
  • 구성 요소 삽입 후보: 텍스트, 이미지, 표, 링크, 콜아웃
  • 블록 순서 변경 / 복제 / 삭제 후보
  • 에디터 내 임시 편집 이력 후보 (Editor-local history candidate)
에디터 범위 한계
  • 에디터 엔진은 검증된 오픈소스 후보를 결합하는 방향 (→ S22 Technical Engine Candidates)
  • Editor-local editing history candidate — 에디터 내부 임시 편집 이력 후보. 제품 차원의 전면 복원 보장 / 단일 액션 되돌림 보장은 포함하지 않는다.
  • 저장 / 충돌 처리는 Save & Conflict Guide에 위임
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
10Wireframe Editor ScopePolicy
편집 기능 후보
  • 화면 레이아웃 요소 배치 / 이동 / 크기 조정 후보
  • 기본 UI 컴포넌트 (버튼, 입력, 카드, 모달 등) 배치 후보
  • 와이어프레임 페이지 단위 편집 후보
  • 요소 간 연결선 / 흐름 표시 후보
  • 콜아웃 연결 후보 (→ S12 Callout & Description)
범위 한계 — 반드시 확인
  • PowerPoint / Figma급 범용 그리기 도구를 직접 구현한다고 약속하지 않는다.
  • STAM 도메인 위에 검증된 Canvas 엔진 후보를 결합하는 방향 (→ S22)
  • Editor-local editing history candidate — 에디터 내 임시 편집 이력 후보. Product-level recovery promise is not included.
  • 저장 / 충돌 처리는 Save & Conflict Guide에 위임
Product-level recovery promise is not included — 와이어프레임 에디터 내부 임시 편집 이력은 후보이며, 제품 차원의 전면 복원 보장 / 단일 액션 되돌림 보장은 포함하지 않는다.
11Flowchart Editor ScopePolicy

  • 플로우차트 레이아웃 자동 배치 후보
  • 화면설계서 페이지와 플로우차트 연결 후보

  • 에디터 엔진은 검증된 Graph 엔진 후보 결합 방향 (→ S22 Technical Engine Candidates)
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
12Callout & Description Mapping PolicyPolicy
매핑 기준
  • 화면 요소(와이어프레임 요소 / 텍스트 블록)에 콜아웃(번호 / 라벨)을 연결하는 정책
  • 콜아웃 번호 / 라벨 자동 생성 후보
  • 콜아웃과 설명 텍스트의 1:N 또는 N:1 매핑 구조 후보
  • 콜아웃 편집 / 연결 해제 후보
동기화 정책 후보
  • 화면설계서 본문과 콜아웃 설명의 동기화 정책 후보
  • 콜아웃 삭제 시 연결 설명 처리 정책 후보
  • 콜아웃 변경은 변경이력 후보 — Change History Guide에 위임
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
13Review / Approval Workflow PolicyPolicy
워크플로 기준
  • 검토 요청 생성 후보 (요청자 지정, 문서 범위 지정)
  • 검토자 지정 / 알림 후보
  • 검토 상태 전환: 요청됨 → 검토 중 → 승인 / 반려
  • 반려 시 사유 기록 후보
  • 복수 검토자 / 단계별 승인 후보
엄격 처리 기준
  • 승인 이후 문서 편집 제한 정책 후보
  • 관리자 승인 대상은 엄격 처리 — Save & Conflict Guide 기준 적용
  • 승인/반려 이벤트는 감사 로그 후보 — Audit Log Guide에 위임
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
14Comment / Collaboration PolicyPolicy
  • 댓글 작성 / 수정 / 삭제 후보
  • 댓글과 화면 요소(콜아웃, 섹션, 와이어프레임 요소) 연결 후보
  • 댓글 상태: 열림(Open) / 해결됨(Resolved) 후보
  • 댓글 멘션(@) 후보
  • 댓글 알림 후보
  • 실시간 동시 편집은 약속하지 않는다. 협업은 저장 / 댓글 기반으로 정의한다.
  • 댓글 변경은 변경이력 후보 — Change History Guide에 위임
  • Full Stack Scope v1에 포함. 구현 순서 분리 가능.
15Versioning / Change History Policy위임
  • 저장 성공 시 버전 / 이력 후보 생성
  • 수동 버전 고정 (Snapshot) 후보
  • 버전 목록 조회 / 버전 비교 후보
  • 버전 복구 정책 기준은 Change History / Recovery Policy Guide v1에 위임
  • 제품 차원의 단일 액션 되돌림 보장 / 전면 복원 보장은 약속하지 않는다.
  • Field Diff / 복구 요청 / 관리자 복구 / Snapshot 상세 기준은 Change History Guide를 따른다
위임 대상 — 버전관리 / 변경이력 / 복구 정책 상세 전체는 Change History / Recovery Policy Guide v1의 범위이다.
16Save / Conflict Handling Policy위임
전체 위임 — 저장 요청, 다중 탭 저장, Last Save Wins, Duplicate Request 차단, Strict Handling Areas, 저장 실패/재시도 정책 전체는 Save & Conflict Handling Policy Guide v1의 범위이다. 이 섹션은 위임 선언만 포함한다.
  • 화면설계서 저장 요청 정책은 Save & Conflict Guide를 따른다
  • 다중 탭 저장 / Last Save Wins는 Save & Conflict Guide를 따른다
  • 권한 변경 / 승인 저장 등 엄격 처리 대상은 Save & Conflict Guide의 Strict Handling Areas 기준을 따른다
  • 충돌 후보 처리 기준은 Save & Conflict Guide를 따른다
