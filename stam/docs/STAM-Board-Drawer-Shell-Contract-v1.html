STAM Board Drawer Shell Contract v1

게시판 상세/등록/수정 Drawer 의 외곽 구조 · header · body · footer · close · scrim · Escape · action button 기준을 공통 stam.drawer.css 기반으로 통일하기 위한 계약 문서.

버전: v1 공통 CSS: stam.drawer.css 기준 PR: #115 Drawer Audit · #116 Form Field Contract 성격: Drawer Shell 계약
Related: Drawer Audit v1 Form Field Contract v1 Drawer Token Mapping v1 Board List Contract v1 Drawer Guide Docs Index
Section 1 · Summary

Drawer 전체를 즉시 공통화하지 않고, shell / header / footer / action 기준부터 계약화한다.

신규 게시판 Drawer 는 본 계약(공통 stam.drawer.css = .stam-drawer · .stam-drawer-scrim · .stam-drawer-head/-body/-foot · .stam-drawer-title/-subtitle)을 우선 따른다. 본문 폼은 Form Field Contract, 본문 목록 상호작용은 Board List Contract 와 연계.

  • 현행 fn/os/msl/rq 는 루트에 .stam-drawer + scrim .stam-drawer-scrim 을 이미 병행(.open/.show class 토글).
  • ss/wbs 는 단일 drawer + mode 전환(JS 렌더 / data-mode) — 특수 구조로 별도 검토(Section 10).
  • 본 계약은 문서다. 제품 CSS/JS 미수정, 화면별 적용은 별도 코드 PR. 새 class 는 "문서 제안" 으로만.

Section 4 · Header 계약
  • title: .stam-drawer-title (17px·700). 항목명 또는 "신규 등록".
  • subtitle/meta: .stam-drawer-subtitle (12px·t3) — id · 담당자 · 최종수정 등 보조 정보.
  • status badge/chip: 상태/우선순위 chip 은 화면 도메인 chip 유지(시각 토큰만 공통). header meta 영역에 배치.
  • close button 위치: header 우상단 고정. aria-label="닫기" 필수. mode 무관 항상 노출.
  • optional action 영역: header 우측에 보조 action(예: 즐겨찾기) 둘 수 있으나, 주요 action 은 footer 에 둔다.
Section 5 · Body 계약
  • scroll 영역: .stam-drawer-body 만 스크롤(flex:1; overflow-y:auto). header/footer 고정.
  • section 구분: 섹션(번호·제목)은 화면별 레이아웃 허용(.fn-fs/.rq-fs 등). 구조는 자유, 필드는 공통.
  • form field contract 연결: 입력 필드는 Board Form Field Contract(.stam-form-field/.stam-label/.stam-input/...) 사용.
  • tab 영역: detail 에 탭 허용(fn 3 / rq 4 / os 6). 탭 컴포넌트 자체 공통화는 후속(현재 화면별 class 유지 허용).
  • empty/readonly/display: detail 값 표시는 .is-readonly input 또는 표시 전용 마크업. 빈 값은 "—" 등 일관 표기.
Section 6 · Footer 계약

.stam-drawer-foot = display:flex; gap:8px. 좌측 = 메타/보조, 우측 = action 버튼(사이 flex:1 spacer). 버튼은 .stam-btn 계열.

action버튼 class배치
닫기/취소.stam-btn .stam-btn-ghost (secondary)우측 그룹 좌단
임시저장(선택).stam-btn .stam-btn-outline취소 옆
저장 / 등록.stam-btn .stam-btn-primary우측 끝(primary, 1개)

Section 7 · Open / Close 계약
트리거동작
row clickdetail drawer open (목록은 Board List Controller onRowActivate → 화면 drawer open). row 는 .is-active.
등록 버튼create/register drawer open (선택 행 없음).
상세 내 수정 버튼edit mode 전환(data-*-open="edit" 또는 setMode('edit')).
close buttondrawer 닫기(.open 제거) + scrim .show 제거.
scrim clickclose 와 동일.
Escape열린 custom-select 우선 닫기 → 없으면 drawer 닫기.
close 시 active row.is-active 만 해제(Board List api.clearActive()), .is-selected/checkbox/삭제 카운트는 유지.

* open/close 메커니즘은 .open/.show class 토글로 통일 권장. WBS 의 data-open attr 방식은 특수 예외(Section 10).

Section 8 · 접근성 / 키보드 기준
  • focus 이동: open 시 drawer 내 첫 포커서블(또는 close)로 이동, close 시 트리거(row/버튼)로 복귀 권장.
  • Escape: custom-select 우선 → drawer 닫기(Section 7).
  • aria: drawer 루트 role="dialog" + aria-modal="true" + aria-label(또는 title id 를 aria-labelledby).
  • button title: close 는 aria-label="닫기", 아이콘 전용 버튼은 aria-label/title 필수.
  • focus trap: tab focus trap(drawer 내 순환)은 향후 과제 로 표시 — 현재 미구현, 후속 공통 JS 검토.

Section 10 · 예외 처리 기준 (특수 구조)
  • 화면설계서: 단일 drawer + S.dwMode 기반 JS 전체 렌더(renderDetail/Register/Edit) + 별도 full editor view(#ss-editor-view). 골격이 JS 문자열 생성 → 공통 shell 마크업 직접 적용 어려움. 구조 검토 선행.
  • WBS: 단일 drawer + data-mode CSS visibility, 전체보기 inline(#wbs-fv-inline), Gantt, 댓글/이력 inline 섹션. open/close 가 data-open attr. WBS Contract 트랙에서 별도 관리.
  • Open Scenario: 현재 detail only — 등록/수정 신설 시 본 계약 따름.
  • → 특수 구조는 별도 계약 또는 구조 검토 선행(Board List 트랙과 동일 원칙). 본 shell 계약 일괄 강제 적용 금지.
Section 11 · 금지 사항

✅ Do

  • 공통 .stam-drawer / -scrim / -head / -body / -foot 사용
  • open/close = .open/.show class 토글
  • footer 순서: 취소 → 보조 → primary(1개)
  • close 시 .is-active 만 해제(선택 유지)
  • body 폼은 Form Field Contract 사용

❌ Don't

  • 화면별 drawer shell 새로 만들기
  • close/Escape 동작 제각각 구현
  • footer button 순서/타입 제각각 구현
  • 저장/API/localStorage/Firestore 로직 변경
  • 제품 CSS(stam.drawer.css 등) 수정 / 복붙
  • stam.board-list.js 수정
Section 12 · QA 체크리스트
  1. drawer open — .open + scrim .show, 우측 슬라이드 인.
  2. close button — 닫힘 + scrim 제거.
  3. scrim click — 닫힘.
  4. Escape close — custom-select 우선 → drawer.
  5. active row 처리 — close 시 .is-active 만 해제, .is-selected/카운트 유지.
  6. footer button 표시 — mode별 순서/타입(취소→보조→primary).
  7. detail/create/edit mode 전환 — header title/footer 기준 일치.
  8. form field contract 유지 — 본문 폼 .stam-* 클래스.
  9. console error 0.
  10. 제품 코드 변경 없음(본 문서는 docs-only).
