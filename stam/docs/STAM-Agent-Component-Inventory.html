역할: Agent / 작업자용 거버넌스 문서

This inventory is the governance reference for agents and implementation work. For human-readable component usage guidance, refer to STAM Component Guide. For token meaning and tone rules, refer to STAM Token Guide.

Cursor / Claude Code / Claude Design가 STAM 화면 작업 시 새 CSS·JS·DOM을 만들지 않고 기존 공통 컴포넌트를 찾아 조립하도록 돕는 인벤토리 문서입니다. 새 화면은 그리는 것이 아니라 기존 STAM 공통 컴포넌트를 조립한다.

01 목적 Governance Draft
  • 에이전트가 STAM 작업 전 반드시 확인해야 할 공통 CSS/JS/가이드 목록을 한곳에 모은다.
  • 임의 컴포넌트 생성·inline style/script·신규 CSS/JS 파일을 방지한다.
  • 화면 작업은 기존 클래스·함수·DOM 패턴의 조립으로만 수행한다.
  • 프로젝트 루트 CLAUDE.md 및 .cursor/rules/stam-agent-governance.mdc와 동일 기준을 따른다.
02 에이전트 작업 규칙 필수
  • 새 화면은 그리는 것이 아니라 기존 STAM 공통 컴포넌트를 조립한다.
  • 작업 전 사용할 기존 클래스/함수 목록을 먼저 보고한다.
  • diff에 새 CSS/JS 파일 또는 inline style/script가 있으면 작업 실패다.
  • 필요한 컴포넌트가 없으면 임의 생성하지 말고 멈춰서 보고한다.
  • 버튼 작업 시 — Before adding or changing any button, agents must check STAM Button Guide. Do not create new button classes, tokens, JS behavior, or DOM structures unless explicitly approved.
03 필수 확인 공통 CSS SSOT

제품 화면 작업 시작 전 아래 파일을 읽고 사용할 클래스를 선언한다.

파일역할대표 클래스/토큰
stam/css/stam.tokens.css 디자인 토큰 SSOT --brand, --t1, --bd, --r-md, --font, --sw
stam/css/stam.shell.css 앱 셸·Topbar·Sidebar·문서 nav .po-sidebar, .po-topbar, .doc-nav, .tbtn
stam/css/stam.components.css 공통 UI 컴포넌트 .stam-btn--*, .sbadge, .stam-summary-strip, .stam-icon
금지 위 파일 외 신규 CSS 생성, inline style, 하드코딩 색상/간격/폰트. 화면별 전용 CSS 수정도 승인 없이는 금지.
04 문서(docs) 전용 CSS 제품 화면 사용 금지

문서(docs) 전용 스타일. 제품 화면에서는 사용하지 않는다.

파일역할용도
stam/css/stam.docs.css 문서 공통 스타일 docs 전용 / 제품 화면 사용 금지
stam/css/stam.ui-baseline-guide.css UI Baseline Guide 전용 스타일 docs 전용 / 제품 화면 사용 금지
05 필수 확인 공통 JS SSOT

제품 화면 작업 시작 전 아래 파일을 읽고 사용할 API를 선언한다.

파일역할대표 API
stam/js/stam.theme.js 다크/라이트 테마 STAM.toggleTheme(), STAM.setTheme(), STAM.getTheme()
stam/js/stam.shell.js Sidebar·Root Cards·Menu Table STAM.shell.renderSidebar(), renderRootCards(), renderMenuTable(), initSearch()
stam/js/stam.nav-render.js Left Nav 공통 렌더 STAM.navRender.init('메뉴코드')
stam/js/stam.topbar-render.js Topbar 공통 렌더 STAM.topbarRender.init()
stam/js/stam.project-context-render.js Project Context Bar 공통 렌더 STAM.projectContextRender.init()

DOM 계약 예시

  • <nav class="po-sidebar" data-stam-left-nav> + STAM.navRender.init('A1')
  • <header class="po-topbar stam-topbar" data-stam-topbar data-tb-crumbs="..."> + STAM.topbarRender.init()
  • <div class="po-ctx" data-stam-project-context ...> + STAM.projectContextRender.init()
  • data-theme-toggle 버튼 + STAM.toggleTheme()

08 화면 작업 전 보고해야 할 것 필수

작업 전 사용할 기존 클래스/함수 목록을 먼저 보고한다.

항목내용
사용할 기존 CSS 파일 예: stam/css/stam.tokens.css, stam/css/stam.shell.css, stam/css/stam.components.css (+ 화면 전용 CSS가 이미 있으면 해당 파일명)
사용할 기존 JS 파일 예: stam/js/stam.theme.js, stam/js/stam.nav-render.js, stam/js/stam.topbar-render.js, stam/js/stam.project-context-render.js
사용할 기존 클래스 예: .stam-btn--primary, .po-sidebar, .stam-summary-strip
수정 대상 파일 예: stam/pages/A/A1-dashboard.html
새 CSS/JS 생성 여부 0건 (1건 이상이면 작업 중단)
09 FAIL 조건 FAIL
  • diff에 stam/css/** 또는 stam/js/** 신규 파일 포함
  • diff에 inline style="", <style>, inline <script> 본문 포함
  • 하드코딩 색상·간격·폰트 (토큰 미사용)
  • 승인 없이 공통 CSS/JS 확장
  • 화면 HTML 안에 새 컴포넌트 정의
  • 가이드에 없는 컴포넌트 임의 생성
  • Button Guide와 충돌하는 버튼 배치·유형
작업 실패 판정 diff에 새 CSS/JS 파일 또는 inline style/script가 있으면 작업 실패다.
