Section 1 · Summary

최종 판정: 일부 공통화 가능  (Drawer shell / header / footer / form-field 레이어 우선, 전면 공통 Drawer 계약 일괄 적용은 보류)

6개 화면 모두 .stam-drawer · .stam-drawer-scrim · .stam-label/.stam-input · .stam-btn 을 부분 병행하나, 실제 Drawer 골격·헤더·푸터·필드는 화면 전용 namespace(.fn-* / .os-* / .msl-* / .rq-* / .ss-* / .wbs-*) 비중이 70~90% 로 높다. Drawer 패턴이 2종(멀티 drawer instance vs 단일 drawer + mode 전환)으로 갈리고, 화면설계서는 JS 전체 렌더라 전면 공통 계약 일괄 적용은 위험하다. 단 shell/header/footer/form-field 레이어는 공통 계약화 여지가 크다.

공통화 우선순위:

  1. Form field 레이어 — 이미 Form Controls(.stam-label/.stam-input/.stam-textarea) 일부 사용 중. required 표기·grid 까지 수렴 (최저 위험).
  2. Drawer shell + scrim + open/close — .stam-drawer 병행 기반 표준화. open/close 메커니즘 2종 통일이 핵심.
  3. Drawer header / footer 레이아웃 — badge·title·meta·close / 좌측메타·우측액션 구조가 화면별로 유사.
  4. 탭 컴포넌트 — detail drawer 탭 패턴 공통화(선택).
  5. 멀티 drawer 3종(fn/msl/rq) 우선 적용 후보(정적·쌍둥이 다수), ss/wbs 는 특수 구조 별도 검토.

Section 6 · 화면별 특수 구조 (현행 유지/별도 검토)
  • 화면설계서 — 단일 drawer + S.dwMode 기반 JS 전체 렌더(renderDetail/Register/Edit/ReviewReq), 별도 full editor view(#ss-editor-view). 골격이 JS 문자열 생성이라 공통 마크업 템플릿 적용이 어렵다. (Board List 에서도 "특수 구조 현행 유지"로 분류된 화면)
  • WBS — 단일 drawer + data-mode CSS visibility, 전체보기 inline 패널(#wbs-fv-inline), 댓글/변경이력 inline 섹션(탭 없음), 폼 전용 class(.wbs-drawer-form-*). open/close 가 data-open attr 로 유일하게 다름.
  • Open Scenario — 등록/수정 drawer 없음(읽기전용 상세 + 6탭 placeholder). 등록/수정 추가 시 공통 계약 따르도록 권장.

Section 8 · 권장 표준 Drawer 계약 초안 (Draft)

전면 일괄 적용이 아닌 레이어별 점진 수렴 기준 초안. (확정 계약은 후속 PR 에서 문서화)

  • Shell: 루트 .stam-drawer + scrim .stam-drawer-scrim, open 상태는 .open(class) 로 통일 권장(wbs data-open 은 특수 예외로 검토).
  • Header: .stam-drawer-head (badge / title / meta(chip) / close 버튼) 표준 슬롯.
  • Body: 섹션/탭은 화면별 자유, 단 폼은 Form Controls 계약(.stam-form-field · .stam-label · .stam-input/.stam-textarea/.stam-select · required) 사용.
  • Footer: .stam-drawer-foot (좌측 메타 + 우측 액션), 버튼은 .stam-btn 계열.
  • 전환: 멀티 drawer / 단일 drawer+mode 둘 다 허용하되, 상세→수정 트리거는 data-*-open="edit" 또는 setMode('edit') 로 명시.
  • Escape/close: custom-select 우선 닫기 → drawer 닫기 패턴 유지(이미 fn/msl/rq/ss/os 유사).

* 이 초안은 판정 근거용 스케치이며, 실제 계약은 별도 Contract 문서(후속)에서 확정한다. Drawer 토큰은 Drawer Token Mapping v1 와 연계.
