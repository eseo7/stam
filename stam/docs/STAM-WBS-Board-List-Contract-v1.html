STAM WBS Board List Contract v1

WBS 목록 화면은 공통 STAMBoardList 컨트롤러를 직접 적용하지 않고 자체 로직으로 운영하되, Board List 상호작용 계약은 동일하게 충족하도록 강제하는 WBS 전용 계약 문서.

Section 1 · Summary

STAMBoardList 직접 적용 보류 WBS 특수 구조 · 현행 유지 자체 WBS 계약으로 관리

WBS 는 22컬럼 · 그룹별 다중 <tbody> · JS 주입 checkbox · Gantt/focus/전체보기 구조 때문에 평면형 STAMBoardList 직접 적용은 보류한다. 다만 PR #111 이후 row=active / checkbox=selected 분리 원칙은 충족한다. WBS 는 당분간 본 계약 문서 기준으로 유지하며, STAMBoardList 확장 설계는 장기 과제(별도 PR)로만 검토한다. stam.board-list.js 는 이 트랙에서 수정하지 않는다.

본 계약은 공통 Board List Contract v1 의 상호작용 규칙(row=active / checkbox=selected / 동시 상태 / Escape 분리 / delete count)을 WBS 자체 구현으로 충족시키기 위한 WBS 한정 명세다. 마크업 hook(.stam-select-table · data-stam-board-list · data-stam-check-all · .stam-delete-label)은 WBS 에 적용하지 않는다(공통 컨트롤러 미사용).

Section 3 · WBS 전용 Board List 계약

WBS 목록은 아래 규칙을 반드시 충족한다. (PR #111 반영 기준)

3-1. row click 계약

  • 데이터 행(.wbs-data-row) 본문 클릭(체크박스 셀 .wbs-td-chk 제외) → 클릭 행에만 .is-active 부여.
  • 다른 행의 .is-active 는 해제(active = 상세 drawer 대상, 한 번에 하나).
  • row click 은 checkbox checked 상태와 delete count 를 변경하지 않는다. .is-selected 를 부여/제거하지 않는다.
  • row click 시 상세 drawer 가 열린다.

3-2. checkbox 계약

  • 행 checkbox(.wbs-row-chk) 변경 → 해당 행에 .is-selected(+ 레거시 .selected) 토글 + checked 반영.
  • checkbox click 은 drawer 를 열지 않는다(버블링 차단).
  • checkbox click 은 .is-active 를 부여하지 않는다.
  • 선택 수에 따라 delete count 갱신(3-6).

3-3. selected + active 동시 상태 계약

  • 이미 checkbox 로 선택된(.is-selected) 행을 row click 하면 .is-selected.is-active 동시 상태가 가능해야 한다.
  • 이때 checkbox checked 와 delete count 는 유지된다(active 부여가 선택을 건드리지 않음).

3-4. header checkbox / indeterminate 계약

  • 전체선택 checkbox(#wbs-chk-all) 체크 → 모든 행 checkbox checked + .is-selected + count 반영. 해제 → 전체 해제.
  • 일부만 선택 시 header 는 indeterminate 표시.
  • 전체 선택 시 header checked, 0개 시 checked=false / indeterminate=false.

3-5. drawer close / Escape 계약

  • drawer close(닫기 버튼 · overlay 클릭 · Escape) 시 .is-active 만 해제한다.
  • .is-selected · checkbox checked · delete count 는 유지한다.

3-6. delete count 계약

  • 삭제 버튼(#wbs-delete-btn)은 선택 0개 시 disabled + 라벨 삭제, N개 시 enabled + 삭제 N.
  • 카운트 기준은 checkbox checked 수(.wbs-row-chk:checked). row click(active)은 카운트에 영향 없음.
  • 표기 차이 공통 계약은 삭제 (N)(.stam-delete-label) 이나 WBS 는 자체 라벨 삭제 N 표기를 사용한다. 동작은 동일하며, 표기/마크업 정합은 향후 컨트롤러 확장 시 정리한다.
Section 4 · group / Gantt / focus toggle 관련 주의사항
  • 그룹 헤더행(.wbs-grp-hdr-body)은 데이터 행이 아니다 — .is-active/.is-selected 대상이 아니며 row click 계약을 적용하지 않는다. 그룹 토글 버튼(.wbs-grp-toggle-btn)은 접기/펼치기 전용.
  • 그룹 접기/펼치기로 행이 숨겨져도 선택 상태(checkbox/count)는 보존된다. 접기/펼치기 동작 자체는 변경 금지.
  • 전체 펼치기/접기 버튼 동작 유지.
  • Gantt 섹션 접기/펼치기, focus 모드(data-focus-toggle/data-focus-exit, .is-focus-col/.is-shared-col 표시 토글), 드로어 전체보기는 목록 선택/active 계약과 독립적으로 기존 동작 유지.
  • 가로 스크롤 구조(.wbs-tbl-inner) 유지 — 컬럼/레이아웃 변경 금지.
  • portal 마운트 custom select(PR #100)는 별도 트랙이며 본 계약과 무관하게 유지.
Section 5 · 금지 사항

✅ Do

  • row click = .is-active 만, checkbox = .is-selected 분리 유지
  • drawer close / Escape 시 .is-active 만 해제
  • delete count 는 checkbox 기준
  • WBS 변경은 본 계약 + Structure Review 기준으로 QA

❌ Don't

  • row click 에서 .is-selected/.selected 부여·제거 (PR #111 이전 회귀)
  • row click 으로 checkbox checked/count 변경
  • checkbox click 으로 drawer 열기
  • stam.board-list.js 수정 / WBS 에 평면 컨트롤러 강제 적용
  • 그룹 다중 tbody · 주입형 checkbox · Gantt/focus/전체보기 · 가로스크롤 구조 변경
  • 저장/삭제/API/localStorage/Firestore 로직, WBS 데이터/컬럼 구조 변경
Section 6 · QA 체크리스트
  1. WBS 화면 status 200 / 콘솔 오류 0.
  2. row click → 클릭 행 .is-active / checkbox unchecked 유지 / .is-selected 미부여 / delete count 미증가 / drawer open.
  3. 다른 row click → 이전 행 .is-active 해제 · 새 행 .is-active / checkbox·count 무영향.
  4. checkbox click → .is-selected / checked / 삭제 1 / drawer 미오픈 / .is-active 미부여.
  5. selected row click → .is-selected.is-active 동시 / checked·count 유지 / drawer open.
  6. Escape · drawer close → .is-active 만 해제 / .is-selected·checked·count 유지.
  7. header checkbox 전체선택 / 전체해제 / 부분선택 indeterminate.
  8. 그룹 접기/펼치기 · 전체 펼치기/접기 · Gantt 접기 · focus toggle · 드로어 전체보기 smoke.
  9. 가로 스크롤 정상 · portal custom select 정상.
  10. 저장/삭제/API 관련 동작 변경 없음 · 다른 게시판 화면 변경 없음.

* 검증은 Playwright(Chromium) 실측 + Bare 비교 권장. PR #111 재검증에서 위 2~7 항목 30/30 PASS, 콘솔 오류 0 확인.
