Screen Template
Selector UX Flow Prototype v1

사용자가 어디서 화면설계서 템플릿을 선택하고, 어떤 정보를 가져오며, 어떤 미리보기를 보고, 어떤 초안을 생성하는지의 UX 흐름을 정리한 프로토타입(prototype) 기준 문서다. 이 문서는 제품 기능을 구현하거나 제품 파일을 수정하지 않는 docs-only UX flow 문서이며, 후속 제품 보정 시 템플릿 선택 · 미리보기 · 초안 생성 흐름의 기준으로 사용한다. P1~P4(Master Guide · Priority 30 · Top 10 Detail Pack · Integration Blueprint)에서 정의한 템플릿 체계와 연결 설계를 실제 선택 화면 흐름으로 잇는 P5 단계다. 기준 커밋은 main 8015834 (PR #173 merge)이다.

00

왜 Selector UX Flow가 필요한가

문서 목적

P1~P4는 무엇을(템플릿) · 어떻게 연결할지(필드↔화면)를 정의했다. 이 문서는 사용자가 그 템플릿을 실제로 어떻게 고르고 초안을 만들지의 화면 흐름을 정의한다.

Selector가 필요한 이유

템플릿 30종 · Top 10 상세 기준이 있어도, 선택 진입점 · 카드 구조 · 필드 소스 · 미리보기 · 초안 생성 흐름이 합의되지 않으면 제품 보정 시 화면마다 흐름이 갈라진다.

P1~P4와의 관계

P1 원칙 · P2 30종 목록 · P3 Top 10 상세 · P4 Board Builder 연결을 선택 UX로 묶는 상위 흐름 문서. 카드 내용·매핑은 P2·P3·P4를 그대로 인용한다.

이 문서의 성격

실제 Selector 기능 · Board Builder 기능 · 화면설계서 제품 기능을 구현하지 않는다. 화면을 그리는 wireframe/pseudo-screen은 흐름 이해용이며 제품 코드가 아니다. merge/deploy는 사용자 승인 전 대기한다.

용어 기준: 이 문서는 단계 범위를 표현할 때 "1차 범위 / 우선 적용 / 후속 단계" 표현을 사용한다. 자동 생성 결과는 모두 초안이며 Master Guide의 "선택 → 수정" 원칙에 따라 사용자가 보정한다.
01

Entry Points — 템플릿 선택 진입 경로

화면설계서 템플릿 선택으로 진입하는 경로를 구분한다. 1차 범위는 ①~③이고, ④는 참조 진입, ⑤는 후속 단계 진입이다.

#진입 경로시작 위치(실제 repo)도달 흐름범위
E1새 화면설계서 만들기../pages/boards/screen-specification.html목록 → 새로 만들기 → 템플릿 선택 (Flow A)1차 범위
E2템플릿으로 시작../pages/boards/screen-specification.html목록 상단 "템플릿으로 시작" → 선택기 직접 진입 (Flow A)1차 범위
E3화면설계서 초안 생성../pages/boards-v2/board-builder.html게시판/필드 정의 → "화면설계서로 보내기" → 추천 템플릿 (Flow B)1차 범위
E4가이드/Index 참조STAM-Docs-Index.html · 가이드 문서문서에서 템플릿 체계 이해 → 제품 진입은 E1~E3으로 이동참조
E5컨텍스트 진입프로젝트 / 메뉴 / 산출물 컨텍스트특정 프로젝트·산출물에서 화면설계서 추가 → 선택기 (컨텍스트 사전 채움)후속 단계
E1·E2는 화면설계서 제품 페이지에서 시작하는 Flow A, E3은 Board Builder에서 시작하는 Flow B로 이어진다. E4는 docs 참조일 뿐 선택기 자체를 열지 않는다. E5는 프로젝트/산출물 컨텍스트를 선택기에 사전 주입하는 후속 단계 경로다. 모든 경로는 동일한 Template Selector(섹션 04)로 수렴한다.
02

User Flow A — Screen Specification First

E1 · E2

화면설계서 제품 페이지에서 시작해 템플릿을 고르고 초안을 만드는 흐름이다.

1
화면설계서 목록 진입Screen Template

화면설계서 목록(../pages/boards/screen-specification.html)에 진입한다.

2
새 화면설계서 만들기사용자

"새 화면설계서 만들기" 또는 "템플릿으로 시작"을 누른다.

3
템플릿 선택 모달/페이지 진입Screen Template

Template Selector(섹션 04 레이아웃)가 열린다. 컨텍스트(프로젝트/산출물)가 있으면 헤더에 표시된다.

4
카테고리 또는 추천 템플릿 선택사용자

Category Tabs(List/Detail/Form/Search/Dashboard/Approval/Settings) 또는 상단 Recommended(Top 10)에서 카드 선택.

5
필드 구조 선택 방식 결정사용자

Field Source(섹션 07): Board Builder 가져오기 / 기존 구조 / 직접 입력 / 샘플 / 빈 템플릿 중 선택.

6
미리보기 확인Screen Template

Preview Panel(섹션 08)에서 화면 유형·필드·액션·상태·모바일 변환 예시 확인.

7
초안 생성Screen Template

"이 템플릿으로 시작" → Draft Generation Result(섹션 09) 구조로 화면설계서 초안 생성.

8
에디터에서 보정사용자

액션·권한·상태·문구·연결 화면을 맥락에 맞게 수정(선택 → 수정 원칙).

9
저장 또는 검토 요청사용자

화면설계서 저장 또는 검토/승인 흐름 연결(섹션 11).

10
내보내기 또는 산출물 연결사용자

내보내기 또는 관련 산출물(relation)과 연결. 저장/내보내기 실제 동작은 후속 제품 구현 범위.

03

User Flow B — Board Builder First

E3

Board Builder에서 게시판/필드를 정의한 뒤 화면설계서 템플릿으로 보내 초안을 만드는 흐름이다. Board Builder는 preview/local 기준(실제 DB 저장 아님)이다.

1
Board Builder 진입Board Builder

게시판 만들기(../pages/boards-v2/board-builder.html) 진입.

2
게시판 유형 선택Board Builder

카테고리·게시판명·코드(자동)·기본 템플릿 선택.

3
필드 추가/수정/정렬Board Builder

필드 추가, 드래그 정렬, 목록/필터/입력폼 표시 토글.

4
필드 타입/옵션/필수 설정Board Builder

12종 타입·옵션값·필수 여부·검증·권한 힌트 설정.

5
게시판 미리보기 확인Board Builder

우측 실시간 미리보기 확인, config(JSON) 출력/localStorage(로컬).

6
화면설계서 템플릿으로 보내기사용자

"화면설계서 초안 생성"으로 Template Selector 진입(필드 소스=Board Builder 사전 선택).

7
추천 템플릿 자동 제안Screen Template

Recommendation Logic(섹션 10) 기준으로 적합 템플릿 우선 노출.

8
목록/등록/상세/검색 초안 생성Screen Template

선택 유형별로 필드/컬럼/폼 초안 자동 구성(P4 매핑 기준).

9
화면설계서 미리보기 확인Screen Template

Preview Panel에서 생성 초안 확인.

10
에디터에서 보정사용자

권한·상태·문구·연결 화면을 보정(선택 → 수정). 저장/검토/내보내기는 Flow A 9~10과 동일하게 연결.

Flow B 1~5는 현재 Board Builder preview의 실제 동작 범위이고, 6단계 이후 화면설계서 초안 생성·미리보기는 후속 제품 구현 대상이다. 두 흐름 모두 Template Selector(섹션 04)로 수렴하며, Field Source(섹션 07)만 진입 시 기본값이 다르다(Flow B는 Board Builder 가져오기 기본).

Header

문서 제목 · 설명 · 현재 프로젝트/산출물 컨텍스트(있을 때). 컨텍스트는 E5 진입 시 사전 채움.

Category Tabs

List · Detail · Form · Search · Dashboard · Approval · Settings. 좁은 화면에서 가로 스크롤 또는 Select 전환(섹션 12).

Recommended Templates

Top 10을 우선 노출. Flow B는 Recommendation Logic 결과를 상단에 정렬.

Template Card Grid

카드 그리드(섹션 05 구조). PC 3열 내외, 모바일 1열 스택.

Filter / Search

카테고리·키워드·연계 여부(Board-linked)·권한 복잡도 등으로 필터. 결과 없음 상태는 섹션 13.

Preview Panel

선택 카드의 미리보기(섹션 08). 좁은 화면에서는 하단/별도 화면 전환.

Selected Template Summary

선택 템플릿 ID/이름 + 필드 소스 요약. 초안 생성 전 확인 지점.

Primary CTA

"이 템플릿으로 시작" — Draft Generation(섹션 09) 트리거.

Secondary CTA

"샘플 보기"(Sample) · "기준 문서 보기"(Top 10 Detail).

Recommendation Logic

UX 기준 · 구현 로직 아님

실제 구현 로직이 아니라 UX 추천 기준을 제안한다. Flow B에서 Board Builder 필드 신호를 바탕으로 적합 템플릿을 우선 노출하는 규칙이다.

신호(필드/구성)추천 템플릿근거
필드 수가 많음 (다수 visibleInTable)T05 Wide Data Table컬럼 다수 → 가로 스크롤/고정 컬럼 정책 필요
검색 조건이 많음 (다수 visibleInFilter)T02 Advanced Search List필터 필드 다수 → 검색 패널 중심
상태/담당자/기한 중심T03 Grouped Work Liststatus/user/date → 그룹/작업 목록 적합
승인자/검토자 존재 (user 역할)T08 Approval Detail View승인/검토 흐름 → 결재선 상세
커스텀 필드가 많음T27 Custom Field Settings필드 정의 관리 화면 필요
게시판 설정에서 시작T26 Board Settings게시판 기본/표시 설정 화면
추천은 제안일 뿐 강제가 아니다. 사용자는 Recommended를 무시하고 카테고리에서 직접 선택할 수 있다. 신호가 약하거나 충돌하면 기본값으로 T01 Basic Board List(목록) 또는 T07 Basic Detail View(상세)를 우선 노출한다.
11

Permission / Review / History Connection

Selector 단계에서 권한·검토·변경이력을 어떻게 표시할지 정의한다. 선택 시점에 미리 알려 초안 생성 후 누락을 줄인다.

권한 포인트 안내

선택 템플릿의 화면/필드/액션 권한 포인트를 카드·Preview에 안내. 관리자 전용(T26·T27)은 진입 단계 표시.

승인/검토 필요 표시

승인/검토 흐름이 필요한 템플릿(T08 등)은 Review-ready 배지로 표시.

변경이력 포함 표시

변경이력(Change History) 연결 여부 표시. 상세/폼 계열은 변경자·변경일 노출 권장.

댓글/멘션 필요 표시

협업(댓글/멘션) 필요 여부 표시. 검토 흐름과 함께 D그룹 공통 레이어 참조.

내보내기 포함 표시

내보내기(exportable) 포함 여부 표시. 권한 있는 액션만 노출.

연결 기준

권한 세부는 권한 정책 화면(후속)과 연결해 확정. 공통 레이어는 IA Overview 참조.

권한/검토/이력의 상세 기준은 Integration Blueprint의 Permission/Action/Field Control과 STAM Sitemap & IA Overview를 참조한다. Selector는 안내·표시까지만 하고 정책 확정은 후속 단계다.

Mobile / Narrow Width UX

Template Card

좁은 화면에서 1열 스택. 카드 정보는 ID·이름·용도·배지 우선, 부가 항목 접힘.

Category Tabs

가로 스크롤 또는 Select 드롭다운 전환. 탭 7종이 가로로 넘칠 때 Select 우선.

Preview Panel

그리드 옆이 아닌 하단 또는 별도 화면 전환(카드 탭 → 미리보기 화면).

CTA

"이 템플릿으로 시작" 등 하단 고정 바 가능. 스크롤과 무관하게 접근.

Wide Table 미리보기

T05는 요약 카드 + 가로 스크롤 정책 표시. 전체 표를 모바일에 그대로 넣지 않음.

Drawer / Modal

화면 80% 이상 덮을 때 전체 화면 전환 고려. 선택기 모달도 동일 정책.

이 문서 자체도 동일 정책을 따른다: 표는 컨테이너 overflow:auto로 처리하고 980px 이하에서 그리드를 1열로 전환해 과도한 가로 스크롤을 만들지 않는다.
