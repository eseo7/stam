00 원칙 & 정책 — Principles v1.0
신규 보드 화면 Form 작성 규칙 1. 신규 보드 화면은 반드시 stam.form-controls.css를 로드한다.
2. input / select / date / textarea는 화면별 클래스 대신 공통 클래스(stam-input, stam-select, stam-date, stam-textarea)를 우선 사용한다.
3. 화면별 CSS는 배치(layout) / 해당 화면 전용 chip / table / drawer만 담당한다.
4. 기존 화면(rq-*, msl-*, screen-spec, wbs)은 안정화 후 점진적으로 전환한다.
5. 기본 브라우저 select/date가 다크모드에서 깨지면 공통 클래스 또는 후속 stam-cselect로 처리한다.
CSS 로드 순서 (신규 화면 기준)
1. stam.tokens.css 2. stam.shell.css 3. stam.components.css 4. stam.form-controls.css ← 신규 화면에서 공통 form 스타일 5. stam.[page].css ← 화면별 레이아웃/전용 스타일 (form 재정의 금지)

03 Input
기본 텍스트 Input
height: 36px, width: 100%, border-radius: 8px, focus 시 brand ring
<div class="stam-form-field"> <label class="stam-label">제목 <span class="req">*</span></label> <input class="stam-input" type="text" placeholder="제목을 입력하세요" /> </div>
Search Input
type="search". 기본 X 버튼은 STAM 스타일 SVG로 대체됩니다.
<input class="stam-input" type="search" placeholder="키워드를 입력하세요" />
소형(--sm) Input
height: 28px. 필터 영역, toolbar inline 입력 등에 사용.
<input class="stam-input stam-input--sm" type="text" />
04 Textarea
min-height: 80px, resize: vertical. 다크/라이트 동일 tone.

최대 500자까지 입력할 수 있습니다.

<div class="stam-form-field"> <label class="stam-label">설명</label> <textarea class="stam-textarea" placeholder="설명을 입력하세요" rows="3"></textarea> <p class="stam-field-help">최대 500자까지 입력할 수 있습니다.</p> </div>
05 Select
appearance: none + SVG caret (배경 이미지, background-repeat: no-repeat). 다크모드에서 기본 화살표/반복 패턴이 표시되지 않습니다.
<select class="stam-select"> <option value="">선택하세요</option> <option>높음</option> </select>
소형(--sm) Select
Custom Select 후속 작업 안내 기존 화면별 커스텀 select(rq-cs-*, msl-cs-*)는 이번 1차 작업에서 건드리지 않습니다.
후속 공통화 후보: stam-cselect / stam-cselect-trigger / stam-cselect-panel / stam-cselect-option
06 Date / Datetime-local
input과 동일 height/padding/tone. 다크모드 calendar icon에 filter: invert(.72) brightness(1.1) 적용.
<input class="stam-date" type="date" /> <input class="stam-date" type="datetime-local" />
07 상태 클래스 — States
Disabled
disabled 속성 + .is-disabled 클래스 함께 사용을 권장합니다.
<input class="stam-input" disabled /> <select class="stam-select" disabled>...</select> <input class="stam-date" type="date" disabled /> <textarea class="stam-textarea" disabled>...</textarea>
Readonly
readonly 속성 또는 .is-readonly 클래스. 배경이 bg-sur2로 변경됩니다.
<input class="stam-input" readonly /> <textarea class="stam-textarea" readonly>...</textarea>

.is-error를 control 또는 .stam-form-field에 적용합니다. .stam-field-error로 메시지를 표시합니다.

제목을 입력해 주세요.

우선순위를 선택해 주세요.

내용을 입력해 주세요.

<div class="stam-form-field is-error"> <label class="stam-label">제목 <span class="req">*</span></label> <input class="stam-input is-error" type="text" /> <p class="stam-field-error">제목을 입력해 주세요.</p> </div>

08 Form Grid — 2컬럼 레이아웃
기본 2컬럼 그리드. 640px 이하에서 1컬럼으로 전환됩니다. .full-span으로 전체 폭을 점유합니다.

최대 500자까지 입력 가능합니다.

<div class="stam-form-grid"> <div class="stam-form-field">...</div> <div class="stam-form-field">...</div> <!-- 전체 폭 --> <div class="stam-form-field full-span">...</div> </div>

10 스크립트 로드 순서
신규 보드 페이지 스크립트 로드 순서 기준
theme → nav-data → shell → nav-render → topbar-render → project-context-render → page.js
각 스크립트 역할:
stam.theme.js — 다크/라이트 테마 초기화 (최우선 로드)
stam.nav-data.js — 메뉴 구조 데이터
stam.shell.js — 셸 레이아웃 초기화
stam.nav-render.js — 좌측 네비게이션 렌더링
stam.topbar-render.js — 상단바 렌더링
stam.project-context-render.js — 프로젝트 컨텍스트 렌더링
stam.[page].js — 페이지별 인터랙션 (최후 로드)
<!-- ① 테마 최우선 --> <script src="../js/stam.theme.js"></script> <script src="../js/stam.nav-data.js"></script> <script src="../js/stam.shell.js"></script> <script src="../js/stam.nav-render.js"></script> <script src="../js/stam.topbar-render.js"></script> <script src="../js/stam.project-context-render.js"></script> <!-- ② 페이지별 마지막 --> <script src="../js/stam.[page].js"></script>
