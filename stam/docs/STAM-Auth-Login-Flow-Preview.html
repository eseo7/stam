Auth Login Flow Preview — Draft Preview Pending PR

이 문서는 STAM Auth 로그인 흐름의 Preview 기준을 정의합니다. 실제 Firebase Auth 구현이 아니라 Login / Project Select / Access Pending / Access Denied 4개 화면 상태의 UI Contract와 공통 클래스 기준을 확정하는 것이 목적입니다.

Firebase SDK, initializeApp, GoogleAuthProvider, signInWithPopup 등 실제 인증 구현은 이 문서 범위 밖입니다. Auth 실제 구현 PR은 별도로 진행됩니다.

01 목적 Preview 기준 문서
  • STAM Auth 로그인 흐름을 4개 화면 상태로 정의하고 각 화면의 UI Contract를 확정한다.
  • Google Only 인증 정책을 유지하면서 프로젝트 접근 판정 흐름을 명시한다.
  • Auth 화면에서 사용할 최소 공통 class contract를 확정한다.
  • Firebase Auth 실제 구현 PR과 Preview PR을 명확히 분리한다.
  • stam/css/stam.components.css의 기존 .status-badge 구조를 확장하여 .s-active / .s-pending / .s-denied 세 배지를 추가한다.
02 범위와 제한 FAIL 조건

이번 Preview에서 포함된 것

  • Login / Project Select / Access Pending / Access Denied 4개 화면 상태 정의
  • 각 화면의 UI Contract (레이아웃, 텍스트 밀도, 배지, 버튼 위계)
  • stam/css/stam.components.css 확장 — .s-active, .s-pending, .s-denied
  • Docs Index 카드 등록

이번 Preview에서 포함하지 않은 것

  • Firebase SDK 추가 금지 — initializeApp, GoogleAuthProvider, signInWithPopup, getAuth
  • Firebase config, apiKey, authDomain 추가 금지
  • 실제 로그인 / 로그아웃 동작 구현 금지
  • Firestore read / write 구현 금지
  • 실제 /login, /projects 제품 route 파일 생성 금지
  • Auth bootstrap 구현 금지
  • stam/pages/** 신규 파일 금지 (이번 PR)
03 Auth 화면 상태 흐름 Flow

4단계 흐름

단계 화면 상태 다음 전환
1 Login 미로그인 Google 로그인 → 단계 2 또는 단계 3
2 Project Select 접근 활성 프로젝트 선택 → 제품 내부 (Project Overview)
3 Access Pending 승인 대기 관리자 승인 → 단계 2 / 거부 → 단계 4
4 Access Denied 접근 거부 문의 안내 또는 로그아웃
핵심 정책 Google Only는 유지한다. 이메일/비밀번호 입력폼은 넣지 않는다. STAM은 초대받은 Google 계정만 사용 가능하며, 로그인 화면에서 이 정책을 명시해야 한다. 단계 1 → 단계 2 이동은 active project 존재 여부로 판단한다. active project가 없으면 단계 3(Pending) 또는 단계 4(Denied)로 분기한다.
04 화면 1 — Login UI Contract

목적

STAM 첫 진입점. Google 계정으로 로그인하는 화면이다. B2C 서비스 로그인 화면이 아니라 초대 기반 접근 안내가 포함된 브랜드 진입 화면이어야 한다.

UI Contract

요소기준
레이아웃 중앙 정렬 카드 또는 좌우 분할 (브랜드 영역 + 로그인 영역)
로그인 버튼 .stam-btn--primary — "Google로 로그인" 단일 버튼 (화면당 Primary 1개 원칙)
안내 문구 초대받은 Google 계정만 로그인 가능 / 접근 승인 흐름 안내 포함
금지 이메일 입력폼, 비밀번호 입력폼, 회원가입 링크, 소셜 로그인 복수 제공
폰트 에스코어드림(S-CoreDream) — var(--font)
색상 토큰만 사용 — var(--brand), var(--t1), var(--bg-sur)

필수 포함 내용

  • STAM 로고 + 서비스 이름
  • "초대받은 Google 계정으로만 접속 가능" 안내
  • 접근 승인 절차 간략 설명 (로그인 → 관리자 승인 → 프로젝트 접속)
  • 로그인 후 상태 분기 안내 (Pending 가능성)
주의 이 화면은 Google 버튼 하나만 있는 빈약한 액션 박스가 아니라, 브랜드 진입감과 초대 계정 안내, 접근 권한 흐름이 포함된 정식 로그인 화면이어야 한다.
05 화면 2 — Project Select 접근 활성

목적

로그인 후 active project가 하나 이상 존재할 때 진입하는 화면이다. 프로젝트가 하나뿐이어도 선택 구조는 유지한다 (다중 프로젝트 확장 대비).

UI Contract

요소기준
레이아웃 카드 목록 또는 테이블 — active 프로젝트만 표시
상태 표시 .status-badge.s-active — "접근 활성" 배지
진입 버튼 .stam-btn--primary — "프로젝트 열기" (카드당 1개)
Pending 프로젝트 .status-badge.s-pending으로 표시, 클릭 비활성
빈 상태 프로젝트가 없으면 Pending 화면(단계 3)으로 분기

정책 기준

  • active 프로젝트만 표시한다 (pending / inactive는 별도 섹션 또는 숨김)
  • 프로젝트 선택 후 Project Overview(A1)로 이동한다
  • Project Overview는 선택 이후의 landing이지 로그인 대체 화면이 아니다
06 화면 3 — Access Pending 승인 대기

목적

로그인 후 active project가 없거나 접근 승인이 아직 처리되지 않은 사용자에게 보여주는 화면이다. 사용자가 현재 상태를 명확히 이해하고 다음 행동을 알 수 있도록 안내한다.

UI Contract

요소기준
레이아웃 중앙 정렬 상태 카드 — 아이콘 + 제목 + 설명 + 안내 액션
상태 배지 .status-badge.s-pending — "승인 대기 중"
안내 문구 관리자 승인 진행 중 안내 / 승인 완료 후 자동 접속 가능 안내
액션 버튼 .stam-btn--secondary — "로그아웃" 또는 "새로고침"
문의 링크 .stam-btn--text — "관리자에게 문의"
설계 의도 사용자가 현재 "대기" 상태임을 명확히 인지하도록 한다. 시스템이 처리 중임을 전달하고 불필요한 재시도를 방지한다. Primary 버튼은 이 화면에 두지 않는다 (사용자가 직접 취할 수 있는 Primary 액션 없음).
07 화면 4 — Access Denied 접근 거부

목적

접근이 명시적으로 거부된 사용자 또는 초대받지 않은 Google 계정으로 로그인한 사용자에게 보여주는 화면이다. Pending과 달리 "처리 불가"를 명확히 전달한다.

UI Contract

요소기준
레이아웃 중앙 정렬 상태 카드 — 아이콘 + 제목 + 설명 + 안내 액션
상태 배지 .status-badge.s-denied — "접근 거부"
안내 문구 접근 불가 사유 안내 / 올바른 Google 계정 사용 안내
Primary 버튼 .stam-btn--primary — "로그아웃" (이 화면의 유일한 Primary 액션)
문의 링크 .stam-btn--text — "관리자에게 문의"
Danger 버튼 사용 금지 로그아웃은 Danger 버튼이 아니라 Primary 버튼으로 처리한다. Danger 버튼은 Dialog 확인과 함께 사용하는 파괴적 액션(삭제 등)에만 사용한다. — Button Guide 위계 기준.
08 CSS Class Contract 확정

이번 작업에서 확정된 공통 class

클래스 토큰 (배경) 토큰 (텍스트) 용도 예시
.status-badge.s-active var(--pass-bg) var(--pass) 접근 활성 — active member / 로그인 성공 + project access OK 접근 활성
.status-badge.s-pending var(--warn-bg) var(--warn) 승인 대기 — 로그인 후 project 접근 미승인 상태 승인 대기
.status-badge.s-denied var(--fail-bg) var(--fail) 접근 거부 — 초대 미포함 계정 또는 명시적 거부 접근 거부

기존 구조와의 관계

  • .status-badge — 기존 stam/css/stam.components.css의 베이스 클래스 (변경 없음)
  • .s-todo / .s-wip / .s-done — 기존 배지 (변경 없음)
  • .s-active / .s-pending / .s-denied — 이번 작업에서 추가 (3개 신규)
  • 다크 모드 — stam.tokens.css의 [data-theme="dark"]에서 토큰이 자동 flip됨. stam.components.css에 dark override도 추가.

참조 파일

  • stam/css/stam.components.css — .status-badge 및 신규 3개 추가
  • stam/css/stam.tokens.css — --pass-bg, --pass, --warn-bg, --warn, --fail-bg, --fail
