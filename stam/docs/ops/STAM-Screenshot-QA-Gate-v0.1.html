01 Screenshot QA Gate 목적 v0.1
문서명STAM Screenshot QA Gate v0.1
대상UI PR merge 전 시각 검증
캡쳐 스크립트scripts/qa/capture-ui-pr-screenshots.mjs
  • UI PR은 GitHub source review만으로 merge하지 않는다.
  • 실제 렌더링 화면을 Playwright screenshot으로 남기고, 사람 눈으로 PASS/FAIL 확인한다.
  • PR #250처럼 “공통 class는 썼지만 화면이 어색한” 상황을 막기 위한 Gate다.
범위
본 Gate는 기준·스크립트 초안만 정의한다. 상세 Drawer 공통 CSS/JS 생성, 제품 UI 코드 전환, staging 배포 판단은 이후 PR에서 별도로 진행한다.
02 적용 대상

Screenshot QA 필수

  • UI layout 변경
  • Drawer / modal / popover / dropdown 변경
  • Button / footer / header / toolbar 변경
  • Custom select / form control 변경
  • dark mode / responsive 변경
  • 공통 component 변경
  • WBS / 요구사항 / 메뉴화면 / 화면설계서 / 기능정의 등 주요 board 화면 변경

Screenshot QA 생략 가능

  • docs-only PR
  • ops log / schema-only PR
  • comment / text-only 문서 수정
  • 제품 UI에 영향 없는 CI / infra 문서 PR
03 필수 캡쳐 세트

UI PR마다 최소 5장

# 캡쳐 설명
1 기준 화면 light desktop 변경 전 또는 SSOT 기준 화면 (예: 요구사항정의서 detail drawer)
2 변경 화면 light desktop 이번 PR이 수정한 화면
3 기준/변경 비교용 light desktop 나란히 비교 가능한 캡쳐 또는 동일 viewport·동일 drawer 상태
4 변경 화면 dark desktop data-theme="dark" 또는 토글 후 캡쳐
5 변경 화면 mobile 430px overflow·버튼 겹침 확인

권장 viewport

  • desktop: 1440 × 900 또는 1366 × 900
  • mobile: 430 × 932
04 파일명 규칙

형식: pr<번호>-<대상>-<theme>-<viewport>.png

  • pr254-requirements-detail-light-desktop.png
  • pr254-menu-detail-light-desktop.png
  • pr254-compare-light-desktop.png
  • pr254-menu-detail-dark-desktop.png
  • pr254-menu-detail-mobile-430.png
생성된 screenshot 파일은 PR 코드에 커밋하지 않는 것을 기본으로 한다. PR comment 또는 외부 artifact로 제출한다.
05 PR body / PR comment 체크리스트

UI PR merge 요청 시 반드시 포함

  • screenshot 파일 경로 또는 첨부 (최소 5장)
  • 캡쳐 대상 URL
  • viewport (desktop / mobile 430)
  • theme (light / dark)
  • console error count (0이어야 PASS)
  • drawer / modal open 확인
  • mobile overflow 확인 (가로 스크롤·버튼 겹침 없음)
  • human visual reviewer (이름 또는 역할)
  • final visual PASS / FAIL
06 PASS / FAIL 기준
PASS 조건
  • 필수 캡쳐 5장 모두 제출
  • console red error 0건
  • drawer/header/footer가 기준 패턴과 시각적으로 일치
  • dark mode 배경·텍스트 정상
  • mobile 430px에서 버튼 겹침·가로 스크롤 없음
  • human visual reviewer가 PASS 판정
FAIL 조건
  • 스크린샷 누락
  • console red error 발생
  • drawer / header / footer가 기준 패턴과 명확히 다름
  • dark mode에서 흰 배경 깨짐
  • mobile 430px에서 버튼 겹침 / 가로 스크롤
  • 스크린샷은 있지만 기준 화면과 비교 불가
  • CSS 값만 보고하고 이미지 증빙이 없음
07 안전 기준
  • 캡쳐는 read-only 시나리오 우선 — 목록 조회, drawer 열기, 탭 전환 등
  • save / delete / submit 클릭 금지
  • IndexedDB local test data는 허용하되 Firestore / API write 금지
  • staging / preview URL 사용 시 운영 데이터 변경 금지
  • generated screenshot files는 PR에 코드 파일로 커밋하지 않는 것을 기본으로 함
  • PR comment 또는 외부 artifact로 제출
08 PR #250 ~ #252 사례 연결

Screenshot QA Gate 도입 근거

PR #250 MERGED_NEEDS_CORRECTION
공통 shell/class 재배선 source / class 기준은 통과했지만 상세 시각 패턴 QA가 부족했다. 공통 class를 썼어도 화면이 어색한 상태로 merge되어 PR #252 보정이 필요했다.
PR #252 MERGED_PASS
요구사항정의서 패턴 보정 Playwright QA와 시각 기준을 통해 보정 완료. 이 사례를 근거로 Screenshot QA Gate를 도입한다.
상세 사례는 PR Audit Log Guide v0.1 및 stam/ops/pr-audit-log.jsonl 참조.
