01 목적 v0.1

STAM 주요 board 화면의 상세 Drawer 본문을 동일한 구조·스타일로 전환하기 위한 공통 CSS/JS 기반을 제공한다.

  • Drawer shell은 기존 stam-drawer* (stam.drawer.css)를 유지한다.
  • Drawer body는 stam-detail* + STAM.detailDrawer로 렌더한다.
  • board별 legacy prefix (rq-*, msv2-* 등)를 점진 제거한다.
공통화 완료가 아님 — 본 문서와 자산은 “기반 생성” 단계이다. 실제 화면 전환은 후속 PR에서 진행한다.

05 CSS class contract

로드 순서

stam.tokens.css → stam.drawer.css → stam.detail-drawer.css → (페이지 CSS)

본문 class (board prefix 금지)

  • .stam-detail — root
  • .stam-detail-section, .stam-detail-section-head, .stam-detail-section-title, .stam-detail-section-meta
  • .stam-detail-grid, .stam-detail-field, .stam-detail-label, .stam-detail-value, .stam-detail-empty
  • .stam-detail-note, .stam-detail-list, .stam-detail-chip-row, .stam-detail-chip
  • .stam-detail-actions, .stam-detail-divider

반응형

  • 640px 이하: .stam-detail-grid 1열
  • 가로 스크롤 유발 금지 (min-width:0, overflow-wrap)
06 JS renderer API
함수설명
STAM.detailDrawer.escape(value)HTML escape. null/undefined → 빈 문자열
renderField(field){ label, value, emptyText, type, className, full } → field HTML
renderSection(section){ title, meta, fields, body, emptyText } → section HTML
render(config){ title, sections, actions, dividerBeforeActions } → root HTML
mount(target, config)selector 또는 HTMLElement. 없으면 false. 성공 true

field.type

  • text (default), plain, note, list, chips
  • chips item: { label, tone?, className? } — tone: brand / pass / warn / fail
section.body는 trusted HTML 용도(마이그레이션 단계). 일반 텍스트는 fields + escape 경로 사용.

09 Light / Dark / Mobile QA 기준
  • Light: grid border·label hierarchy·note accent bar 정상
  • Dark: data-theme="dark" surface/border token flip, chip semantic 색 유지
  • Mobile 430px: grid 1열, chip wrap, 가로 스크롤 없음
  • Shell 충돌 없음: .stam-drawer-body 내부에 .stam-detail만 삽입
