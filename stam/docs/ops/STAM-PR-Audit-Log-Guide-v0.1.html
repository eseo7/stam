01 PR Audit Log 목적 v0.1
문서명STAM PR Audit Log Guide v0.1
데이터 파일stam/ops/pr-audit-log.jsonl
스키마stam/ops/pr-audit-log.schema.json
  • PR별 변경 범위, 검증 기준, 실패 사유, 보정 PR, 최종 승인 여부를 repo에 기록한다.
  • ChatGPT / Cursor / Claude 세션 기억에 의존하지 않는다.
  • 추후 “왜 이 PR을 승인했는지”, “무엇을 놓쳤는지” 추적 가능하게 한다.
기록 형식
JSON Lines (JSONL) — 한 줄 = 한 PR 기록. 각 줄은 독립적으로 JSON.parse 가능해야 한다.
02 기록 대상
type 값 (예시) 대상 비고
UI UI PR 제품 화면·레이아웃·스타일 변경
UI_COMMON_COMPONENT 공통 컴포넌트 PR 공통 CSS/JS renderer 도입·확장
CRUD_DB_API CRUD/DB/API PR 데이터 계층·API 경계 변경
DOCS_INFRA Docs/Infra PR 문서·CI·배포·운영 기준
FAILURE_CORRECTION 실패/폐기/보정 PR closed·보정·회귀 수정 PR 포함
03 필수 필드 설명
prNumber
GitHub PR 번호 (number)
title
PR 제목 (string)
type
PR 유형 — UI, UI_COMMON_COMPONENT, CRUD_DB_API, DOCS_INFRA 등
baseSha
PR base commit SHA (merge 전 main 기준)
headSha
PR head commit SHA (PR 브랜치 최종 커밋)
changedFiles
변경 파일 경로 배열 (array of string)
allowedFiles
PR 작업 시 허용된 파일 목록 (있는 경우)
forbiddenTouched
금지 파일이 diff에 포함됐는지 (boolean)
sourceReview
소스 리뷰 결과 객체 — { pass, notes, reviewer } 등
screenshotQa
Playwright screenshot QA 결과 객체 — { pass, reportPath, notes } 등
visualPass
시각 검증 통과 여부 (boolean 또는 null — 미실시 시 null)
failureReason
실패·보정 필요 사유 (string 또는 null)
correctionPr
후속 보정 PR 번호 (number 또는 null)
mergeSha
main에 merge된 커밋 SHA (string 또는 null)
stagingUrl
staging 검증 URL (string 또는 null)
finalStatus
최종 상태 — MERGED_PASS, MERGED_NEEDS_CORRECTION, CLOSED 등
notes
운영자·리뷰어 메모 (string)

05 운영 원칙
  • UI PR은 source review만으로 merge 금지 — 코드 리뷰만으로는 시각 회귀를 잡지 못한다.
  • screenshot QA 필요 — Playwright screenshot으로 패턴·레이아웃 일치를 검증한다.
  • 공통화 PR은 같은 코드를 실제 2개 이상 화면이 쓰는지 확인 — 한 화면만 쓰면 공통화 PR이 아니다.
  • 전용 class를 새로 만들면 공통화가 아니다 — rq-*, msv2-* 등 화면 전용 prefix 생성은 FAIL.
  • 최종 승인 전 merge 금지 — finalStatus가 MERGED_PASS 또는 명시적 예외 승인 전까지 merge하지 않는다.
  • 보정 PR 연결 — MERGED_NEEDS_CORRECTION 또는 CLOSED인 경우 correctionPr 필드로 후속 PR을 연결한다.
  • 모든 PR을 jsonl에 기록 — merge·close·폐기 모두 기록 대상이다.
