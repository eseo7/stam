STAM WBS Top / Left / Shell Applied Preview v1 Preview WBS Board Product Not Yet

본 문서는 WBS 작업 화면에 STAM 공통 Topbar Anatomy · Left Navigation · App Shell 가이드를 적용한 후 어떤 화면이 나와야 하는지 시각 기준을 고정한다. 현재 WBS 화면은 topbar가 48px보다 얇아 보이고, LEFT / CENTER / RIGHT 영역 분리가 약하며, CENTER 검색 영역이 비어 있고, RIGHT 액션이 과밀해 있다. Cursor가 실제 제품을 보정하기 전에 본 시안으로 기준 화면을 먼저 합의한다.

적용 범위 — 본 문서는 stam/docs/** 하위 1개 HTML만 생성하며, stam/pages/**, stam/js/**, stam/css/**, stam/index.html, stam/assets/**는 단 한 줄도 수정하지 않는다. Docs Index에는 카드 1개를 추가한다. 적용은 Cursor가 별도 commit에서 수행한다.

01 Overview — 문서 목적 목적

현재 WBS 화면의 topbar / left / shell 구조가 STAM 가이드 기준과 다르게 보인다. 본 문서는 Cursor가 실제 코드를 수정하기 전에 “가이드가 적용된 WBS 화면이 어떻게 보여야 하는가”를 정적 시각 기준으로 고정하기 위한 Preview이다. 본 문서는 단일 페이지 안에서 격리 스타일(.ws-* prefix)로 동작하며, 실제 App Shell · 공통 CSS · 페이지에 영향을 주지 않는다.

이 문서가 다루는 것
  • WBS 화면 기준 Topbar 48px · 3영역(LEFT/CENTER/RIGHT) 분리
  • CENTER 현재 화면 검색 input 신규 배치
  • RIGHT 액션 정리안 — 알림 / 사용자 / 테마 + Quick Action 1~2
  • WBS 본문 상단 Page Toolbar 분리안
  • Left Navigation 고정 / 스크롤 / 현재 메뉴 active 표시
  • WBS 본문 가로 스크롤 금지 · 테이블 내부 overflow만 허용
  • topbar / page toolbar / drawer footer 버튼 사이즈 기준
  • Cursor가 실제 적용할 후보 파일 명시
이 문서가 다루지 않는 것
  • stam/pages/**, stam/js/**, stam/css/** 직접 수정
  • stam/index.html, stam/assets/** 변경
  • 가이드 문서(Topbar Guide · UI Baseline Guide) 자체 수정
  • 전역 .tbtn 일괄 치환 / 다른 화면 일괄 적용
  • Drawer open/close · 전체 보기 · 등록·수정·상세 등 기능 로직 변경
  • Firebase · GitHub Actions · package.json 변경

  • STAM 로고 (22px) + Wordmark

  • 우측 끝에 ⌘ K 키 힌트
  • focus 시 brand 1.5px ring (rgba(84,81,232,.18))
  • 검색 영역이 없는 화면에서도 center slot은 비워두고 유지
RIGHT 영역 — System Actions
  • 알림 / 테마 / 사용자 메뉴 — 항상 표시

  • 내보내기 / PM / 보기 등 업무 액션은 page toolbar로 이동
  • 아이콘 버튼 사이즈: 28×28 (icon-only sm)
  • 사용자 메뉴는 avatar + chevron, label 생략
공통 규칙
  • 높이 48px 고정 · 배경 #0A0E1A
  • border-bottom rgba(255,255,255,.07)
  • LEFT / CENTER / RIGHT — flex layout · CENTER만 flex:1
  • 모바일 폭에서 CENTER는 숨김 가능, LEFT / RIGHT는 유지
  • topbar는 라이트 테마에서도 다크로 고정
04 Page Toolbar — 업무 액션 분리안 분리안

현재 WBS topbar에 몰려 있는 내보내기 / 필터 / 보기 / 신규 등록은 topbar가 아닌 본문 상단 별도 Page Toolbar로 이동한다. topbar는 “시스템 전역 컨텍스트(LEFT/CENTER/RIGHT)”에 집중하고, page toolbar는 “현재 화면 업무 액션”에 집중한다.

05 Left Navigation — 좌측 영역 기준안 기준

Left Navigation은 topbar 아래에 들어가며, topbar와 겹치지 않는다. 상단 고정 영역(검색)과 스크롤 영역(그룹 + 메뉴), 하단 고정 푸터로 구성된다. WBS는 B 그룹 — 업무 보드에 속하며, 메뉴 항목은 항상 active 표시를 갖는다.

  • 그룹 카운트 badge는 brand soft 컬러

  • active 라벨은 white · weight 600
  • WBS가 어느 그룹인지 한눈에 확인 가능
Topbar 겹침 방지 Left Navigation은 top: var(--topbar-h, 48px) 아래에서 시작한다. topbar가 position: sticky이므로 left 영역은 topbar height만큼 내려간 곳에서 100% 높이를 가진다. 현재 화면에서 left 상단이 topbar에 가려진 형태로 보인다면 이 기준이 깨진 것이다.
06 WBS Content — 본문 / Scroll Policy 기준

WBS 본문은 page toolbar 아래에서 시작한다. 페이지 자체에는 가로 스크롤이 발생하지 않는다. 테이블이 좁아 보일 때는 테이블 컨테이너만 자체 가로 스크롤을 갖는다.

허용되는 Overflow
  • 테이블 wrap 내부 overflow-x: auto
  • Drawer 내부 long-form content overflow-y: auto
  • Left Navigation 메뉴 목록 overflow-y: auto
  • Filter chip row — flex-wrap: wrap (스크롤 X)
금지되는 Overflow
  • page-level html, body 가로 스크롤
  • App Shell content 영역 가로 스크롤
  • page toolbar 가로 스크롤
  • topbar 가로 스크롤 (3영역 분리로 해결)
Drawer footer 기준 유지 commit 9c3f0b1에서 보정된 Drawer footer 전체 보기 버튼은 md 32px 기준을 유지한다. 본 시안의 Drawer footer mock도 동일한 사이즈로 표기되어 있다. 전체 보기 / 닫기 / 저장 등 footer 버튼은 모두 md 32px 기준으로 정렬된다.

07 Button Applied — 영역별 사이즈 / 역할 샘플
Topbar Button — sm
  • icon-only 28×28 (.stam-btn--sm.is-icon)
  • 역할: 알림 / 테마 / 사용자 메뉴 trigger
  • label 없음 — title attribute로 접근성 보장
Page Toolbar Button — sm / md
  • sm(28px) — 필터 · 보기 · 정렬 등 view actions
  • md(32px) — 내보내기 · 신규 등록 · CTA
  • Primary — 신규 등록 / 등록 등 단일 핵심 CTA
  • Secondary — 보조 작업 (내보내기 / 저장 전 단계)
  • Ghost — view / 필터 toggle 등 부수 작업
Drawer Footer Button — md
  • md 32px 기준 — 모든 footer 버튼 정렬 통일
  • Primary — 저장 · 확인
  • Secondary — 전체 보기 (Drawer ↔ Full)
  • Ghost — 닫기 · 취소
  • commit 9c3f0b1에서 이미 보정 완료
변환 금지 / 부분 적용
  • 전역 .tbtn → .stam-btn 일괄 치환 금지
  • Docs 페이지의 .tbtn은 그대로 둔다
  • WBS 화면에 한정해서 새 사용처에만 .stam-btn 적용
  • 기존 동작(open/close, 등록/수정)은 절대 변경 금지
  • 버튼 사이즈는 영역별 기준을 정확히 따른다
