Section 00 · Audit 개요 & 판정 범례

본 문서는 Board List Template Contract v1 의 공통 목록 구조와 공통 JS 컨트롤러 stam.board-list.js (window.STAMBoardList) 가 각 게시판 화면에 어디까지 적용되었는지, 화면별 자체 row/checkbox/delete 로직이 남아있는지, 다음 조치는 무엇인지를 조사한 결과다.

이번 PR(#105)은 Audit 문서 생성만 한다. 제품 화면 코드 · JS · CSS 는 수정하지 않으며, Controller 적용이나 legacy class 정리는 후속 PR 에서 진행한다.

완료 Controller 적용 필요 legacy class 정리 필요 특수 구조 · 별도 검토 보류

기준 컨트롤러 hook: data-stam-board-list (root) · .stam-select-table · .stam-table-row · .stam-check · [data-stam-check-all] · .stam-delete-btn / .stam-delete-label · STAMBoardList.init().

Section 02 · 화면별 상세 소견

기능정의서 완료

pages/boards/functional-specification.html · js/stam.functional-specification.js
  • 공통 8종 hook 전부 적용. STAMBoardList.init(listRoot, { deleteBtn:'#fn-del-btn', onRowActivate→openDrawer('detail') }) (JS 약 223行).
  • 자체 bindRows() / bindCheckboxes() / delete-count 로직은 PR #93 에서 제거 — 컨트롤러 위임.

Open Scenario Controller 적용 필요 근접

pages/boards/open-scenario.html (table:344) · js/stam.open-scenario.js (254行)
  • 마크업은 거의 완성형: <table class="os-tbl stam-select-table">, 행 .stam-table-row(24), checkbox .os-row-cb + .stam-check, 헤더 .os-cb-all + .stam-check, 삭제 버튼 .stam-delete-btn + .stam-delete-label.
  • 미적용: stam.board-list.js 미로드, root data-stam-board-list 없음, 헤더 data-stam-check-all 없음, STAMBoardList.init() 미호출.
  • 잔존 자체 로직: .os-cb-all 전체선택 핸들러(JS 약 165~)와 delete-count 갱신(JS 182~183, 이미 .stam-delete-label 사용)이 화면 자체 구현.
  • 행이 정적 마크업이라 회귀 위험 낮음 → 파일럿 2호로 가장 적합. 필요한 작업: board-list.js 로드 + root attr + check-all attr + init() + 자체 로직 제거.

화면설계서 Controller 적용 필요

Section 04 · 권장 다음 PR 순서

적용 근접도(낮은 회귀 위험) → 쌍둥이 패턴 재사용 효율 → 대형/특수 화면 순으로 권장한다.

  1. Open Scenario — 마크업 거의 완성, 정적 행, 최소 변경. 파일럿 2호로 컨트롤러 적용 패턴을 2번째로 검증.
  2. 메뉴/화면목록 — 쌍둥이 패턴 A 확립(공통 class 보강 + delete-btn/label 전환 + init + 자체 로직 제거 + .msl-* 정리).
  3. 요구사항 — 쌍둥이 패턴 B, A 의 매핑 그대로 재사용. (2·3 은 연속 처리 권장)
  4. 화면설계서 — 동적 렌더링·대형 파일, 중간 난이도. 렌더 경로 공통 class 일관화 후 init.
  5. WBS — 직접 적용 전 설계 검토 PR/문서 선행(그룹 행·presentation/모바일·portal). 검토 후 적용 여부/방식 결정.

* 각 적용 PR 은 Bare Preview 육안 비교 + Playwright smoke(기능정의서 PR #104 기준 항목) 를 동일하게 수행한다. 계약 기준: Board List Contract v1.
* 기능정의서 legacy class 정리는 별도 저우선 PR 로 분리 가능(현행 동작에 영향 없음).
