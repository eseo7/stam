Section 00 · Audit 개요 & 판정 범례

본 문서는 Board List Template Contract v1 의 공통 목록 구조와 공통 JS 컨트롤러 stam.board-list.js (window.STAMBoardList) 가 각 게시판 화면에 어디까지 적용되었는지, 화면별 자체 row/checkbox/delete 로직이 남아있는지, 다음 조치는 무엇인지를 조사한 결과다.

본 문서는 PR #105 에서 생성된 후, 후속 적용 PR(#106 Open Scenario · #107 메뉴/화면목록 · #108 요구사항)과 #109 화면설계서 조사 결과를 반영해 현황을 갱신했다. 본 보정(PR #109)은 본 Audit 문서만 수정하며 제품 화면 코드 · JS · CSS 는 변경하지 않는다.

화면설계서 특수 구조 · 현행 유지 권장 자체 엔진으로 계약 충족

pages/boards/screen-specification.html (table:187) · js/stam.screen-specification.js (2,490行) · PR #109 조사
  • PR #109 조사 결과 Controller 직접 적용을 보류(현행 유지)한다. 마크업 일부는 공통화되어 있으나(<table class="ss-table stam-select-table">, 헤더 #ss-cb-all .stam-check .stam-check-cell, 삭제 .stam-delete-btn/.stam-delete-label), 자체 엔진 구조가 컨트롤러 모델과 근본적으로 다르다.
  • row 는 JS renderTable() 에서 동적 렌더링 — 빈 <tbody id="ss-tbody"> 에 매 상호작용마다 tbody.innerHTML 통째 교체(검색·필터·그룹 펼침·그룹선택·drawer open/close).
  • 선택 상태의 source of truth 가 DOM 이 아니라 S.sel Set — 행 .is-selected / 헤더 / 그룹 체크박스 / delete count 가 모두 S.sel 에서 파생·재렌더된다. .is-active 도 S.openId 기반 재렌더.
  • 그룹 헤더행(.ss-gr-row) + 데이터행(.ss-sc-row.stam-table-row) 2종 구조. 그룹 헤더는 그룹 전체선택 checkbox + indeterminate 보유(data-ss-grp-sel). 그룹 checkbox 도 .stam-check 라 컨트롤러 change 핸들러와 충돌.
  • 현행이 이미 Board List 계약 동작을 충족 — 미수정 상태 Playwright(Chromium) 20/20 PASS: row click = .is-active + drawer open(checkbox unchecked·count 미증가), checkbox click = .is-selected + checked + 삭제 (N)(drawer 미오픈), selected row click = .is-selected.is-active 동시, Escape = .is-active 해제 / .is-selected 유지, header 전체선택·전체해제·indeterminate.
  • 적용 보류 사유: Controller 강제 적용 시 ① S.sel vs 컨트롤러 DOM 상태의 이중 진실원, ② 전체 tbody 재렌더링이 컨트롤러가 쓴 class 를 소실, ③ 그룹 checkbox / indeterminate 충돌 위험. 최소 수정 불가 · 대규모 리라이트(2.5K行) = 회귀 위험 큼, 동작 이득 없음.
