STAM WBS Board List Structure Review v1

WBS 화면에 공통 목록 컨트롤러 window.STAMBoardList 를 적용할 수 있는지 구조를 조사하고 판정한 검토 문서. 조사·문서화 전용 — 제품 코드 미수정.

Section 1 · Summary

최종 판정: STAMBoardList 직접 적용 보류 WBS 특수 구조 · 현행 유지 자체 WBS 계약으로 관리

WBS 는 그룹 다중 <tbody> + 런타임 주입 행 checkbox + 22컬럼 가로스크롤/포커스/Gantt/전체보기 모드가 얽힌 특수 구조라, 평면 행 모델인 STAMBoardList 직접 적용은 보류한다. 다만 PR #111 에서 row=active / checkbox=selected 분리 원칙은 정상화 완료되어, WBS 도 Board List 상호작용 계약을 (자체 구현으로) 충족한다. WBS 는 당분간 WBS 전용 Board List 계약 기준으로 유지하고, STAMBoardList 확장 설계는 장기 과제(별도 PR)로 둔다.

  • STAMBoardList 직접 적용: ❌ 보류 — 그룹 다중 tbody / 주입형 checkbox / 가로스크롤·포커스·Gantt·전체보기 얽힘.
  • row active / checkbox selected 분리: ✅ PR #111 에서 정상화 완료 (WBS 자체 로직, board-list.js 미수정).
  • 관리 기준: WBS Board List Contract v1 — 자체 계약 문서로 유지.
  • 장기 과제: STAMBoardList 그룹/주입형 어댑터 확장 설계는 별도 PR 에서만 검토(stam.board-list.js 변경 동반).

DOM 구조

  • stam.board-list.js 미로드. <table class="wbs-table"> — .stam-select-table · data-stam-board-list 없음.
  • 22 컬럼 와이드 테이블 → .wbs-tbl-inner#wbs-tbl-inner 기반 가로 스크롤. 컬럼은 .is-shared-col / .is-focus-col / .col-d 로 구분(포커스 모드에서 표시 토글).
  • 그룹마다 <tbody> 2개: .wbs-grp-hdr-body(그룹 헤더행 — colspan="22", 토글 버튼·진행바·통계) + .wbs-grp-rows[data-grp](데이터행). 현재 6개 그룹 / 17개 데이터행.
  • 데이터행: <tr class="wbs-data-row stam-table-row" data-wbs-id="WBS-00X"> — 정적 HTML. .stam-table-row 는 이미 부여됨.
  • 행 checkbox 는 정적 HTML 에 없고 JS 가 런타임 주입(initCheckboxes): 각 행 맨 앞에 <td class="wbs-td-chk"><input class="wbs-chk wbs-row-chk"> 삽입. 헤더에는 전체선택 #wbs-chk-all 존재.
  • 삭제 버튼 #wbs-delete-btn — 라벨이 별도 .stam-delete-label span 이 아니라 버튼 자체 textContent, 형식은 삭제 N(괄호 없음).
  • 부가 구조: Gantt 섹션(접기/펼치기 maxHeight 애니메이션) · 포커스 모드(data-focus-toggle/data-focus-exit) · 그룹 토글 / 전체 펼치기·접기 · 드로어 전체보기(full view) · portal 마운트 custom select(PR #100).

상태 관리 방식

  • 선택 source of truth = DOM (.wbs-row-chk:checked) — 별도 JS state Set 없음. 전체선택/indeterminate/삭제 카운트 모두 DOM 쿼리로 계산(updateMasterChk / updateDeleteBtn).
  • 행 시각 상태는 .selected + .is-selected 병행 토글. .is-active 개념은 사용하지 않는다.
