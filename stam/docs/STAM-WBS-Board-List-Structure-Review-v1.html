Section 1 · Summary

최종 판정: Controller 확장 설계 필요  (직접 적용 보류 · 현행 유지)

WBS 는 그룹 다중 <tbody> + 런타임 주입 행 checkbox + 22컬럼 가로스크롤/포커스/Gantt/전체보기 모드가 얽힌 특수 구조다. 현행은 checkbox 선택·삭제 카운트·전체선택은 동작하지만, row click 이 .is-active 가 아니라 .is-selected 를 부여하여 계약(row=active / checkbox=selected 분리)을 충족하지 못한다. 평면 행 모델인 STAMBoardList 를 그대로 적용하기에는 위험이 크다.

  • 직접 Controller 적용: ❌ 보류 — 그룹 다중 tbody / 주입형 checkbox / 행클릭·드로어·포커스·Gantt 얽힘과 충돌.
  • 권장 방향: ① 단기 — 현행 유지. ② 중기 — WBS 전용 계약 문서화 + STAMBoardList 그룹/주입형 어댑터 확장 설계(별도 PR, stam.board-list.js 변경 동반). ③ (선택) 소규모 정상화 PR 로 row click → .is-active 분리 및 삭제 라벨 표준화만 우선 교정 가능.

DOM 구조

  • stam.board-list.js 미로드. <table class="wbs-table"> — .stam-select-table · data-stam-board-list 없음.
  • 22 컬럼 와이드 테이블 → .wbs-tbl-inner#wbs-tbl-inner 기반 가로 스크롤. 컬럼은 .is-shared-col / .is-focus-col / .col-d 로 구분(포커스 모드에서 표시 토글).
  • 그룹마다 <tbody> 2개: .wbs-grp-hdr-body(그룹 헤더행 — colspan="22", 토글 버튼·진행바·통계) + .wbs-grp-rows[data-grp](데이터행). 현재 6개 그룹 / 17개 데이터행.
  • 데이터행: <tr class="wbs-data-row stam-table-row" data-wbs-id="WBS-00X"> — 정적 HTML. .stam-table-row 는 이미 부여됨.
  • 행 checkbox 는 정적 HTML 에 없고 JS 가 런타임 주입(initCheckboxes): 각 행 맨 앞에 <td class="wbs-td-chk"><input class="wbs-chk wbs-row-chk"> 삽입. 헤더에는 전체선택 #wbs-chk-all 존재.
  • 삭제 버튼 #wbs-delete-btn — 라벨이 별도 .stam-delete-label span 이 아니라 버튼 자체 textContent, 형식은 삭제 N(괄호 없음).
  • 부가 구조: Gantt 섹션(접기/펼치기 maxHeight 애니메이션) · 포커스 모드(data-focus-toggle/data-focus-exit) · 그룹 토글 / 전체 펼치기·접기 · 드로어 전체보기(full view) · portal 마운트 custom select(PR #100).

상태 관리 방식

  • 선택 source of truth = DOM (.wbs-row-chk:checked) — 별도 JS state Set 없음. 전체선택/indeterminate/삭제 카운트 모두 DOM 쿼리로 계산(updateMasterChk / updateDeleteBtn).
  • 행 시각 상태는 .selected + .is-selected 병행 토글. .is-active 개념은 사용하지 않는다.

확인 결과 (probe):

  • 화면 로드 status 200, 데이터행 17 / 주입 checkbox 17 / 그룹 헤더 tbody 6 / 그룹 행 tbody 6.
  • .stam-select-table 0 · [data-stam-board-list] 0 · [data-stam-check-all] 0 · .stam-delete-label 0 (모두 미적용).
  • checkbox click → 행 .is-selected + checked + 삭제 1, drawer 미오픈 ✓. 2개 선택 → 삭제 2 + master indeterminate ✓.
  • row click → .is-active=false / .is-selected=true, drawer open ✓. 동시에 다른 선택행의 .is-selected 시각은 해제되지만 그 행 checkbox 는 checked 유지·삭제 2 카운트 유지 → 시각·논리 선택 desync 확인.
  • Escape → drawer 닫힘, 선택 유지(active 개념 없음).
  • master 전체선택 → 17/17 checked, 삭제 17.
  • 콘솔 오류 0.
