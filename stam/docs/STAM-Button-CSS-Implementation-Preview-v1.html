STAM Button CSS Implementation Preview v1 Implementation Preview

본 문서는 Cursor의 Button CSS 제한 구현 직전 단계에서 “최종 시각 기준”을 정리한 정적 Preview이다. variant · size · state · icon-only · loading · danger · focus 상태를 한 화면에서 비교 가능하게 정리하며, Cursor 구현 후에는 본 문서가 Button CSS QA Baseline으로 사용된다. 본 문서로는 stam/css/**, stam/js/**, stam/pages/**, stam/index.html, stam/assets/**를 절대 수정하지 않는다.

Implementation Preview Button CSS QA Baseline No CSS / JS / Pages Edit

판정 근거 — Button Token Mapping v1 · Token Inventory v1 · P0 Decision Approval Sheet v1 · Implementation Readiness Dashboard v1까지 매핑·승인 시퀀스가 끝났고, Cursor가 staging에서 Implementation Readiness Dashboard v1을 반영 중인 상태이다. Button CSS 구현 직전, “구현 후 어떤 화면이 나와야 하는가”를 정적 화면 하나로 고정해 두면 QA 합의가 빨라진다.

01 Overview — 문서 목적 목적

Button Token Mapping v1이 “어떤 토큰이 어디에 매핑되는가”를 표로 확정했다면, 본 Preview는 “그 매핑이 화면에서 어떻게 보여야 하는가”를 정적 시각 기준으로 고정한다. Cursor가 Button CSS 제한 구현(stam.tokens.css · stam.components.css)을 마친 뒤, 본 문서를 좌·우로 띄워 놓고 staging 화면과 시각 비교하는 QA Baseline으로 사용한다.

이 문서가 다루는 것
  • variant 7종(Primary / Secondary / Ghost / Text / Danger / Icon / Icon-only) Preview
  • size 4종(xs / sm / md / lg)별 height · padding · font-size · radius 기준
  • state 8종(default · hover · active · focus-visible · disabled · loading · selected · danger) 매트릭스
  • Icon Button — icon-left / icon-right / icon-only square / icon-only circle
  • Loading 상태 후보(텍스트 유지 + spinner / 텍스트 숨김 + spinner only)
  • Focus & Accessibility 기준 (focus-visible · aria-label · aria-busy 등)
  • variant별 사용처 가이드
  • Cursor 구현 후 QA Checklist
이 문서가 다루지 않는 것
  • stam/css/** 토큰 추가 / 값 변경 / 클래스 작성
  • stam/js/**, stam/pages/**, stam/index.html 변경
  • stam/assets/** 변경
  • Form · Dialog · Drawer · Table 등 다른 컴포넌트 CSS
  • Firebase · GitHub Actions · package*.json 변경
  • 실제 제품 App Shell · 페이지에 부착
Button Token Mapping v1과의 관계 · 본 Preview의 모든 버튼 샘플 색·치수는 Button Token Mapping v1 Section 04 · 05 · 06 후보값을 그대로 인용한다.
· 단, 문서 내부 격리를 위해 실제 --btn-* 토큰이 아닌 Preview 전용 --bcp-* 변수로 재선언한다.
· 따라서 본 문서는 실제 토큰 추가 없이도 라이트/다크에서 정확한 시각을 보여준다.
· Cursor 구현 시 --bcp-*를 --btn-*로 치환하면 동일한 결과가 나오도록 설계되었다.
주의 — Preview는 코드 변경이 아니다 본 문서의 .bcp-btn 계열 클래스는 이 문서 안에서만 동작하는 Preview 전용 클래스이다. 실제 제품·다른 docs에 영향을 주지 않으며, .stam-btn · .tbtn · .sbadge 등 기존 공통 클래스는 단 한 줄도 덮어쓰지 않는다. Cursor가 본 문서를 보고 새 토큰명·새 클래스명을 임의로 추가하지 않는다.

State Sample (Primary md) Visual Rule Token Candidate QA Check
default variant 기본 배경 · 텍스트 · 테두리. 그림자 없음. --btn-primary-bg · --btn-primary-text 대비비 4.5:1 이상 · 라이트/다크 동일
hover default보다 1단계 어둡게 (라이트) / 밝게 (다크). --btn-primary-bg-hover hover 단독 의존 금지 · 색만 변하고 크기 변화 없음

Loading 권장 규칙 — v1 결정안
  • v1 기본은 텍스트 유지 + small spinner 조합 — 텍스트는 “저장 중”처럼 진행 상태로 변경 권장

  • opacity만 낮춰 표시 금지 — cursor/state를 명확히 표현
  • 최소 300ms 이상의 작업에만 loading 표시 (그 이하는 깜빡임 회피)
  • spinner 색상 = variant text 색상 (currentColor 사용)
07 Focus & Accessibility Preview — 접근성 기준 필수

키보드 Tab으로 이동할 때만 ring이 떠야 하고, 마우스 클릭으로는 ring이 보이지 않아야 한다. focus-visible 효과를 강제 표시한 샘플과 함께 접근성 규칙을 정리한다.

focus-visible Ring Preview .is-focus 보조 클래스로 강제 표시
Primary focus ring --btn-focus-ring
Danger focus ring --btn-danger-ring
Icon-only focus ring aria-label 필수
focus-visible Ring
  • 모든 Button은 :focus-visible 시 외곽 ring 3px 필수
  • 마우스 클릭만으로는 ring이 떠오르지 않음 — :focus가 아닌 :focus-visible 사용
  • native focus outline은 대체 ring이 있을 때만 제거
  • Primary ring과 Danger ring은 색·hue 분리 (--btn-focus-ring vs --btn-danger-ring)
키보드 Tab 이동
  • Tab 키 순서는 시각 순서와 일치 — DOM 순서를 깨지 않음
  • Enter / Space 키로 활성화 가능
  • disabled Button은 Tab 순서에서 제외 — 단, 사유 안내 필요 시 aria-disabled="true"로 focusable 유지 가능
  • focus-trap이 필요한 컴포넌트(Dialog · Drawer) 내부에서도 동일 규칙 유지
aria 속성
  • icon-only Button — aria-label 필수 (시각 텍스트 없음)
  • loading 상태 — aria-busy="true" 부착
  • 토글성 Button(selected) — aria-pressed="true" 사용
  • disabled로 보여주되 클릭은 받고 싶을 때 — aria-disabled="true" 사용
Hit Area · 색 대비
  • 모바일 hit-area 최소 32×32, 권장 40×40
  • 텍스트 vs 배경 색 대비 4.5:1 이상
  • focus ring vs 배경 색 대비 3:1 이상
  • 라이트/다크 모두 동일 대비 보장
08 Button Usage Guide — variant별 사용처 사용처

화면 안에서 variant를 어떤 액션에 쓰는지 기준이다. 한 화면에 Primary 1개가 원칙이며, 보조 액션은 Secondary 또는 Ghost · Text로 내린다.

Variant 사용 액션 사용 위치 Sample Visual Priority
Primary 주요 저장 · 등록 · 승인 · 제출 Form 푸터 · Dialog footer 주요 CTA · 빈 상태 메인 액션 Highest
Secondary 취소 · 보조 액션 Primary 옆 페어 · Drawer footer 보조 High
Ghost Drawer footer 보조 액션 · Toolbar 보조 Drawer footer 좌측 · Toolbar 내부 · 메뉴 인라인 Medium
Text 링크성 액션 본문 내 인라인 · Card 우측 “자세히 보기” · Toast 내 액션 Low
Danger 삭제 · 반려 · 위험 작업 Confirm Dialog 주요 CTA · 삭제 컨펌 한정 Highest (warn)
Icon Table row action · Toolbar action Table row 우측 · Toolbar · Topbar 우측 Low–Medium
조합 원칙 한 화면 안 표준 조합: Primary 1 + Secondary 1 + (Ghost / Text 0~N). Danger는 일반 화면 CTA 자리에 두지 않으며, Confirm Dialog · 삭제 컨펌에 한정한다. Toolbar 우측 액션은 Ghost · Icon으로 우선 처리하고, “등록 +” 같은 주요 진입 액션만 Primary로 둔다.

  • console error / warning 0건
  • git diff에 허용 파일 외 변경 없음
핸드오프 시 명시 Cursor는 본 Preview 문서를 “구현 후 결과 화면의 시각 기준”으로 본다. Preview 전용 .bcp-btn 클래스는 실제 구현 시 .stam-btn으로 매핑되며, Preview 전용 --bcp-* 변수는 --btn-*로 매핑된다. 새 토큰명·새 클래스명을 본 문서 범위 밖에서 임의로 추가하지 않는다.
