상태: Skeleton — 구조만 생성된 상태입니다. 상세 규칙은 후속 작업에서 확장합니다.
01 목적 Skeleton
  • STAM 화면에서 사용하는 Button 컴포넌트의 위계 · 사이즈 · 상태 기준을 정의한다.
  • 상세 규칙은 후속 작업에서 확장한다.
02 적용 대상 Skeleton
  • 모든 화면의 Primary · Secondary · Danger · Ghost 버튼
  • Drawer 등록 · 수정 · 저장 액션 버튼
  • Dialog 내 확인 · 취소 버튼
03 기본 원칙 Skeleton
  • Primary · Secondary · Danger · Ghost 위계를 준수한다.
  • 크기는 sm · md · lg 3단계를 사용한다.
  • 아이콘 + 텍스트 조합은 허용한다.
04 Do Skeleton
  • Primary는 화면당 1개만 배치한다.
  • 위험 · 되돌릴 수 없는 작업에 Danger 버튼을 사용한다.
  • 아이콘 단독 버튼에 aria-label을 추가한다.
05 Don't Skeleton
  • 임의 배경색 · 크기 인라인 오버라이드 금지.
  • Disabled 버튼에 클릭 이벤트 처리 금지.
  • 버튼 텍스트를 아이콘만으로 대체 금지 (aria-label 없는 경우).
06 QA 체크리스트 Skeleton
  • 버튼 위계가 화면당 준수되었는지 확인.
  • Hover · :focus-visible 링 표시 확인.
  • Disabled 상태 pointer-events 처리 확인.
