01 Overview 가이드
컴포넌트Topbar (Navigation Bar)
높이--topbar-h: 48px
배경#0A0E1A (항상 다크)
위치sticky · top:0
관련 가이드Button · Icon · Drawer

목적 및 역할

  • STAM 모든 화면 상단에 고정되는 영역. 프로젝트 내부 전역 상태와 빠른 액션을 제공한다.
  • Left Navigation과 역할을 명확히 구분한다. Topbar는 전역 상태·빠른 액션, Left Nav는 IA 메뉴를 담당한다.
  • 두 가지 컨텍스트가 존재한다 — Doc Nav(가이드 문서)와 App Topbar(제품 화면).
  • 배경은 항상 #0A0E1A(다크 네이비). 라이트 테마에서도 변하지 않는다.
  • 좌·중·우 세 영역 구조를 유지한다. 영역 경계가 흐려지지 않게 한다.
이 문서가 다루는 범위 Topbar의 역할 정의, 구조(좌·중·우), 검색·알림·사용자 메뉴 처리, 반응형 축약 기준을 정리한다. 구체적 픽셀 값·CSS 변수는 UI Baseline Guide를 참조한다.
02 Topbar Role — 역할 정의 완료
Topbar가 담당하는 것
  • 현재 프로젝트 / 문서 위치 표시
  • 전역 검색 또는 현재 화면 검색 진입
  • 알림 · 멘션 진입
  • 사용자 메뉴 (프로필 · 로그아웃 등)
  • 테마 전환 (라이트 / 다크)
  • 도움말 · 가이드 진입
  • 화면 단위의 핵심 빠른 액션 (제한적)
Topbar가 담당하지 않는 것
  • 전체 IA 메뉴 대체 (← Left Nav)
  • 복잡한 필터 전체 배치 (← Table Toolbar / 페이지 본문)
  • 긴 업무 폼 입력
  • 제품 주요 네비게이션 전체
  • 본문 영역의 콘텐츠 액션 (등록·수정·삭제 본 액션)
  • 여러 줄로 늘어나는 정보 표시

03 Topbar Structure — 구조 (좌 · 중 · 우) 완료

Anatomy 다이어그램

LEFT STAM HR Renewal›요구사항정의서
CENTER 현재 화면 검색
RIGHT 3 김민

높이 48px · 배경 #0A0E1A · 좌·중·우 세 영역으로 분할 · 영역 사이 1px dashed 가이드선은 다이어그램 표시용이며 실제 UI에는 없음.

영역별 구성 요소

LEFT좌측 영역
  • STAM 로고 — 홈/대시보드 진입
  • Project Switcher 후보 — 현재 프로젝트명 표시 + 전환
  • Breadcrumb — 현재 위치 (메뉴 · 화면)
  • Page Title — 현재 화면 제목 (필요 시)
  • Back Link — 가이드 문서 등 이전 페이지 복귀
CENTER중앙 영역
  • Search — 전역/현재 화면 검색 진입
  • Context Summary 후보 — 마일스톤·현재 단계 등 요약
  • 중앙 영역은 비워두는 것도 허용 (좁은 화면 우선)
  • 입력형 위젯은 1개 이하로 제한
RIGHT우측 영역
  • Quick Action — 화면당 1–2개로 제한 (예: 신규 등록)
  • Notification — 알림·멘션 진입
  • Help / Docs — 가이드 문서 진입
  • Theme Toggle — 라이트/다크 전환
  • User Menu — 아바타 + 드롭다운
현재 구현 현황 현재 stam.shell.css의 .doc-nav는 좌측(로고·back·제목·pill)과 우측(테마 토글)을 중심으로 구성되어 있다. App Topbar 패턴은 같은 골격 위에 중앙 검색·우측 알림/사용자 메뉴를 더한 형태로 확장한다.
04 Topbar Button Rules — 버튼 기준 완료
  • Topbar 아이콘 버튼은 Button Guide의 Icon Button 또는 Secondary 계열을 사용한다.
  • Primary CTA(꽉 찬 브랜드 색 버튼)는 화면에 정말 필요한 경우에만 1개 이하로 둔다.
  • 알림·도움말·테마·사용자 메뉴는 같은 크기·같은 간격으로 배치한다. (32×32 Icon Button 권장)
  • 버튼 그룹 사이에는 6–8px 간격, 영역 사이에는 1px 세로 구분선 또는 그룹 간격을 사용한다.
  • 아이콘 단독 버튼은 aria-label 필수다. Icon Guide Icon Button Rules 참조.
  • 위험 액션은 Topbar에 두지 않는다. (삭제·반려 등은 Dialog 확인을 거친 본문 액션으로)
  • 버튼 색상은 본 Topbar의 다크 배경 위에서 충분한 대비를 확보한다.

05 Search / Filter Placement — 검색·필터 배치 완료
  • 전역 검색과 화면 내부 검색을 구분한다.
    • 전역 검색 — 프로젝트 전체에서 메뉴·화면·산출물을 찾는다. Topbar 중앙 또는 좌측에 둔다.
    • 화면 내부 검색 — 현재 화면 내 목록·필드를 좁힌다. Table Toolbar 또는 페이지 본문에 둔다.
  • Topbar에는 전역 검색 진입 또는 현재 화면 검색 트리거만 둔다. 복잡한 필터 전체를 노출하지 않는다.
  • 복잡한 필터는 Table Guide의 Toolbar 또는 페이지 본문 카드 안에 배치한다.
  • 검색 초기화(×) 아이콘은 Icon Guide 기준을 따른다. (12–14px Ghost Icon Button)
  • 검색 결과는 Topbar에서 직접 보여주지 않는다. 결과는 본문 콘텐츠 영역에서 표시한다.
  • 검색 입력 시 자동완성·최근 검색 등 보조 패널은 Popover로 띄운다. (Drawer 아님)
금지 패턴 Topbar에 다중 선택 필터·날짜 범위·고급 검색 옵션 전체를 직접 노출하지 않는다. Topbar는 진입점이고, 실제 필터 UI는 본문 또는 Drawer로 분리한다.
06 Notification / Mention Rules — 알림·멘션 완료
  • 알림·멘션 진입은 Topbar 우측에 배치한다.
  • 읽지 않은 알림은 카운트 배지로 표시한다. 99건 초과는 99+로 축약한다.
  • 읽지 않은 상태와 긴급 상태를 색상·텍스트로 구분한다. (예: 일반 = 브랜드, 긴급 = Red Orange)
  • 알림 목록 자체는 Topbar 안에 펼치지 않는다. Popover 또는 Drawer 후보로 분리한다.
  • 알림 항목에는 시각·발신자·요약 텍스트가 함께 노출된다. 색상만으로 우선순위를 표현하지 않는다.
  • 알림 카운트는 30초–1분 간격으로 갱신한다. 새 알림이 도착하면 시각·청각 알림이 과하지 않게 처리한다.

07 User Menu Rules — 사용자 메뉴 완료
  • 사용자 메뉴 트리거는 아바타(또는 이니셜) + 화살표 형태로 우측 끝에 둔다.
  • 드롭다운에는 사용자 이름·소속·역할을 상단에 표시한다. 마우스 hover 만으로 정보를 노출하지 않는다.
  • 메뉴 항목 후보 — 마이페이지, 알림 설정, 테마 설정, 도움말, 로그아웃.
  • 이메일·연락처·생년월일 같은 민감정보는 Topbar에 직접 노출하지 않는다.
  • 프로젝트별 권한과 전역 계정 정보는 메뉴 안에서도 구분해 표시한다.
  • 로그아웃은 메뉴 맨 아래에 두고, 시각적으로 다른 항목과 구분한다.
  • 여러 프로젝트에 속한 사용자는 사용자 메뉴와 별도로 Project Switcher를 좌측에 두는 것이 원칙이다.

08 Responsive Rules — 반응형 기준 완료
  • Topbar는 절대 두 줄로 늘어나지 않는다. 항상 한 줄을 유지한다.
  • 좁은 화면에서는 중앙 검색을 아이콘 트리거로 축약한다. (검색 입력은 Popover 또는 Drawer로)
  • 긴 페이지 제목·프로젝트명은 말줄임(`text-overflow: ellipsis`)으로 처리한다.
  • 모바일에서는 우측 액션의 일부를 More(⋯) 메뉴로 묶는다. 핵심(알림·사용자)만 직접 노출한다.
  • 좌측 Breadcrumb은 좁은 화면에서 마지막 단계만 남기고 나머지를 …로 축약한다.
  • 아이콘 버튼 우선 노출, 텍스트 라벨은 공간이 있을 때만 노출한다.
  • Topbar가 너무 좁아져 핵심 정보(현재 위치·사용자)가 사라지지 않도록 최소 노출 우선순위를 정한다.

09 Do / Don't 완료
Do
  • 현재 프로젝트·화면 위치를 명확히 보여준다
  • 우측 액션 크기·간격을 통일한다
  • 검색·알림·사용자 메뉴 역할을 분리한다
  • Left Navigation과 역할을 분명히 구분한다
  • 중요한 액션만 배치한다 (화면당 Quick Action 1–2개)
  • 좁은 화면에서는 단계적으로 축약한다
  • 아이콘 버튼에는 aria-label을 제공한다
  • 배경은 항상 다크 (#0A0E1A) 유지한다
Don't
  • Topbar에 모든 메뉴를 다 넣지 않는다
  • 긴 필터·날짜 범위·다중 선택을 Topbar에 직접 두지 않는다
  • 버튼 크기·정렬을 화면마다 다르게 만들지 않는다
  • 현재 프로젝트·화면 정보를 숨기지 않는다
  • 사용자 메뉴에 민감정보(이메일·연락처)를 직접 노출하지 않는다
  • Topbar가 두 줄로 늘어나게 하지 않는다
  • 라이트 테마에서 배경을 흰색으로 바꾸지 않는다
  • 위험 액션(삭제·반려)을 Topbar에서 즉시 실행시키지 않는다

11 Accessibility — 접근성 완료
  • Topbar 루트 요소는 <header> 또는 role="banner"로 표시한다.
  • 아이콘 단독 버튼에는 의미 있는 aria-label을 부여한다. (예: "알림 3건")
  • 현재 위치(Breadcrumb)는 aria-current="page"로 명시한다.
  • 키보드 Tab 순서는 좌 → 중 → 우 자연 순서를 유지한다. DOM 순서와 시각 순서를 일치시킨다.
  • 테마 토글 버튼은 현재 상태를 라벨로 설명한다. (예: "다크 모드 전환" / "라이트 모드 전환")
  • 알림 배지는 aria-label에 카운트와 내역 요약을 함께 제공한다. (배지 시각만으로 의미가 닫히지 않게)
  • 색상만으로 알림 우선순위·테마 상태를 표현하지 않는다. 아이콘 형태와 텍스트를 함께 사용한다.
  • Focus ring은 다크 배경 위에서도 명확히 보이는 색상(밝은 브랜드 톤)을 사용한다.
