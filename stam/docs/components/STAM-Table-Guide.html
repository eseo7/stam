01 Overview 가이드
컴포넌트Table (목록형 데이터 그리드)
기본 모드보통 모드 (핵심 컬럼)
확장 모드전체보기 / Focus View
행 클릭상세 Drawer 열기

목적 및 역할

  • Table은 STAM 전체 목록형 산출물 관리의 기본 컴포넌트다.
  • WBS, 요구사항정의서, 메뉴구조/화면목록, 화면설계서, 기능정의서, 프로그램 목록정의서, API 명세서, 테이블 정의서, DB 정보, 서버 정보, 접근/보안 정보, 정책정의서, 오픈 시나리오 등 주요 게시판의 목록 패턴을 통일한다.
  • 같은 화면이라도 보통 모드와 전체보기 / Focus View를 명확히 구분한다.
  • 보통 모드는 다른 작업과 병행하기 위한 축약 목록이고, 전체보기는 폭넓은 분석·편집을 위한 작업 환경이다.
  • 행 클릭은 상세 Drawer로 연결되는 것이 기본이며, 큰 상세는 Drawer 안 전체보기 버튼으로 별도 전체 화면으로 연결한다.

04 Column Rules — 컬럼 기준 완료
  • 보통 모드는 핵심 컬럼만 노출한다. 화면당 5–7개를 목표로 한다.
  • 상세 컬럼·부가 컬럼은 전체보기에서 확장한다.
  • 컬럼명은 업무 용어 기준으로 짧고 명확하게 쓴다. 기술 용어·축약 코드는 사용하지 않는다.
  • 상태 · 단계 · 담당자 · 기간 · 업데이트일은 STAM 공통 컬럼이다. 가능한 한 명칭과 표시 방식을 통일한다.
  • 숫자·날짜는 정렬 기준을 명확히 한다. 숫자는 우측 정렬 + tabular-nums, 날짜는 YYYY-MM-DD.
  • 긴 텍스트는 말줄임 + 상세 Drawer 연결을 사용한다. Cell 안에 줄바꿈을 강제하지 않는다.
  • 컬럼 순서는 업무 판단 순서에 맞춘다. (식별 → 분류 → 상태 → 담당 → 기간 → 업데이트)
  • 컬럼 폭은 화면마다 임의로 다르게 만들지 않는다. 공통 컬럼은 동일 폭으로 통일한다.

05 Scroll / Width Rules — 스크롤·폭 기준 완료
  • 페이지 레벨 상단 별도 가로 스크롤 바는 만들지 않는다.
  • 기본 스크롤은 브라우저 세로 스크롤 기준으로 둔다.
  • 가로 스크롤이 필요한 경우 컨텐츠 / 테이블 영역 내부에서만 발생시킨다.
  • 좌측 네비는 고정하고 컨텐츠 영역만 가로 스크롤한다. 좌측 메뉴가 밀리지 않는다.
  • Drawer가 열려도 테이블 레이아웃을 밀거나 줄이지 않는다. Drawer는 콘텐츠 영역 위 Overlay다.
  • 테이블 카드는 페이지 전체 폭을 불필요하게 넘기지 않는다. Card max-width 안에서 동작한다.
  • 전체보기에서는 좌측 메뉴를 숨기고 테이블 가용 폭을 확대한다.
  • 모바일에서는 카드형 목록 또는 중요 컬럼 우선 노출(2–3개)을 검토한다.

06 Row Interaction Rules — 행 인터랙션 기준 완료
  • 행 클릭은 상세 Drawer를 기본으로 한다.
  • 더블클릭 또는 별도 버튼으로 전체 상세 화면 이동을 고려할 수 있다.
  • 행 안의 액션 버튼은 Button Guide의 Small / Icon Button 기준을 따른다.
  • 체크박스 선택과 행 클릭은 충돌하지 않게 한다. 체크박스 영역은 행 클릭 이벤트와 분리한다.
  • 링크·버튼·체크박스 영역은 행 클릭 이벤트와 분리한다. 이벤트 버블링을 막는다.
  • 위험 액션(삭제·반려)은 Dialog Guide 확인을 거친다. 행 액션에서 즉시 실행하지 않는다.
  • 수정은 Drawer 안에서 모드 전환(상세 → 수정)을 우선한다. 별도 페이지 이동을 강제하지 않는다.

07 Filter / Search Rules — 필터·검색 기준 완료
  • 빠른 필터는 실제 필터처럼 조건 외 행을 숨긴다.
  • 조건 외 행을 단순 dim 처리만 하는 방식은 기본 패턴으로 사용하지 않는다.
  • 검색어 입력 시 메뉴명 · 화면명 · 작업명 · 담당자 · 상태 등 주요 텍스트를 대상으로 한다.
  • 필터 초기화 버튼을 제공한다. 활성 필터가 하나라도 있으면 노출한다.
  • 활성 필터는 칩 또는 배지로 명확히 표시한다. 각 칩에 제거(×)를 둔다.
  • 필터 결과 없음 상태를 제공한다. "검색 조건에 맞는 항목이 없습니다" + 필터 초기화 액션.
  • 필터와 검색은 목록 카운트와 연동한다. 전체 124 · 필터 18 형식으로 보여준다.
금지 패턴 조건에 맞지 않는 행을 단순히 회색으로 dim 처리해서 "필터처럼" 보이게 만드는 패턴은 금지한다. 사용자가 실제 데이터 건수를 오해할 수 있다.
08 WBS Table Special Rules — WBS 전용 기준 완료
  • 보통 모드에서는 컬럼 수를 줄여 스크롤을 최소화한다. (작업명 · 담당자 · 상태 · 진행률 · 기간 위주)
  • 전체보기에서는 상세 컬럼과 간트 / 일정 영역을 확장한다.
  • 간트차트 영역은 접기 / 펼치기를 제공한다. 기본은 펼침, 보통 모드에서는 접힘 가능.
  • 메뉴 / 기능 단위 그룹핑을 지원한다.
  • 그룹 헤더는 접기 / 펼치기, 그룹명, 작업 수, 대표 메타(담당 · 평균 진행률 등)를 표시한다.
  • 내 담당 · 리스크 · 지연 같은 빠른 필터는 조건 행만 남긴다. (dim 처리 금지)
  • 기간 · 담당자 · 상태 · 진행률 · 리스크는 WBS 주요 컬럼으로 본다.

10 Do / Don't 완료
Do
  • 보통 모드와 전체보기를 명확히 분리한다
  • 핵심 컬럼 중심으로 기본 목록을 구성한다
  • 빠른 필터는 실제 필터로 동작시킨다
  • 행 클릭과 행 액션을 분리한다
  • 긴 텍스트는 상세 Drawer로 보완한다
  • 전체보기에서 넓은 작업 환경을 제공한다
  • 공통 컬럼은 화면마다 동일 폭으로 통일한다
  • 필터·검색은 목록 카운트와 연동한다
Don't
  • 기본 목록에 모든 컬럼을 다 넣지 않는다
  • 조건 외 행을 dim 처리만 해서 필터처럼 보이게 만들지 않는다
  • Drawer가 테이블 폭을 밀어내게 만들지 않는다
  • 상단 별도 가로 스크롤 바를 만들지 않는다
  • 컬럼 폭을 화면마다 임의로 다르게 만들지 않는다
  • 삭제·반려를 행 액션에서 즉시 실행하지 않는다
  • 한 셀에 줄바꿈을 강제해 행 높이를 변화시키지 않는다
  • 좌측 메뉴를 가로 스크롤 영역에 포함하지 않는다

12 Accessibility — 접근성 완료
항목기준
Header ↔ Cell 관계 <th scope="col"> 사용. 복잡한 경우 id ↔ headers 매핑으로 명확히 한다
정렬 상태 정렬 가능한 컬럼은 aria-sort="ascending|descending|none" + 텍스트/아이콘으로 함께 안내
행 액션 버튼 아이콘 버튼은 aria-label 필수. 행 컨텍스트("WBS-031 수정") 포함 권장
키보드 포커스 Tab으로 Toolbar → Header → Row → Action 순서. 행 단위 포커스 이동 시 시각 인디케이터 유지
선택된 행 aria-selected="true" + 색상 외 시각 단서(좌측 강조 라인 등) 제공
필터 결과 없음 "조건에 맞는 항목이 없습니다" 등 명확한 문구. 단순 빈 화면 금지
가로 스크롤 영역 스크롤 컨테이너에 tabindex="0" + role="region" + aria-label로 키보드 접근 보장
색상 단독 상태 표현 금지 상태는 색상 + Badge 텍스트 / 아이콘을 항상 함께 사용한다
