01 목적 Skeleton
  • STAM 화면에서 사용하는 입력 폼의 레이아웃 · 검증 · 레이블 기준을 정의한다.
  • 상세 규칙은 후속 작업에서 확장한다.
02 적용 대상 Skeleton
  • 등록 · 수정 Drawer 내 폼 필드
  • 검색 · 필터 바
  • 설정 · 관리 화면 폼
03 기본 원칙 Skeleton
  • 레이블은 필드 상단에 위치한다.
  • 필수 항목은 *로 표시한다.
  • 검증 상태(정상/오류/경고)를 색상으로 구분한다.
04 Do Skeleton
  • Error 메시지는 해당 필드 바로 아래에 배치한다.
  • 선택 항목은 "(선택)"으로 명시한다.
  • 레이블과 입력 영역은 항상 함께 노출한다.
05 Don't Skeleton
  • Placeholder만으로 레이블을 대체 금지.
  • 검증 오류를 팝업(Toast/Alert) 단독 처리 금지.
  • 필수 항목 * 표시 없이 저장 오류 처리 금지.
06 QA 체크리스트 Skeleton
  • 레이블 위치(필드 상단) 확인.
  • 필수 항목 * 표시 확인.
  • 검증 오류 색상 및 메시지 위치 확인.
  • Focus 링(:focus-visible) 표시 확인.
