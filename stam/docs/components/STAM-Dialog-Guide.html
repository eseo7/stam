상태: Skeleton — 구조만 생성된 상태입니다. 상세 규칙은 후속 작업에서 확장합니다.
01 목적 Skeleton
  • STAM 화면에서 사용하는 Dialog의 유형 · 버튼 구성 · Focus Trap 기준을 정의한다.
  • 상세 규칙은 후속 작업에서 확장한다.
02 적용 대상 Skeleton
  • 확인 · 경고 · 위험 상황에서 사용자 의사 확인이 필요한 경우
  • 삭제 · 강제 종료 · 되돌릴 수 없는 작업
  • 간단한 정보 입력이 필요한 경우 (Input Dialog)
03 기본 원칙 Skeleton
  • Confirm · Alert · Danger 유형을 구분해서 사용한다.
  • 버튼은 최대 2개로 구성한다.
  • Focus Trap 적용은 필수다.
04 Do Skeleton
  • 위험 작업에 Danger Dialog를 사용한다.
  • Overlay 배경 클릭 시 닫기를 지원한다.
  • 취소 버튼을 항상 포함한다.
05 Don't Skeleton
  • Dialog 중첩 금지.
  • 단순 정보 전달 목적으로 Dialog 남용 금지.
  • 취소 버튼 생략 금지.
06 QA 체크리스트 Skeleton
  • Focus Trap 동작 확인.
  • Overlay dim 적용 확인 (rgba(23,23,25,.45)).
  • 버튼 구성 기준 (최대 2개) 확인.
