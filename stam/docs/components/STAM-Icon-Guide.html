01 Overview 가이드
컴포넌트Icon (SVG)
렌더 방식inline SVG (currentColor)
기본 크기16 · 18 · 20 · 24
기본 색상텍스트 색상 상속

목적 및 역할

  • STAM 전체 화면에서 사용하는 아이콘 기준을 통일한다.
  • 아이콘은 보조 의미 전달 수단이며 텍스트를 완전히 대체하지 않는다. 라벨이 항상 우선이다.
  • 화면마다 임의로 새 아이콘 스타일을 만들지 않는다. Set·크기·두께·정렬을 통일한다.
  • 모든 아이콘은 inline SVG로 삽입하고 currentColor로 색상을 상속받는다.
  • 아이콘 폰트, 이모지, 유니코드 글리프(예: ←, ✓, ★)는 아이콘 자리로 사용하지 않는다.
이 문서가 다루는 범위 아이콘의 의미 분류, 크기·정렬·색상 기준, 아이콘 단독 버튼 처리 기준, 접근성 라벨링 기준을 정리한다. 개별 SVG path 정의·아이콘 셋 인벤토리는 추후 별도 문서로 분리한다.
02 Icon Usage Principles — 사용 원칙 완료
  • 아이콘은 의미가 명확하고 보편적으로 해석되는 경우에만 사용한다.
  • 모호한 아이콘을 단독으로 쓰지 않는다. 라벨 텍스트가 함께 있어야 한다.
  • 아이콘만 있는 버튼은 aria-label 또는 title 속성이 필수다.
  • 색상만으로 상태를 전달하지 않는다. 텍스트·아이콘 형태·라벨로 함께 표현한다.
  • 같은 의미는 같은 아이콘으로 통일한다. (예: 검색 = 돋보기, 닫기 = ×)
  • 크기·두께·정렬은 본 가이드 기준을 따른다. 화면마다 임의 변경 금지.
  • 텍스트와 함께 쓰는 아이콘은 baseline이 아니라 visual middle 정렬을 사용한다.
  • 테이블 행 액션 아이콘은 너무 강하게 보이지 않게 처리한다. (Ghost 또는 dim 색상)
  • 위험 액션(삭제·반려)에 사용하는 아이콘은 Danger 색상을 사용한다.
  • 장식용 아이콘(브랜드 로고 등)은 aria-hidden="true"를 부여한다.

04 Icon Size Rules — 크기 기준 완료
  • 아이콘 크기는 12 / 14 / 16 / 18 / 20 / 24 6단계만 사용한다.
  • SVG는 항상 width·height를 명시한다. viewBox만으로 크기를 추정하지 않는다.
  • 같은 컨텍스트(같은 Toolbar·같은 행) 안에서는 같은 크기로 통일한다.

컨텍스트별 권장 크기

12px 보조 메타·작은 상태 — 카드 메타 라인, 표 내부 작은 인디케이터, 보조 캡션 옆 점·체크.
14px 칩 / 배지 내부 — 상태 Badge 내부 아이콘, 작은 Chip 내부 ×, 작은 알림 배지.
16px 기본 텍스트 옆 아이콘 — 본문(12–13px) 텍스트와 함께 쓰는 인라인 아이콘, Form Label 옆 도움말 아이콘.
18px 버튼 내부 기본 아이콘 — Medium 버튼(32–36px) 내부 보조 아이콘. Button Guide 기본값.
20px Topbar / Toolbar 아이콘 버튼 — 32–36px 정사각 Icon Button 내부 아이콘. Left Nav 메뉴 아이콘 기본 크기.
24px 빈 상태·큰 안내 카드 — Empty State 일러스트, 안내 카드 헤더 아이콘, 큰 결과 메시지.
피해야 할 크기 10px 이하 아이콘은 라인 가독성이 떨어져 사용하지 않는다. 26·28·30px 등 비표준 크기도 사용하지 않는다.
05 Icon Button Rules — 아이콘 버튼 기준 완료
  • 아이콘 단독 버튼은 Button Guide의 Icon Button 기준을 따른다.
  • 최소 클릭 영역은 32×32(데스크톱) · 44×44(모바일)를 보장한다.
  • 아이콘만 있는 버튼에는 반드시 aria-label 또는 title이 있어야 한다.
  • 삭제·반려 등 위험 액션에 쓰는 Icon Button은 Danger 색상을 사용한다.
  • 같은 도구 그룹의 Icon Button은 같은 크기로 통일한다. (예: Topbar 우측은 모두 32×32)
  • Hover·Focus·Pressed 상태가 시각적으로 구분되어야 한다.

06 Icon Alignment Rules — 정렬 기준 완료
  • 텍스트 앞 아이콘은 visual middle 정렬을 사용한다. baseline 정렬 금지.
  • SVG는 항상 width·height를 명시한다.
  • 아이콘과 텍스트가 같이 있는 컨테이너는 display: inline-flex; align-items: center;로 처리한다.
  • line-height가 1.4–1.5인 본문 텍스트와 16px 아이콘을 함께 쓰면 자연스럽게 정렬된다.
  • 테이블 셀 안 아이콘은 컬럼 의미에 맞춰 위치를 통일한다.
    • 상태/구분 아이콘 → 텍스트 좌측 (텍스트 앞)
    • 액션 아이콘 → 행 우측 끝 (Row Action 컬럼)
    • 식별 텍스트 옆 보조 아이콘 → 텍스트 좌측, 동일 간격
  • 아이콘과 텍스트 사이 간격은 gap: 4px(컴팩트) · gap: 6px(기본) · gap: 8px(여유)로 통일한다.

07 Do / Don't 완료
Do
  • 의미가 명확한 아이콘만 사용한다
  • 아이콘 단독 버튼에 aria-label 또는 title 라벨을 제공한다
  • 같은 의미는 같은 아이콘으로 통일한다
  • 크기·두께·정렬을 Icon Guide 기준대로 맞춘다
  • 상태는 아이콘 + 상태 텍스트 함께 사용한다
  • 위험 아이콘은 신중하게, Danger 색상과 함께 사용한다
  • 장식용 아이콘은 aria-hidden="true" 처리한다
  • SVG는 currentColor로 텍스트 색을 상속받는다
Don't
  • 의미가 모호한 아이콘을 단독으로 쓰지 않는다
  • 색상만으로 상태를 전달하지 않는다
  • 화면마다 다른 크기·두께를 만들지 않는다
  • 텍스트와 아이콘 세로 정렬을 어긋나게 두지 않는다
  • 클릭 영역이 32px 미만인 Icon Button을 만들지 않는다
  • 아이콘 폰트·이모지·유니코드 글리프를 아이콘 자리에 쓰지 않는다
  • 같은 행 안에 outline·fill 변형을 섞지 않는다
  • 위험 아이콘을 안전 액션에 남용하지 않는다

09 Accessibility — 접근성 완료
  • 아이콘 단독 버튼은 aria-label로 동작 의미를 제공한다.
  • 마우스 hover 보조 라벨이 필요하면 title도 함께 사용한다. (스크린리더는 aria-label 우선)
  • 장식용 아이콘은 aria-hidden="true"를 부여하고 라벨 텍스트가 모든 의미를 담당한다.
  • 상태를 색상만으로 표현하지 않는다. 아이콘 형태·텍스트로 동시 표현한다.
  • Icon Button은 :focus-visible 상태에서 명확한 포커스 링을 노출한다.
  • 키보드 Tab 순서가 자연스러워야 한다. 시각 순서와 DOM 순서를 일치시킨다.
  • 접기/펼치기 트리거 아이콘은 aria-expanded를 동기화한다.
  • 토글 상태(예: 즐겨찾기 ON/OFF)는 aria-pressed로 표현한다.

Topbar 전역 성격만 테마, 사용자, 프로젝트 전환, 도움말 위험 액션 아이콘 배치 금지. 화면 단위 액션은 toolbar로 이동.
Page Toolbar 보조 액션 / 보기 전환 검색, 필터, 정렬, 보기 전환, 내보내기 보조 등록 · 저장 같은 대표 액션은 텍스트 버튼을 우선한다. 아이콘만으로는 표현하지 않는다.
Drawer Header / Footer 닫기 · 접기 · 보조 정보 닫기(✕), 접기 / 펼치기, 정보(ⓘ) 저장 · 검토 요청 · 승인 같은 footer 액션은 텍스트 버튼 기준을 따른다.
Dialog 경고 · 확인 · 정보 보조 경고 ⚠ · 정보 ⓘ · 성공 ✓ · 닫기 ✕ 닫기 아이콘은 명확한 라벨이 필요하다. 위험 액션은 아이콘만으로 표현하지 않는다.
Table Row Action 공간 절약용 액션 보기 · 수정 · 복사 · 더보기 삭제는 직접 노출보다 더보기 메뉴 또는 확인 단계로 분리하는 것을 권장한다.
Empty State 분위기 / 설명 보조 비어있음 · 검색 결과 없음 일러스트성 아이콘 CTA는 텍스트 버튼으로 제공한다. 아이콘이 CTA를 대체하지 않는다.
Mobile / Narrow Width 제한적 사용 닫기 · 더보기 · 검색 터치 영역 최소 44px. 아이콘 사이 간격 확보. 아이콘만 나열해 의미가 모호해지는 패턴 금지.
주의 영역별 기준은 “아이콘을 어디까지 보조적으로 쓸 것인가”에 대한 원칙이다. 실제 제품 CSS class · 아이콘 세트 / 이름은 본 PR에서 단정하지 않는다.
13 Icon + Text — 조합 패턴 1차 상세 기준

5가지 기본 패턴

패턴 의미 적합 사용처
텍스트 앞 아이콘 액션 의미를 먼저 암시 등록(＋) · 저장(💾) · 검색(🔍) 같이 동작이 즉시 인지되는 버튼
텍스트 뒤 아이콘 보조 결과 / 외부 이동 / 열기 외부 링크(↗) · 다운로드(↓) · 펼치기(▾) · 새 탭
아이콘 단독 버튼 공간 제약 + 익숙한 의미 닫기(✕) · 새로고침 · 더보기(⋯) · 필터 · 검색 초기화 — 반드시 aria-label / 숨김 텍스트 동반
상태 chip 안 아이콘 색상 의존도를 낮춤 승인(✓) · 반려(✕) · 경고(⚠) · 진행 중 같이 상태가 색 외에도 형태로 구분되어야 할 때
안내 문구 옆 아이콘 tone 보조 (정보 · 주의 · 위험) ibox / banner / inline help 의 ⓘ ⚠ ⛔ — 문구를 대체하지 않고 보조한다

조합 원칙

  • 텍스트 앞 아이콘은 “무엇이 일어나는가”를 먼저 보여준다. 동작이 명확한 액션에 적합하다.
  • 텍스트 뒤 아이콘은 “결과 / 이동 방향”을 보조한다. 외부 이동 · 다운로드 · 펼치기 등 보조 의미에 적합하다.
  • 아이콘 단독 버튼은 의미가 보편적으로 인지되는 경우(닫기 · 더보기 등)에만 허용한다. 그 외에는 텍스트를 함께 둔다.
  • 상태 chip 안 아이콘은 색약 사용자 / 흑백 출력 / 스크린리더 환경을 고려해 형태가 의미를 보강하도록 한다.
  • 안내 문구의 아이콘은 tone(정보 / 주의 / 위험)을 직관적으로 알리되, 문구를 대체하거나 줄이지 않는다.
본 조합 패턴은 “아이콘이 텍스트와 어떻게 함께 동작하는가”에 대한 기준이다. 실제 SVG · 아이콘 세트 · class 이름은 별도 토큰 / 아이콘 시스템 작업에서 확정한다.
