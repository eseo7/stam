STAM Board Form Field Contract v1

게시판 상세/등록/수정 Drawer 안에서 쓰는 Form field(label·input·help·error·required·grid)를 공통 stam.form-controls.css 기준으로 통일하기 위한 계약 문서. Drawer 전면 공통화 전, field 단위부터 수렴한다.

버전: v1 공통 CSS: stam.form-controls.css 기준 PR: #102 · #115 Drawer Audit 성격: Form Field 계약
Related: Drawer Audit v1 Form Controls Preview v1 Board List Contract v1 Custom Select Contract v1 Form Controls Guide Docs Index
Section 1 · Summary

신규/기존 게시판 Drawer Form 은 가능한 한 동일한 field 구조를 사용한다.

Drawer 전체 공통화(shell/header/footer) 이전에, label / input / help / error / required / grid 단위부터 공통 stam.form-controls.css 클래스로 수렴한다. PR #115 Drawer Audit 의 1순위 권장 항목이다. 이미 fn/msl/rq drawer 가 .stam-label/.stam-input/.stam-textarea/.stam-select 를 병행 사용 중이므로 진입 위험이 낮다.

* 본 계약은 문서다. 제품 CSS/JS 는 수정하지 않으며, 화면별 적용은 별도 코드 PR 로 진행한다. 새 class 가 필요하면 "문서 제안" 으로만 남기고 제품 CSS 는 건드리지 않는다.

Section 5 · Label / Required 계약
  • label 위치: 컨트롤 위(top-label), .stam-label (block, margin-bottom 4px). 가로 라벨은 .stam-form-row.is-horizontal 만 예외.
  • 필수값 표시: 라벨 끝에 <span class="req">*</span>. 색 --fc-err-bd(red), 700.
  • 선택 표시: 필요 시 <span class="opt">(선택)</span> — t3, 약하게.
  • required mark 문구/시각: 기호는 * 통일(텍스트 "필수" 남발 금지). 화면별 색/위치 재정의 금지.
  • 접근성: 필수 입력은 required 또는 aria-required="true", 오류 시 aria-invalid="true" + 에러 메시지 id 를 컨트롤 aria-describedby 로 연결 권장. label 은 for/id 또는 wrapping 으로 연결.
Section 6 · Grid / Layout 계약
<div class="stam-form-grid">
  <div class="stam-form-field"> ... </div>          <!-- 1칸 -->
  <div class="stam-form-field"> ... </div>          <!-- 1칸 -->
  <div class="stam-form-field full-span"> ... </div> <!-- 2칸 전체폭 -->
</div>
  • 1열: .stam-form-row(필드 세로 스택) 또는 grid 미사용.
  • 2열: .stam-form-grid (gap 16px, 640px 이하 자동 1열).
  • full-width field: grid 안에서 .full-span (grid-column: 1 / -1).
  • section divider / field group: 섹션 구분(번호·제목)은 화면별 레이아웃 허용 — field 자체 구조만 공통. 섹션 헤더는 도메인 자유(.fn-fs/.rq-fs 등 유지 가능).

Section 8 · Drawer 모드별 사용 기준
모드field 기준
상세(detail)읽기 표시 — 값은 .is-readonly input 또는 표시 전용 마크업. 편집 컨트롤 비활성/숨김. required mark 불필요.
등록(register)편집 가능 .stam-input/.stam-select/.stam-textarea/.stam-date + required mark + help/error.
수정(edit)등록과 동일 컨트롤 + 기존 값 prefill. 변경 불가 키(예: ID)는 .is-readonly.
상세 → 수정 전환전환 시 같은 필드 구조·라벨·grid·required 기준 유지. 멀티 drawer(fn/msl/rq) / 단일 drawer+mode(ss/wbs) 어느 방식이든 field 마크업은 동일 계약.
Section 9 · 금지 사항

✅ Do

  • 공통 .stam-form-field/.stam-label/.stam-input/... 사용
  • required = .stam-label .req 통일
  • help/error = 컨트롤 아래 .stam-field-help/.stam-field-error
  • grid = .stam-form-grid + .full-span
  • 화면 고유는 섹션 레이아웃·도메인 chip 만

❌ Don't

  • 화면별 label/input 시각 스타일 새로 만들기
  • required 표시 방식 제각각(색/위치/문구) 구현
  • help/error 위치 제각각 구현
  • 제품 CSS(stam.form-controls.css 등) 수정 / 복붙
  • 저장/API/localStorage/Firestore 로직 변경
Section 10 · 적용 우선순위
  1. 기능정의서 — 이미 .stam-label/.stam-input/.stam-textarea 병행. required(.fn-req-mark→.req)·help/error 정리 위주, 최저 위험.
  2. 메뉴/화면목록 — .msl-flbl.stam-label/.msl-inp.stam-input 병행. 요구사항과 쌍둥이.
  3. 요구사항 — .rq-* + .stam-* 병행. 메뉴/화면목록 패턴 재사용.
  4. Open Scenario — 신규 등록/수정 drawer 생길 때 본 계약 적용(현재 읽기전용).
  5. 화면설계서 / WBS — 특수 구조(JS 렌더 / 전용 form class)로 별도 검토. 화면설계서 렌더 폼·WBS .wbs-drawer-form-* 는 구조 검토 후 수렴 여부 결정.
Section 11 · QA 체크리스트

  2. required 표시 — .req 빨강 *.
  3. input/select/textarea 높이 — 36px(--fc-h) 일관.
  4. focus 상태 — border --fc-bd-focus + ring.

  6. readonly display — bg --fc-bg-ro (disabled 와 구분).
  7. drawer 상세/등록/수정 mode 전환 시 field 구조 유지.
