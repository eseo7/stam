STAM Screen Template
Priority 30 Korean Baseline v1

STAM 화면설계서(Screen Specification) 시스템에서 우선 제공할 템플릿 30종을 한국형 업무 시스템(웹/관리자/그룹웨어/전자결재/프로젝트 산출물) 기준으로 정의한 Baseline 문서다. 이 문서는 상세 와이어프레임이나 최종 UI 구현 문서가 아니라, "무엇을 우선 템플릿으로 제공할지"를 정하는 우선순위 목록이다. 각 템플릿의 목적·유형·기본 필드·기본 액션·권한 포인트·모바일 고려사항을 정리하고, 후속 Top 10 Detail Pack에서 상세화할 후보를 선정할 수 있게 한다. 기준 커밋은 main 557beff (PR #170 merge) 이다.

Selection Criteria — 30종 선정 기준

① 국내 업무 시스템 빈출한국형 웹/관리자/그룹웨어/전자결재/프로젝트 산출물 화면에서 자주 반복되는 화면 유형인가.
② STAM 산출물 흐름 기여요구사항·화면설계·WBS·테스트·승인 등 STAM 산출물 관리 흐름에 직접 도움이 되는가.
③ 화면설계서 초안 가속해당 템플릿으로 화면설계서 초안을 빠르게 만들 수 있는가(선택 → 수정 흐름 적용 가능 여부).
④ Board Builder 연결성Board Builder의 필드/게시판 구조와 연결 가능한가. 게시판 구조 → 화면 산출물 흐름이 자연스러운가.
⑤ 권한 / 검토 / 승인 / 이력 연결화면/문서/버튼/액션/필드/컴포넌트 단위 권한과 검토·승인·변경이력 흐름에 연결 가능한가.
⑥ 모바일 / 좁은 화면 대응PC 기준 설계에 더해 모바일/좁은 화면 대응 기준(카드 전환, 컬럼 우선순위, 드로워)을 줄 수 있는가.
⑦ Top 10 확장 가능성후속 Top 10 Detail Pack에서 레이아웃·필드·상태·액션을 상세화할 후보로 확장 가능한가.
⑧ 1차 범위 우선 적용모든 화면을 한 번에 다루지 않는다. 빈도·중요도·연결성이 높은 30종을 우선 적용하고 나머지는 후속 단계로 분리한다.

Board Builder Relationship

직접 연계 템플릿 (6종)

Board Builder의 필드/게시판 구조와 직접 연결 가능한 템플릿: T01 Basic Board List · T05 Wide Data Table · T11 Basic Create Form · T14 Custom Field Form · T26 Board Settings · T27 Custom Field Settings.

→ Board Builder (pages/boards-v2/board-builder.html)

역할 분담

Board Builder는 필드/게시판 구조를 만들고, 화면설계서 템플릿은 그 구조를 화면 UX 산출물로 표현한다. 게시판 구조 정의 → 화면설계 템플릿 선택 → 화면설계서 작성 흐름으로 보완 관계를 형성한다.

현재 Board Builder는 preview / local 기준이며 실제 저장/배포 기능이 아니다. 입력 결과는 localStorage 기준 Preview이고 Firestore/API/fetch 호출이 없는 상태를 유지한다. 이 문서 작업은 Board Builder 제품 파일(stam/pages/boards-v2/board-builder.html)을 수정하지 않으며 참조 링크만 제공한다.

07

Permission / Review / History Notes

권한 단위

각 템플릿은 화면 / 문서 / 버튼 / 액션 / 필드 / 컴포넌트 단위 권한을 고려해야 한다. 목록·상세·폼·설정 템플릿마다 노출/편집/실행 권한 포인트를 구분한다.

검토 / 승인 연결

각 템플릿은 검토 요청 · 승인 요청 · 댓글/멘션 흐름(D그룹 공통 레이어)과 연결될 수 있어야 한다. 작성형/상세형 템플릿은 검토·승인 액션을 기본 액션에 포함한다.

변경이력 연결

단순 화면 모양이 아니라 산출물 운영 흐름까지 포함한다. 수정 가능한 템플릿은 버전·변경자·변경일 이력(T09 Change History Detail 패턴)과 연결한다.

D그룹(협업/검토/승인) 공통 레이어 구조는 STAM-Sitemap-IA-Overview-v1.html#layer를 참조한다.
08

Mobile / Narrow Width Notes

목록형 (List / Board)

좁은 화면에서 테이블 → 카드 전환 또는 필수 컬럼 우선순위를 적용한다. 핵심 필드(상태·이름·날짜)만 우선 노출하고 부가 필드는 전체보기로 제공한다.

Wide Data Table (T05)

가로 스크롤 정책을 명확히 가진다. 핵심 컬럼(키/이름)을 좌측 고정하고 나머지는 가로 스크롤로 처리한다.

Drawer / Modal 계열 (T16 등)

좁은 화면에서 우측 드로워/모달은 전체 화면 전환을 고려한다. 화면 80% 이상을 덮는 경우 Full Screen 모드로 대체한다.

Filter 계열 (T15–T17)

검색/필터 영역은 접힘/펼침 토글 또는 드로워 전환을 고려한다. 기본 접힘 상태로 본문 공간을 우선한다.
