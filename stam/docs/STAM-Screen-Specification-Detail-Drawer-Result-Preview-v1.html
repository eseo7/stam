01Executive Summary — 개요목적
문서 목적: 저장된 화면설계서를 상세 Drawer에서 조회하는 결과 화면 기준. Cursor 복귀 후 Detail Drawer 구현의 기준 문서로 사용한다.
Register Drawer에서 작성한 내용은 저장 후 Detail Drawer에서 읽기 중심으로 표시된다.
상세 Drawer는 목록 맥락을 유지하면서 기본정보·필드·액션·문구·연결·첨부·번호주석·변경이력·검토를 한 번에 확인하는 구조다.
저장 전 미리보기와 저장 후 상세 결과는 같은 렌더링 구조를 공유한다.
상세 Drawer에서 수정, 복제, 새 버전 생성, 검토요청, 내보내기 등 후속 액션을 제공한다.
번호 주석형 Preview가 상세 결과의 핵심 콘텐츠다.
승인 상태에서는 직접 수정하지 않고 새 버전 생성으로 연결한다.

17Cursor Implementation Notes — 구현 기준 메모후속 작업 참고
중요: 이 문서는 정적 Preview다. 실제 구현은 Cursor 복귀 후 별도 PR로 분리한다.
Detail Drawer는 Register Drawer와 공통 Drawer shell을 공유할 수 있다.
저장 전 미리보기와 저장 후 Detail Result는 같은 Visual Annotation renderer를 공유하는 것이 좋다.
저장 후에는 읽기 전용 상태로 표시한다.
수정 버튼 클릭 시 Register/Edit Drawer로 전환한다.
승인 상태에서는 직접 수정하지 않고 새 버전 생성으로 연결한다.
Item Usage 상태는 상세 결과에서도 조회 가능해야 한다.
원번호와 Preview 번호는 분리 저장해야 한다.
변경이력은 원번호 기준으로 추적한다.
내보내기는 후속 작업으로 분리 가능하다.
