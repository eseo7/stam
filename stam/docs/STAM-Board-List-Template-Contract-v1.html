STAM Board List Template Contract v1

STAM 의 모든 게시판 목록 화면이 동일한 목록 구조와 상태 규칙을 사용하도록 강제하는 기준 문서.

공통 CSS: stam.table-selection.css 기준 PR: #86 / #88 / #89 버전: v1
Related: Audit Report v1 Bare Preview (table-selection) Board Toolbar Guide Board Filter Guide Docs Index
Section 00 · 핵심 원칙
  1. 새 게시판 목록은 화면별로 검색/필터/삭제/checkbox/table row 스타일을 새로 만들지 않는다.
  2. 공통 class 와 공통 CSS/JS 규칙을 사용한다.
  3. 화면별 CSS 는 컬럼 폭 · Zone 배치 · 카드 배치 · 업무별 뱃지만 담당한다.
  4. checkbox · selected row · active row · left bar · delete button · filter button · search input 스타일을 화면별 CSS 에서 재정의 금지.
Section 01 · Board List 표준 HTML 구조
<!-- 1) 목록 상단 toolbar -->
<div class="stam-list-toolbar">

  <!-- 2) 검색 input -->
  <div class="stam-list-search">
    <svg class="stam-list-search-icon" width="13" height="13" viewBox="0 0 24 24" ...></svg>
    <input class="stam-list-search-input" type="search" placeholder="ID · 제목 · 담당자 검색">
  </div>

  <!-- 3) 필터 버튼 -->
  <button class="stam-filter-btn" type="button">
    <svg class="stam-btn-icon" ...></svg> 필터
  </button>

  <!-- 4) 삭제 버튼 -->
  <button class="stam-delete-btn" type="button" disabled>
    <svg class="stam-btn-icon" ...></svg>
    <span class="stam-delete-label">삭제</span>
  </button>

</div>

<!-- 5) selectable table -->
<table class="stam-select-table">
  <thead>
    <tr>
      <!-- 6) header checkbox -->
      <th class="stam-check-cell"><input class="stam-check" type="checkbox"></th>
      <th>ID</th><th>제목</th><th>담당자</th> ...
    </tr>
  </thead>
  <tbody>
    <!-- 7) row — row click 시 .is-active, checkbox 시 .is-selected -->
    <tr class="stam-table-row" data-id="...">
      <td class="stam-check-cell"><input class="stam-check" type="checkbox"></td>
      <td>...</td> ...
    </tr>
  </tbody>
</table>

* 페이지별 namespace 클래스(.os-row, .msl-data-row 등) 를 부가적으로 붙이는 것은 허용. 단 공통 class 는 반드시 포함.

Section 04 · 행 상태 규칙 (normal / .is-active / .is-selected / 둘 다)

* .is-active 는 drawer 대상 — 한 번에 하나만. 다른 row 클릭 시 이전 .is-active 자동 제거.
* .is-selected 는 bulk action 대상 — 다중 선택 가능. checkbox click 시에만 토글.

Section 06 · 필터 버튼 규칙
  • 버튼 class: .stam-filter-btn
  • open 상태: .is-open (열려있을 때 토글)
  • icon class: .stam-btn-icon
  • 화면별 색상 커스텀 금지 — var(--stam) · var(--stam-soft) 만 사용
  • 필터 팝업(.stam-board-filter-panel 등) 은 별도 가이드 (Board Filter Guide) 참조
Section 07 · 화면별 CSS 허용/금지

✅ 화면별 CSS 에서 허용

  • 컬럼 폭 (th/td width)
  • Zone / 카드 / 섹션 배치 (grid, flex layout)
  • 화면 고유 여백 (page padding, section gap)
  • 특수 업무 뱃지 (상태 chip, 등급 chip 등 도메인 표시)

❌ 화면별 CSS 에서 금지

  • checkbox 스타일 재정의
  • selected row background 재정의
  • active row bar 재정의
  • delete button 스타일 재정의
  • filter button 스타일 재정의
  • search input 기본 스타일 재정의
  • row height 재정의 (기본 36px)
Section 08 · 새 게시판 생성 체크리스트
  1. stam.table-selection.css 로드 여부 (페이지 <head> 확인)
  2. toolbar class .stam-list-toolbar 적용 여부
  3. 검색 wrapper/input/icon class 적용 여부
  4. 필터 버튼 .stam-filter-btn 적용 여부
  5. 삭제 버튼 .stam-delete-btn + .stam-delete-label 적용 여부
  6. 테이블 .stam-select-table 적용 여부
  7. 체크박스 cell/input class .stam-check-cell · .stam-check 적용 여부
  8. 행 .stam-table-row 적용 여부
  9. JS: stam.board-list.js 로드 및 window.STAMBoardList.init(table, options) 호출 (Section 10' 참조)
  10. JS: row click → .is-active 부여 / 다른 row 클릭 시 해제 (Controller 가 처리)
  11. JS: checkbox change → .is-selected 토글 (Controller 가 처리)
  12. JS: delete count = .is-selected 행 수 ↔ .stam-delete-label 갱신 (Controller 가 처리)
  13. placeholder 가 패턴({도메인 ID} · {도메인 명} · 담당자 검색) 을 따르는지
  14. 화면별 CSS 에 금지 셀렉터 부재 확인 (Section 07 ❌ 항목)
  15. Bare Preview (STAM-Common-Select-Table-Preview.html) 와 육안 비교
Section 09 · Do / Don't

✅ Do

  • 공통 class 사용
  • Bare Preview 기준과 육안 비교
  • .is-active 와 .is-selected 상태 분리
  • 레이블 텍스트 갱신은 .stam-delete-label 내부만
  • row 36px 유지

❌ Don't

  • .wbs-* · .ss-* · .os-* · .msl-* · .rq-* · .fn-* 로 공통 UI 재구현
  • 화면별 CSS 에서 checkbox/delete/filter/search/table-row 스타일 재정의
  • computed style 만 보고 PASS 처리 (실제 인터랙션 미검증)
  • row click 으로 checkbox 를 체크하는 동작
  • checkbox click 으로 drawer 를 여는 동작
Section 10' · 공통 JS Controller — window.STAMBoardList

새 게시판이 본 계약의 HTML 구조를 따른다면, 화면별 JS 에서 row active / checkbox selected / delete count / header checkbox 로직을 재구현하지 않는다. stam.board-list.js 로드 후 한 줄 init 호출만으로 동일한 동작을 얻는다.

<!-- 1) 컨트롤러 로드 (defer 권장) -->
<script src="../js/stam.board-list.js" defer></script>

<!-- 2) 테이블에 hook attribute 부여 -->
<table class="stam-select-table" data-stam-board-list>
  <thead><tr>
    <th class="stam-check-cell">
      <input class="stam-check" type="checkbox" data-stam-check-all>
    </th> ...
  </tr></thead>
  <tbody> ... </tbody>
</table>

<!-- 3) init — drawer 연동만 화면별로 작성 -->
<script>
window.STAMBoardList.init(document.querySelector('[data-stam-board-list]'), {
  onRowActivate: function (row, api) { openDetailDrawer(row.dataset.id); },
  onClearActive: function (row, api) { closeDetailDrawer(); },
  onDelete:       function (rows, api) { /* 화면별 데이터 로직 */ },
  // clearActiveOnCheckbox: false,
});
</script>
API설명
STAMBoardList.init(root, options)테이블에 인터랙션 위임을 부착. delete 버튼은 options.deleteBtn selector 또는 root 인근의 .stam-delete-btn 자동 탐지.
STAMBoardList.getSelectedRows(root)현재 .is-selected row 배열 반환.
STAMBoardList.clearActive(root)현재 .is-active 해제 + onClearActive callback 호출.
STAMBoardList.clearSelected(root)모든 .is-selected 해제, checkbox unchecked, delete count 초기화.
STAMBoardList.refresh(root)DOM 재렌더 후 호출 — delete count / header checkbox 재계산.
Options기본값설명
onRowActivate(row, api)—row click 직후 호출. drawer open 등을 여기서.
onClearActive(row, api)—Escape / 다른 row click 으로 active 해제 시 호출.
onSelectionChange(rows, api)—checkbox 토글 후 호출. selected row 배열 전달.
onDelete(rows, api)—delete 버튼 클릭 시 호출 (선택>=1일 때만). 실 삭제 로직은 화면별로.
clearActiveOnCheckboxfalsetrue 시 checkbox click 도 .is-active 해제.
deleteBtn.stam-delete-btndelete 버튼 selector 명시.

* 실제 데이터 삭제(API · localStorage · Firestore)는 본 컨트롤러에서 수행하지 않는다. onDelete callback 안에서 화면별로만 처리.
