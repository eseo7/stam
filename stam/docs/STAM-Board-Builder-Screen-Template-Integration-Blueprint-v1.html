Board Builder ↔ Screen Template
Integration Blueprint v1

Board Builder(게시판 만들기)에서 정의한 게시판/필드 구조가 화면설계서 Screen Template 체계의 화면 초안 생성 흐름으로 어떻게 연결되는지를 정의한 연결 설계(blueprint) 문서다. 실제 제품 로직·저장·API를 구현하거나 수정하지 않고, 두 시스템 사이의 데이터 모델 · 필드 타입 · 화면 매핑 · 사용자 흐름 · 권한 기준을 문서로 정리한다. P1~P3(Master Guide · Priority 30 · Top 10 Detail Pack)에서 정의한 템플릿 체계를 실제 Board Builder 출력과 잇는 P4 단계다. 기준 커밋은 main 44148cb (PR #172 merge)이다.

01

Role Definition — Board Builder vs Screen Template

Board Builder (게시판 만들기)

  • 게시판 구조 정의 (카테고리·게시판명·코드·설명·기본 템플릿)
  • 필드 정의 (라벨·타입·필수여부·옵션·정렬·표시 토글)
  • 목록/필터/입력폼 표시 여부 설정
  • 필드 타입/검증/권한 힌트 설정
  • 결과를 config(JSON) → localStorage → 미리보기로 출력
  • preview / local 기준 — 실제 Firestore 저장·배포 기능 아님
→ Board Builder (pages/boards-v2/board-builder.html)

Screen Template (화면설계서 템플릿)

  • 화면 유형 선택 (목록·등록·수정·상세·검색·설정)
  • 선택한 유형의 화면 초안 생성
  • 필드·액션·권한·상태·연결 화면을 화면설계서 산출물로 표현
  • 검토·승인·변경이력·내보내기 흐름과 연결
  • 선택 → 수정 원칙(Master Guide): 초안 생성 후 맥락에 맞게 보정
→ Master Guide
핵심 분담: Board Builder는 "무엇을 담는 게시판인가(구조·필드)"를 정의하고, Screen Template은 "그 구조를 어떤 화면으로 보여줄 것인가(UX 산출물)"를 정의한다. Board Builder 출력이 Screen Template 초안의 입력이 되는 단방향 연결을 기준으로 한다.

Board Builder Output Model

06

End-to-End User Flow

1
Board Builder 진입Board Builder

관리자가 게시판 만들기 화면 진입 (preview/local).

2
게시판 기본 정보 입력Board Builder

카테고리·게시판명·코드(자동)·설명·기본 템플릿 선택.

3
필드 추가/수정/정렬Board Builder

필드 추가, 드래그로 순서 변경, 목록/필터/입력폼 표시 토글.

4
필드 타입/옵션/필수 설정Board Builder

12종 타입 선택, 옵션값·필수 여부·검증·권한 힌트 설정.

5
미리보기 확인 + config 출력Board Builder

우측 실시간 미리보기 확인, JSON 복사 또는 localStorage 저장(로컬).

6
화면설계서 템플릿 선택Screen Template

목록/등록/수정/상세/검색/설정 등 화면 유형 선택 (Top 10 기준).

7
화면 초안 생성Screen Template

Board Builder config → 필드/컬럼/폼 초안 자동 구성 (섹션 04·05 매핑).

8
액션/권한/상태/문구 보정사용자

행 액션·권한 규칙·상태 흐름·빈상태/오류 문구를 맥락에 맞게 수정.

9
화면설계서 미리보기Screen Template

생성된 화면설계서 초안 미리보기로 확인.

10
검토 요청 또는 내보내기사용자

검토/승인 흐름 연결 또는 산출물 내보내기 (후속 제품 구현 범위).

5단계까지가 현재 Board Builder preview의 실제 동작 범위이고, 6단계 이후 화면설계서 초안 생성·미리보기·내보내기는 후속 제품 구현 대상이다. 이 Blueprint는 그 연결 기준을 제공할 뿐 구현을 지시하지 않는다.
07

Generated Draft Structure

Board Builder 결과가 화면설계서 초안으로 변환될 때 생성되어야 할 구조다. 이 구조는 화면설계서 템플릿 기준(Baseline)과 정합하도록 설계한다.

화면 식별

  • Screen Title — 게시판명 + 화면유형
  • Screen Type — 목록/등록/상세/검색/설정
  • Route / Menu Reference — nav-data 참조(SSOT, 수정 없음)

본문 구조

  • Field Table — 필드·타입·표시·필수
  • Action List — 화면별 기본 액션
  • Related Artifacts — relation 필드 기반 연결

규칙

  • Permission Rules — 화면/필드/액션 권한
  • Status Rules — 상태값·전이
  • Validation Rules — required/validation 기반

상태 화면

  • Empty State — 빈 상태 문구/액션
  • Error State — 오류 문구(404/403/네트워크)

대응/이력

  • Mobile Notes — 좁은 화면 기준
  • Export Notes — exportable 기반 내보내기
  • Change History Notes — 변경이력 연결

자동/수동 구분

  • 자동 필드표·기본 액션·검증·내보내기 후보
  • 수동 권한 세부·상태 전이·문구·연결 화면
08

Permission / Action / Field Control

화면 단위

화면 진입 권한 (목록/설정 화면 등 화면 자체 접근 제어). 관리자 전용 화면(T26·T27)은 진입 단계에서 차단.

문서/레코드 단위

레코드별 열람/편집/삭제 권한. 공개 범위(전체·프로젝트·본인)에 따라 행 노출 분기.

버튼/액션 단위

등록·삭제·내보내기·승인/반려 등 액션 버튼 권한. 권한 없을 때 버튼 미노출.

필드 단위

필드별 열람/편집 권한(permission 힌트). 역할별 필드 노출/마스킹.

컴포넌트 단위

상태 배지·결재선·이력 등 컴포넌트 단위 노출 제어. 참조자에게 읽기 전용 제공.

상태별 / 역할별 분기

상태별 수정 가능/불가(예: 승인완료 후 잠금). 담당자/검토자/승인자별 액션 분기(T08 결재선 기준).

권한 기준은 Board Builder의 permission 필드 힌트를 출발점으로 하되, 실제 권한 정책 화면(T25 Role/Permission Management)과 연결해 확정한다. D그룹 협업/검토/승인 공통 레이어는 STAM Sitemap & IA Overview (#layer)를 참조한다.
09

Preview / Local Constraint

현재 제약을 명확히 한다.

Board Builder는 preview / local

입력 결과는 Board Factory config(JSON) → localStorage 기준 미리보기다. 실제 게시판 생성·저장이 아니다.

저장/배포 기능 아님

Firestore 저장·API·fetch 호출이 없다. JSON 복사/localStorage 외 영속 저장은 후속 작업이다.

구현 지시가 아님

이 문서는 연결 설계 기준이다. 화면설계서 초안 자동 생성 로직 구현을 지시하지 않는다.

후속은 별도 분리

저장 방식·권한 적용·API·배포는 후속 제품 구현에서 별도 PR로 분리해 처리한다.

10

Integration Rules

실제 repo 경로 기준연결 대상은 실제 존재하는 repo 경로로만 기술한다. 가상 경로/예상 파일명을 확정 경로처럼 쓰지 않는다.
nav-data SSOT 기준Route/Menu 참조는 stam/js/stam.nav-data.js SSOT를 따른다. 이 문서는 nav-data를 수정하지 않는다.
임의 경로 생성 금지후속 문서/제품 route를 임의로 만들지 않는다. 생성 후 연결한다.
실제 존재 파일만 링크모든 <a href>는 실재 파일만 가리킨다. Missing 문서는 텍스트로만 표기.
docs-only 유지제품 파일 수정 없이 docs 영역에서만 작업한다.
화면설계서 제품 보정은 별도 PR제품 페이지 보정/구현은 이 문서 범위가 아니며 별도 PR에서 처리한다.
11

Mobile / Narrow Width Rules

목록

핵심 컬럼(상태·제목·날짜) 우선 노출, 부가 컬럼은 전체보기/카드 전환. visibleInTable 다수 필드는 모바일 우선순위 지정 필요.

상세 / 폼

섹션 스택(1열). 긴 폼은 하단 액션바 고정. visibleInDrawer 필드 순서대로 세로 배치.

설정 화면

설정 섹션은 카드/아코디언 전환 고려. 권한 매트릭스는 가로 스크롤 정책 명시.

Wide Data Table (T05)

Focus View 또는 명확한 가로 스크롤 정책 필수. 핵심 컬럼 좌측 고정.

Drawer / Modal

좁은 화면에서 전체 화면 전환 가능성 고려(화면 80% 이상 덮을 때 Full Screen).

chip / 배지

status/priority/select chip은 모바일에서 줄바꿈/요약(n+) 처리. 배지 우선 노출.
