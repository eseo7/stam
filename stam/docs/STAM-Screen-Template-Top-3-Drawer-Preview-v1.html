이 문서의 의미

문서 목적

P3 Top 10 Detail Pack 중 드로워 UX에 직접 연결되는 3개를 실제 화면설계서 Preview로 구체화한다. 상세·등록/수정·설정 드로워의 구조 차이를 명확히 한다.

Top 3 Drawer Preview의 의미

STAM은 목록 이동 없이 우측 드로워로 상세/등록/수정/설정을 처리하는 패턴이 핵심이다. 가장 자주 쓰이는 3개 드로워를 우선 적용 기준으로 잡는다.

P1~P5와의 관계

P3가 정의한 T07·T11·T27을 드로워 표현으로 확장하고, P4(필드 매핑)·P5(선택→미리보기→초안)와 맞물린다. 새 기준을 만들지 않고 인용·구체화한다.

이 문서의 성격

실제 Drawer 기능 · Board Builder 기능 · 화면설계서 제품 기능을 구현하지 않는다. wireframe은 흐름 이해용 pseudo-screen이며 제품 코드가 아니다. merge/deploy는 사용자 승인 전 대기한다.

용어 기준: 단계 범위는 "1차 범위 / 우선 적용 / 후속 단계" 표현을 사용한다. 기존 드로워 계약 문서(Drawer Guide · Drawer Shell Contract · Detail/Form Audit · Form Field Contract)와 정합하도록 작성한다.
01

Drawer Pattern Principle

목록 유지 + 우측 드로워

목록 화면을 유지한 채 우측 드로워로 상세/등록/수정/설정을 처리한다. 페이지 전환 없이 작업해 현재 위치(스크롤·필터·페이지)를 잃지 않게 한다.

영역 구분

드로워는 헤더 / 본문 / 하단 액션 / 보조 정보 영역으로 구분한다. 헤더·하단 액션은 고정, 본문은 스크롤.

읽기 vs 편집 모드

상세는 읽기 모드, 등록/수정은 편집 모드. 같은 드로워 틀을 쓰되 본문 컨트롤(텍스트 표시 ↔ 입력 컨트롤)과 하단 액션이 달라진다.

상세 → 수정 전환

읽기 모드에서 "수정"을 누르면 같은 드로워가 편집 모드로 전환된다. 권한이 없으면 수정 액션을 노출하지 않는다.

등록 완료 후 목록 반영

저장 성공 시 드로워를 닫고 목록에 신규/변경 행을 반영(하이라이트). 실패 시 드로워 유지 + 오류 표시.

닫기 / 취소 / 저장 기준

ESC·backdrop·닫기 버튼으로 닫기. 변경사항(dirty)이 있으면 변경사항 확인 다이얼로그로 유실 방지. 저장은 명시적 버튼.

모바일/좁은 화면에서는 드로워가 전체 화면 전환 가능(섹션 10). 공통 셸 규약은 Drawer Shell Contract를 따른다.

Preview 01 — T07 Basic Detail View Drawer

상세 드로워

적용 화면 예시: 요구사항 상세 · 화면설계서 상세 · 정책 상세 · 일반 산출물 상세. 드로워 목적: 목록을 유지한 채 한 항목의 상세 정보를 읽기 모드로 빠르게 확인하고, 권한이 있으면 수정/검토 요청으로 전환한다.

┌─ Drawer · T07 Detail ───────────────── [X] ┐   ← Header
│ 요구사항 #REQ-014  로그인 2단계 인증        │
│ [검토중]  작성자 김·2026-06-18 · 최종 06-19 │   (Status · Meta)
│ ───────────────────────────────────────── │
│ [개요] [상세] [연결] [이력] [댓글]          │   ← Tabs
│ ───────────────────────────────────────── │
│ 우선순위 : 높음                              │   ← Body / Field
│ 담당자   : 박검토                            │
│ 상태     : 검토중                            │
│ 설명     : 로그인 시 2단계 인증을 …          │
│ 첨부     : 와이어프레임.png (240KB)          │   (Attachment)
│ 연결산출물: 화면설계서 #SS-007              │   (Meta / Relation)
│ ───────────────────────────────────────── │
│ [닫기] [내보내기]      [검토 요청] [수정 ✎] │   ← Footer Actions
└─────────────────────────────────────────────┘

헤더 / 메타

  • Title: 산출물 코드 + 제목
  • Status Badge: 상태 chip
  • Meta: 작성자·작성일·최종수정

본문 섹션

  • 개요/상세: 필드 라벨+값(읽기)
  • 연결: relation 산출물 링크
  • 섹션 많으면 탭 전환

첨부 / 이력 / 댓글

  • 첨부: 파일명·형식·크기(다운로드 권한)
  • 이력: 변경자·변경일·항목
  • 댓글/검토: 의견·멘션

기본 액션

  • 목록 유지(닫아도 위치 유지)
  • 수정 · 검토 요청 · 내보내기 · 닫기

권한 포인트

  • 보기 / 수정 / 삭제
  • 검토 요청 / 내보내기
  • 권한 없는 액션 미노출

상태 배지

초안검토중승인반려보류

빈 상태

본문/연결/이력 없음 시 "내용이 없습니다" 안내. 탭별 빈 상태 분리.

오류 상태

로드 실패(404/403/네트워크) 시 본문 영역 오류 + 재시도. 권한 부족은 권한 안내.

모바일 기준

전체 화면 전환. 탭은 가로 스크롤/세그먼트. 헤더·하단 액션 고정.

T07 QA 체크: 읽기 모드 기본 · 수정 권한 있을 때만 수정 액션 노출 · 탭별 빈 상태 · 닫기 시 목록 위치 유지 · 모바일 전체화면. 상세 필드 기준은 Top 10 Detail Pack T07을 따른다.

Preview 02 — T11 Basic Create Form Drawer

등록 / 수정 드로워

적용 화면 예시: 요구사항 등록 · 정책 등록 · 일반 게시글 등록 · 산출물 등록. 드로워 목적: 목록을 유지한 채 새 항목을 등록하거나 기존 항목을 수정한다. 필수/선택 입력을 구분하고 저장 전 검증한다.

┌─ Drawer · T11 Create/Edit ──────────── [X] ┐   ← Header
│ 새 요구사항 등록                            │
│ ───────────────────────────────────────── │
│ * 필수 입력                                  │   ← Body / Field
│ 제목*    [___________________________]      │
│ 유형*    [▼ 선택 ]                          │
│ 우선순위* [▼ 높음 ]                          │
│ 담당자   [▼ 사용자 검색 ]                    │   (선택 입력)
│ 설명     [__________________________]       │
│          [__________________________]       │
│ 첨부     [＋ 파일 추가]                      │   (Attachment)
│ ───────────────────────────────────────── │
│ ⚠ 제목은 필수입니다.                        │   (Validation)
│ [취소] [초기화] [미리보기]      [저장]      │   ← Footer Actions
└─────────────────────────────────────────────┘

헤더

  • "새 OOO 등록" / "OOO 수정"
  • 수정 시 대상 코드 표시

필수 입력 영역

  • required 필드 상단 그룹 + *
  • 제목·유형·상태 등 핵심값

선택 입력 영역

  • 담당자·설명·우선순위 등
  • 접힘 가능(고급)

첨부 / 임시저장

  • 첨부: 파일 추가(형식·크기)
  • 임시저장/미리보기 여부 표시(우선 적용 범위 명시)

기본 액션

  • 저장 · 취소 · 초기화 · 미리보기
  • 저장 시 검증 통과 필요

권한 포인트

  • 등록 가능 여부 · 수정 가능 여부
  • 필드별 수정 권한(읽기전용 필드)

유효성 검증

  • 필수값 · 형식 · 길이 · 중복
  • 권한(편집 불가 필드)

저장 전/후

  • 저장 전: 검증 요약 + 확인
  • 저장 후: 드로워 닫고 목록 반영(하이라이트)

빈 / 오류 / 모바일

  • 빈: 초기 입력 안내
  • 오류: 필드 하단 + 상단 요약
  • 모바일: 전체화면, 하단 저장 고정
T11 QA 체크: 필수 표시(*) · 저장 전 검증 차단 · dirty 상태 닫기 확인 · 저장 후 목록 반영 · 필드별 수정 권한 · 모바일 하단 저장 고정. 등록 폼 필드는 Board Builder 정의(Integration Blueprint)를 입력으로 사용한다.

Preview 03 — T27 Custom Field Settings Drawer

설정 드로워

적용 화면 예시: Board Builder / 게시판 설정 / 필드 타입 / 옵션 / 필수 여부 / 정렬순서. 드로워 목적: 게시판의 커스텀 필드 하나를 추가/수정한다. 필드 타입·표시 토글·옵션·정렬을 설정하고 우측 미리보기로 확인한다. Board Builder와 직접 연결된다.

Board Builder 관계

이 드로워는 Board Builder 필드 정의를 직접 편집한다. 12종 필드 타입과 18속성(표시·필수·옵션·정렬 등)을 다룬다. preview/local 기준(실제 저장/배포 아님).

필드 타입 선택

texttextareaselectmultiSelectdateuserrelationstatusprioritybooleannumberurl

타입 선택 시 옵션/검증/표시 기본값 자동 조정.

식별 / 문구

  • 필드명* · 설명 · placeholder
  • 필드명 중복 불가

표시 토글

  • 목록 표시(visibleInTable)
  • 상세 표시 / 등록·수정 표시(visibleInDrawer)
  • 검색 가능(visibleInFilter)
  • 내보내기 포함(exportable)

옵션 / 정렬 / 권한

  • 옵션값(select/status 등)
  • 정렬순서(number)
  • 권한 힌트(permission)

미리보기 영역

설정한 필드가 목록 컬럼/입력 컨트롤로 어떻게 보이는지 즉시 미리보기(chip tone 포함).

기본 액션

  • 필드 추가 · 저장 · 삭제 · 복제 · 취소
  • 복제는 옵션/표시 토글 복사

오류 상태

  • 중복 필드명
  • 옵션 필요한 타입에 옵션 없음
  • 필수 설정 누락(타입/필드명)

모바일 기준

전체 화면 전환. 표시 토글은 세로 스택. 옵션 목록·미리보기는 컨테이너 overflow 처리.

T27 QA 체크: 타입 선택 시 기본값 조정 · 필드명 중복 차단 · 옵션 필요 타입 옵션 검증 · 표시 토글 4종 · 미리보기 동반 · 모바일 토글 스택. 필드 타입/속성 표준은 실제 stam/js/stam.board-field-schema.js(12종·18속성, 읽기 전용)와 정합한다.

Validation / Error Pattern

필드 하단 메시지

해당 필드 바로 아래 인라인 메시지(빨강). 포커스 시 우선 표시.

상단 오류 요약

저장 시도 시 드로워 상단/하단 액션 위에 오류 요약(개수+첫 항목 이동).

저장 차단

필수값·형식·중복 위반 시 저장 차단. 위반 필드로 스크롤.

저장 가능 + 경고

권장 위반(긴 길이 등)은 경고만 표시, 저장은 허용(확인 1회).

권한 오류

편집 불가 필드/액션은 읽기전용 + 사유 안내. 저장 시 권한 오류 메시지.

네트워크 / 저장 오류

저장 실패 시 드로워 유지 + 재시도. 입력값 보존(유실 방지).

변경사항 유실 방지

dirty 상태에서 닫기/ESC/backdrop 시 확인 다이얼로그. 저장/계속/취소 선택.

중복 / 형식 오류

중복(필드명·코드), 형식(date/url/number) 위반 인라인 표시.

모바일 오류 표시: 좁은 화면에서는 상단 요약을 접고 위반 필드로 이동 + 하단 고정 저장 영역 위에 한 줄 요약. 폼 필드 검증 기준은 Form Field Contract를 따른다.

Mobile / Narrow Width Rules

전체 화면 전환

좁은 화면에서 드로워는 전체 화면으로 전환(화면 80% 이상 덮을 때).

Header / Footer 고정

헤더(제목·닫기)와 하단 액션(저장/취소) 고정. 본문만 스크롤.

Body 1열 스택

필드/섹션은 1열 세로 스택. 2열 그리드 금지.

Tabs 전환

탭은 세그먼트 컨트롤 또는 아코디언으로 전환(가로 넘침 방지).

Footer CTA 우선순위

하단 고정. 저장(Primary) 우선, 취소/닫기 보조, 삭제는 분리(오발 방지).

Wide / 옵션 목록

옵션 목록·미리보기·표는 컨테이너 overflow:auto 처리. 과도한 가로 스크롤 금지.

이 문서 자체도 동일 정책을 따른다: 표·wireframe은 컨테이너 overflow:auto로 처리하고 980px 이하에서 그리드를 1열로 전환한다. 공통 모바일 셸 기준은 Mobile Shell Guide를 참조한다.
