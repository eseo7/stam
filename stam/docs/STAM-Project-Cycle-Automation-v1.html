01 Purpose — 이 문서의 목적 Strategy

이 문서는 STAM Works가 무엇을 만드는 서비스인지에 대한 제품 중심축을 한 문장으로 고정한다: STAM Works는 고객 상담·엑셀 산출물·요구사항을 시작점으로, 기능정의 · 메뉴/화면 · WBS · 화면설계서 · 테스트 시나리오 · 추적성 매트릭스까지 연결 생성·관리·추적하는 한 사이클 자동화 서비스다.

  • 향후 로드맵 · 화면 설계 · 구현 순서를 결정할 때 기준이 되는 상위 문서다.
  • 기존 산출물 화면을 폐기하지 않고 Project Cycle Automation이라는 상위 개념 아래에 재배치한다.
  • Excel Import Builder / 상담 Intake / 요구사항 / 기능정의 / 메뉴·화면 / WBS / 화면설계서 / 테스트 시나리오 / 추적성 매트릭스의 관계를 명확히 한다.
  • 자동화의 의미를 “최종 확정”이 아니라 “초안 생성 + 연결 추천 + 누락 감지 + 변경 영향 분석 + 사람 승인”으로 정의한다.
핵심 결론 8가지 ① STAM Works는 단순 산출물 게시판 모음이 아니다. ② 상담·엑셀·요구사항을 시작점으로 기능정의·메뉴/화면·WBS·화면설계서·테스트 시나리오까지 연결 생성·관리한다. ③ 기존 요구사항/WBS/화면설계서 화면은 폐기 대상이 아니라 하위 실행 화면이다. ④ 자동화는 “초안 생성 + 연결 추천 + 누락 감지 + 변경 영향 분석 + 사람 승인”이다. ⑤ Excel Import Builder는 부가기능이 아니라 주요 진입 기능이다. ⑥ 공통화는 컬럼 고정이 아니다. 구조·동작은 공통화, 필드/컬럼은 표준 필드 + customFields 로 확장한다. ⑦ API 명세·배치잡·테이블정의·DB·서버·접근보안은 기능정의에서 파생되는 후속 산출물이다. ⑧ 1차 핵심 범위는 요구사항 → 기능정의 → 메뉴/화면 → WBS → 화면설계서 → 테스트 시나리오 → 추적성 매트릭스다.
02 Product Reframing — 제품 중심축 재정의 Reframe
기존 관점 — 산출물 게시판 모음
  • 요구사항 게시판
  • WBS 게시판
  • 화면설계서 게시판
  • 각 게시판이 서로 분리된 독립 목록으로만 존재
새 관점 — 산출물 한 사이클 자동화
  • 고객 상담 / 엑셀 / 요구사항에서 시작
  • 산출물 한 사이클을 생성·연결·추적하는 자동화 서비스
  • 각 산출물이 앞뒤 산출물과 연결(linked)되어 추적 가능
  • 변경 시 영향 범위를 사이클 전체에서 확인
기존 작업은 버리지 않는다 지금까지 만든 요구사항 / WBS / 화면설계서 / Component Guide / Product Screen Implementation Gate 는 폐기 대상이 아니다. 이들은 새 상위 개념인 Project Cycle Automation 아래의 하위 실행 화면·실행 기준으로 재배치된다. 상위 사이클 개념만 새로 추가하고, 하위 화면은 그대로 활용한다.
03 Project Cycle Overview — 한 사이클 흐름 Flow

STAM Project Cycle 은 다음 흐름을 기준으로 한다. 각 단계는 앞 단계의 출력을 입력으로 받아 다음 산출물의 초안을 생성·연결한다.

Intake / 상담·견적→ Excel / Artifact Import→ Requirement / 요구사항→ Feature Definition / 기능정의→ Menu & Screen / 메뉴·화면
WBS / 작업계획→ Screen Specification / 화면설계서→ Test Scenario / 테스트 시나리오→ Traceability Matrix / 추적성 매트릭스→ Export / 내보내기
사이클의 의미 이 흐름은 강제 단방향 순서가 아니라 연결의 기준 축이다. 어느 단계에서 시작하든(예: 엑셀 import 또는 상담 intake) 앞뒤 산출물과 연결될 수 있어야 하며, 모든 단계의 결과는 초안(Draft)으로 생성되고 사람이 검토·승인한다.
04 Cycle Step Definition — 단계별 정의 Definition

각 단계는 입력 · 처리 · 출력 · 연결 대상 · 사람이 검토해야 할 항목 · 후속 생성 가능 산출물로 정의한다.

단계입력처리출력연결 대상사람 검토 항목후속 생성 가능
Intake / 상담·견적 상담 메모, 고객 요청, 견적 항목 요청 정리, 범위 후보 추출 상담 기록 초안, 요구 후보 요구사항 범위 적정성, 우선순위 요구사항 초안
Excel / Artifact Import 엑셀/기존 산출물 파일 헤더 감지, 컬럼 분석, 표준 필드 매핑 Draft Import, 매핑 후보 요구사항, 기능정의 매핑 확정, custom field 채택 요구사항 초안
Requirement / 요구사항 상담/엑셀 결과, 직접 입력 요구 항목 정규화, 분류 요구사항 항목 기능정의, 화면, WBS, 화면설계서, 테스트 중복/누락, 분류·우선순위 기능정의 초안
Feature Definition / 기능정의 요구사항 항목 기능 단위 분해, 그룹화 기능 항목 요구사항, 메뉴/화면, 테스트 기능 분해 적정성 메뉴/화면 후보, API·DB 후속 산출물
Menu & Screen / 메뉴·화면 기능 항목 화면 구조 변환, 메뉴 배치 메뉴/화면 목록 기능정의, WBS, 화면설계서 화면 단위 적정성, IA 정합 WBS 초안, 화면설계서 템플릿
WBS / 작업계획 기능/화면 단위 작업 분해, 일정·담당 후보 WBS 작업 항목 기능정의, 메뉴/화면 일정·담당·공수 확정 일정/리소스 후속 산출물
Screen Specification / 화면설계서 메뉴/화면 + 템플릿 템플릿 기반 화면 구체화 화면설계서 항목 메뉴/화면, 기능정의, 테스트 필드·액션·권한 확정 테스트 시나리오 초안
Test Scenario / 테스트 시나리오 화면설계서, 기능정의 시나리오·케이스 초안 생성 테스트 시나리오 항목 화면설계서, 기능정의, 요구사항 케이스 충분성, 예외 흐름 추적성 매트릭스 행
Traceability Matrix / 추적성 전 단계 산출물 연결 연결 집계, 누락 감지 추적성 매트릭스 요구사항 ~ 테스트 전체 누락 연결, 영향 범위 내보내기 산출물
Export / 내보내기 확정 산출물 템플릿 기반 출력 구성 내보내기 문서 전 단계 출력 범위·형식 고객/내부 제출본
05 Excel / Artifact Import Builder — 주요 진입 기능 Entry
Excel Import Builder 는 부가기능이 아니다 엑셀 산출물은 SI 현장에서 가장 흔한 시작점이다. Import Builder 는 STAM 의 주요 진입 기능으로 다룬다.

Import Builder 의 흐름:

파일 업로드→ 시트 선택→ 헤더 행 감지→ 컬럼 자동 분석→ 표준 필드 매핑
custom field 생성 후보→ 데이터 타입 추론→ 상태값/옵션값 추출→ 중복/누락/오류 검사
미리보기→ import 확정→ 요구사항 초안 생성
자동 생성 결과 = Draft Import import 결과는 확정 데이터가 아니라 Draft Import 로 취급한다. 사용자가 매핑을 수정하고 직접 확정해야 실제 산출물 항목이 된다.
원본 추적 메타 (필수)
  • sourceFile — 원본 파일
  • sourceSheet — 원본 시트
  • sourceRow — 원본 행
  • importBatchId — import 배치 식별자
  • importedAt — import 시각
  • importedBy — import 수행자
06 Standard Fields and Custom Fields — 공통화와 확장 Field Model
공통화 ≠ 컬럼 고정 산출물별 표준 필드는 유지하되, 고객사/프로젝트별 컬럼은 customFields 로 흡수한다. 공통화와 커스텀 컬럼은 충돌하지 않는다.
공통화 대상 (구조·동작)
  • 화면 구조
  • 필터 / 검색 / 정렬
  • Drawer
  • 변경이력
  • 권한
  • 내보내기
확장 가능 대상
  • 필드
  • 컬럼
  • 옵션값
  • 고객사/프로젝트별 customFields
예시 필드 (요구사항)
  • requirementId
  • externalRequirementId
  • title
  • description
  • category
  • priority
  • status
  • requester
  • owner
  • linkedFeatures
  • linkedScreens
  • linkedWbs
  • customFields
예시 FieldDefinition
  • fieldKey
  • label
  • type
  • required

  • visibleInList

  • sortable
  • exportable
  • order
07 Draft Generation and Human Approval — 사람 승인 전제 Human-in-loop
STAM 은 최종 산출물을 자동 확정하지 않는다 자동화는 초안 생성 · 연결 추천 · 누락 감지 · 변경 영향 분석까지만 담당한다. 확정은 PM / 기획자 / 담당자의 검토·승인을 거친다.
자동화가 하는 일
  • 초안(Draft) 생성
  • 연결 추천 (linked 후보)
  • 누락 감지
  • 변경 영향 분석
사람이 하는 일
  • 내용 검토
  • 매핑·연결 확정
  • 우선순위 결정
  • 승인

상태 예시:

Imported Draft→ Review Needed→ Confirmed→ Linked→ Export Ready
08 Traceability Matrix — 핵심 차별화 화면 Differentiator

요구사항 · 기능정의 · 메뉴/화면 · WBS · 화면설계서 · 테스트 시나리오 사이의 연결을 한 표로 보여준다.

요구사항기능메뉴/화면WBS화면설계서테스트
REQ-001 로그인F-001 인증S-001 로그인 화면W-001 인증 개발SS-001 로그인 설계T-001 로그인 테스트
REQ-002 권한관리F-002 권한S-002 권한 설정W-002 권한 개발SS-002 권한 설계T-002 권한 테스트
REQ-003 게시판F-003 목록·등록S-003 게시판 목록W-003 게시판 개발SS-003 목록 설계(누락 감지)
이 화면의 역할 추적성 매트릭스는 STAM 의 핵심 차별화 화면이다. 누락된 연결을 감지하고, 산출물 변경 시 영향 범위를 보여주며, 고객사/내부 검토에서 기준 화면으로 사용할 수 있다.

기존 화면은 하위 실행 화면이다 기존 제품 화면은 폐기되지 않고 Project Cycle Automation 의 각 단계 실행 화면으로 재배치된다.

WBS기능/화면 단위 작업계획 생성 대상

12 보류할 항목 — Out of Scope (현재) Hold
현재 단계에서 보류 아래 항목은 이번 기준 단계에서 구현·확정하지 않는다. 이 문서는 기준 문서이며 실제 구현 작업이 아니다.
  • 실제 Excel parsing 구현
  • AI 자동 분해 로직 구현
  • DB schema 확정
  • Firebase backend 도입
  • Storage 업로드 구현
  • HWP 내보내기
  • 온프레미스 / 전용 tenant
  • 공공 / 금융 보안 대응
13 Implementation Principles — 구현 원칙 Principles
  • 기존 화면을 갈아엎지 않는다.
  • 기존 Component Guide / Product Screen Implementation Gate 를 유지한다.
  • 새 기능은 기존 화면 위에 Project Cycle Layer 로 얹는다.
  • 제품 화면 변경 시 반드시 Gate 를 적용한다.
