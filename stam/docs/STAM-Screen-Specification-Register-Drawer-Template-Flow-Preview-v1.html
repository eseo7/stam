01Executive Summary — 작성 흐름 개요기준
  • Register Drawer의 실제 작성 흐름을 8단계로 구체화한다.
  • 사용자는 빈 화면을 처음부터 그리는 것이 아니라, 템플릿을 선택해 초안을 생성한다.
  • 화면명, 목적, 메뉴 경로, 필드, 액션, 문구, 연결 화면, 첨부 참고자료를 직접 수정한다.
  • 템플릿의 핵심 구조는 유지하고, 실무 화면에 맞는 항목만 조정한다.
  • 저장, 임시저장, 검토요청 흐름까지 포함한다. 첨부파일은 참고자료로만 사용한다.
관련 문서: Top 3 Drawer Preview v1 — 검색목록·상세Drawer·등록Drawer 핵심 3종 정적 시안. · Top 10 Detail Pack v1 — 초기 적용 범위 10종 상세 정의 포함. · Visual Annotation Template Preview v1 — 좌측 UI 목업 + 번호 마커 + 우측 Description 패널 구조 포함.

15Output Preview — 생성되는 화면설계서 초안생성 결과
생성 결과: 저장이 완료되면 화면설계서 초안이 생성됩니다. 생성된 초안은 상세 Drawer에서 확인할 수 있으며, 검토 요청 전까지 작성자가 수정할 수 있습니다.
미리보기 연결: 미리보기는 저장 전 현재 입력값을 기반으로 생성된다. 미리보기 결과는 번호 주석형 화면설계서 형태(Visual Annotation 구조)로 표시된다. 사용자는 미리보기에서 번호 마커와 Description 패널을 확인한 뒤 다시 Drawer로 돌아와 수정할 수 있다. 미리보기는 실제 저장 데이터가 아니라 임시 렌더링 결과다. 저장 후에는 Detail Drawer에서 동일한 결과를 조회할 수 있다.

16Accessibility Notes — KWCAG 2.2 기준 메모10 항목
Drawer 열림 시 첫 입력 항목(화면명)으로 초점 이동
Drawer 닫힘 시 「템플릿으로 작성」 버튼으로 초점 복귀
하단 고정 액션바 Tab 순서 마지막 · focus-visible 스타일 유지

17Cursor Implementation Notes — 구현 기준 메모후속 작업 참고
중요: 이 문서는 정적 Preview다. 실제 구현은 Cursor 복귀 후 별도 PR로 분리한다.

구현 원칙

Register Drawer UI와 저장 로직을 분리한다.
Register Drawer와 Detail Drawer는 공통 Drawer shell을 공유할 수 있다.
필드 구성 편집은 별도 컴포넌트로 분리 가능하다.
첨부파일은 참고자료 데이터로 관리하며 화면 구조 데이터와 분리한다.

후속 연결 항목

검토요청은 권한/상태 전환 로직(T09)과 연결한다.
임시저장/저장/검토요청 상태값을 명확히 분리한다.
변경이력은 저장 후 별도 history model과 연결한다.
각 섹션 편집 상태는 로컬 상태로 관리하고 저장 시 일괄 커밋한다.
Register Drawer에는 「미리보기」 액션이 필요하다. 저장 전 입력값을 기준으로 Preview model을 생성한다.
Preview model은 Visual Annotation Template 구조와 연결된다 (번호 마커 + Description 패널).
미리보기 구현은 별도 읽기 전용 Preview Drawer, Modal, 우측 Preview Panel 중 선택 가능. 1차 구현은 별도 Preview Drawer 권장.
저장 전 미리보기와 저장 후 Detail Drawer 결과는 같은 렌더링 구조를 공유해야 한다.
검토요청 전 사용자가 미리보기를 통해 누락 항목을 확인할 수 있어야 한다.
Register Drawer 항목 사용/숨김/삭제/복원 제어 기준 → Template Item Usage Control Preview v1
저장된 화면설계서 상세 결과 조회 기준 → Detail Drawer Result Preview v1
