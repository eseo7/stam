MOBILE BASELINE · v1 · PR #211

STAM 모바일 핵심 업무 흐름 1차 기준

편집-heavy 화면을 모바일에서 완성하려는 시안이 아닙니다. 모바일에서 실무자가 가장 자주 확인하는 업무 요약 · 검토 · 승인 · 알림 · 변경이력 확인 흐름을 우선 적용 기준으로 정리한 정적 Preview 입니다. WBS 전체 테이블이나 화면설계서 편집기 같은 영역은 모바일에 그대로 옮기지 않고, 요약 카드 + 검토 CTA + 데스크톱 연결 링크 형태로 제공합니다.

모바일 기준선 원칙

모바일에서 PC 화면을 축소하지 않습니다.
  • 지금 무엇을 봐야 하는지가 첫 화면에서 즉시 보여야 합니다. 카드 3~5개로 요약하고 숫자/상태 중심으로 표시합니다.
  • 편집보다 확인이 우선입니다. CTA 는 검토하기 / 승인 보기 / 댓글 확인 / 변경 확인 같은 동사로 구성합니다.
  • WBS 전체 테이블 / 화면설계서 편집기 같은 영역은 모바일에서 완전 편집하지 않고 전체 보기 → 데스크톱 CTA 로 분기합니다.
  • 업무 우선순위는 위에서 아래로 자연스럽게 떨어져야 합니다 — 오늘 마감 · 지연 · 검토 요청 · 승인 대기 · 댓글 멘션 순.
  • 한국형 업무 문구를 사용합니다 — 검토 요청 / 승인 대기 / 오늘 마감 / 댓글 멘션 / 변경 확인 필요 / 지연 작업 / 전체 보기 등.
  • STAM 토큰과 S-Core Dream 폰트만 사용합니다. 외부 CDN / 신규 JS / 신규 메뉴 / 신규 데이터 모델 없음.

Bottom Action / Sticky 영역 기준

4 탭 + 선택적 floating primary CTA — Preview 시안만

반응형 QA 기준

390 / 430 / 768 / 1366 + Light / Dark
  • 390 / 430 — 모바일 첫 화면 카드 4~5개가 한 화면에 보임. 카드 grid 1열로 자연 wrap. 텍스트 잘림 없음.
  • 768 — 디바이스 프레임 + 스펙 2열 유지. 폭이 좁아져도 카드 grid 2열 유지.
  • 1366 — 좌측 디바이스(스티키 240px) + 우측 스펙. content max 1320px, 좌우 24px padding.
  • Light / Dark — `var(--bg-base / --bg-sur / --bg-sur2 / --bg-sur3 / --t1 / --t2 / --t3 / --bd / --stam / --stam-soft)` 토큰만 사용. 다크모드 추가 polish 일부 보강.
  • console — JS 없음. CSS import 만 4개 (S-Core Dream / tokens / components / buttons / mobile-core-workflow). console red error 발생 가능성 없음.

보존해야 할 contract

후속 PR 에서 본 시안을 실제 화면으로 옮길 때 지켜야 할 기준
구성 contract. 프로젝트 요약 → My Action Queue → WBS 요약 → 화면설계서 검토 → 알림 / 변경이력 → Bottom action 순서를 모바일 첫 화면 우선 순위로 유지합니다. WBS / 화면설계서 의 모바일 표현은 요약 카드 + 핵심 리스트 + 데스크톱 연결 링크 패턴만 유지합니다.
문구 contract. 검토 요청 / 승인 대기 / 오늘 마감 / 댓글 멘션 / 변경 확인 필요 / 지연 작업 / 전체 보기 / 검토하기 / 승인 보기 / 댓글 확인 / 변경 확인 / 확인하기 — 한국형 업무 동사 톤 유지. “MVP” 표현은 사용하지 않음. “1차 기준 / 우선 적용 / 모바일 핵심 업무 흐름” 표현 유지.
기술 contract. 외부 CDN / 신규 JS / 신규 fetch / Firebase / Firestore / localStorage / sessionStorage / indexedDB / XMLHttpRequest 신규 추가 없음. WBS / 화면설계서 closeout 파일 (wbs.html · stam.wbs.css · screen-specification.html · stam.screen-specification.css) 무변경. App Shell / Topbar / Left Nav / nav-data / shell.js / nav-render / topbar-render / project-context-render 무변경. 신규 메뉴 / 화면 경로 / 데이터 모델 미추가.
WBS / 화면설계서 연결. 본 Preview 안의 데스크톱 진입 링크는 실제 production 경로 ../pages/boards/wbs.html 과 ../pages/boards/screen-specification.html 를 가리킵니다. docs 위치 기준 상대 경로이며, 새 화면 경로를 추가하지 않습니다.
