Section 08 · 후속 PR 분리안
  • PR A — 삭제 버튼 공통화: 화면 5종의 .{ns}-btn-del → .stam-delete-btn + .stam-delete-label. WBS 의 실삭제 동작은 보존 또는 별도 보고 후 처리.
  • PR B — 검색/필터 toolbar 공통화: .stam-list-toolbar · .stam-list-search · .stam-filter-btn 일관 적용. 필터 팝업 (.stam-board-filter-panel vs .ss-filter-pop) 통합은 PR B' 로 분리.
  • PR C — WBS 목록 테이블 class mapping: .wbs-table → .stam-select-table, .wbs-chk → .stam-check, row state → .is-active / .is-selected. 위험도 高 — 단독 PR 권장.
  • PR D — 화면설계서 목록 테이블 class mapping: .ss-table/.ss-cb 일원화 + drawer row 에 .is-active.
  • PR E — 메뉴/화면목록 class mapping: .msl-cb→.stam-check, .sel→.is-selected, drawer row 에 .is-active.
  • PR F — 요구사항/결함 목록 class mapping: .rq-cb→.stam-check, .sel→.is-selected, drawer row 에 .is-active. (결함 목록 파일은 현 시점 미존재 — 추가 시 동일 패턴 적용)
  • PR G(옵션) — 기능정의서 .is-active 보강: 가장 작은 단위, smoke test 용도 선행 가능.
