STAM Board Filter Guide

공통 필터 패널 컴포넌트 — stam.board-filter.css / stam.board-filter.js

v1.1 2026-06-12

1. Board Filter 목적

보드 목록 화면의 필터 기능을 공통 컴포넌트로 분리합니다. 각 화면은 필터 그룹 데이터(groups)만 전달하면 되며, 열기/닫기·chip active·초기화·적용·ESC/외부 클릭 닫기는 공통 JS가 처리합니다.

설계 기준: WBS 필터 패널 — stam.board-filter.css/js의 전체 스타일·구조·동작은 stam/pages/boards/wbs.html의 WBS 필터 패널(.wbs-filter-panel, stam.wbs.css)을 기준으로 맞춥니다. 신규 Board Filter를 추가하거나 기존 스타일을 수정할 때는 WBS 필터를 먼저 확인하세요.

핵심 원칙
Toolbar에 개별 필터 chip을 직접 노출하지 않습니다. 단일 필터 버튼 클릭으로 패널을 열고 그 안에서 조건을 선택합니다.

3. CSS 클래스 기준

WBS 필터 버튼(.wbs-filter-open-btn)을 기준으로 스타일을 통일합니다. stam-btn 클래스는 사용하지 않습니다 — stam-btn은 height:32px를 강제해 버튼 높이가 깨집니다.

클래스 역할 출처
.stam-board-filter-trigger 필터 열기 버튼 — WBS .wbs-filter-open-btn 기준 스타일 (padding:5px 12px, font-size:12px, border:1px solid var(--bd-s)) HTML에 작성
.stam-board-filter-trigger.active trigger 버튼 — 조건 1개 이상 선택 시 brand 강조 (background:var(--stam-soft), color:var(--stam), font-weight:700) JS 동적 추가
.stam-board-filter-trigger.is-open trigger 버튼 — 패널이 열려 있는 동안 active와 동일한 강조 표시 JS 동적 추가
.stam-board-filter-count 선택 조건 수 뱃지 (trigger 내부). visible 클래스가 없으면 display:none HTML에 작성
.stam-board-filter-count.visible 뱃지 표시 상태 — 조건 1개 이상 선택 시 JS가 추가 JS 동적 추가
.stam-board-filter-panel 필터 패널 컨테이너 (position:absolute, width:560px, WBS .wbs-filter-panel 기준 shadow) HTML에 작성
.stam-board-filter-panel.is-open 패널 열림 상태 (opacity/transform 전환) — hidden 제거 + rAF 후 클래스 추가 JS 동적 추가
.stam-board-filter-grid 그룹 섹션을 담는 스크롤 바디 (WBS .wbs-fp-body 기준) HTML에 작성
.stam-board-filter-actions 조건 수 정보 + 초기화/적용 버튼 푸터 (WBS .wbs-fp-foot 기준) HTML에 작성
.sbf-head 패널 헤더 (아이콘 + "필터" 타이틀 + 초기화 버튼) — WBS .wbs-fp-head 기준 JS 생성
.sbf-head-reset 헤더 우측 초기화 버튼 (WBS .wbs-fp-head-reset 기준, font-size:11px, padding:3px 9px) JS 생성
.sbf-foot-info 푸터 좌측 "조건 N개" 텍스트 (WBS .wbs-fp-foot-info 기준, flex:1). JS가 .stam-board-filter-actions 첫 번째 자식으로 삽입 JS 생성
.sbf-sec 필터 그룹 섹션 (WBS .wbs-fp-sec 기준) JS 생성
.sbf-sec-title 그룹 라벨 (UPPERCASE, 10px, WBS .wbs-fp-sec-title 기준) JS 생성
.sbf-chips chip 버튼 래퍼 (WBS .wbs-fp-chips 기준) JS 생성
.sbf-chip 필터 조건 chip 버튼 (pill, padding:5px 12px, border:1.5px solid var(--bd-s), WBS .wbs-fp-chip 기준) JS 생성
.sbf-chip.active 선택된 chip — background:var(--stam-soft), border-color:var(--stam), font-weight:700 JS 동적 추가

4. JS API — window.STAM.boardFilter.init(opts)

옵션 타입 설명
root Element | document 선택자 쿼리 기준 요소 (기본 document)
trigger string (선택자) 필터 버튼 요소
panel string (선택자) 필터 패널 요소 (hidden 속성 초기값)
reset string (선택자) 초기화 버튼 요소
apply string (선택자) 적용 버튼 요소
groups Array 필터 그룹 배열 — 아래 groups 형식 참조
onApply function(values) 적용 클릭 콜백. values: { [key]: string[] }
반환값 (공개 API)
var filter = window.STAM.boardFilter.init({...});
filter.open();       // 패널 열기
filter.close();      // 패널 닫기
filter.reset();      // 모든 chip active 해제
filter.getValues();  // 현재 선택값 반환 { [key]: string[] }
중복 init 방지

동일 trigger 요소에 data-sbf-init="1"이 설정된 경우 null을 반환하고 중복 등록하지 않습니다.

4-B. chip 토글 · aria-pressed · 초기화/적용 동작 규칙

WBS 필터와 동일한 동작을 보장합니다. stam.board-filter.js에서 공통 처리하므로 화면별 JS에서 중복 구현하지 않습니다.

chip on/off 토글
chip 클릭 시 classList.toggle('active')와 aria-pressed를 동시에 동기화합니다.
// JS 내부 동작 (stam.board-filter.js)
panelEl.addEventListener('click', function (e) {
  var chip = e.target.closest('.sbf-chip');
  if (!chip) return;
  var nowActive = !chip.classList.contains('active');
  chip.classList.toggle('active');
  chip.setAttribute('aria-pressed', nowActive ? 'true' : 'false');
  updateCount(); // 뱃지 + trigger.active + sbf-foot-info 갱신
});
aria-pressed 초기값
JS가 chip을 렌더링할 때 aria-pressed="false"를 기본으로 설정합니다.
chip.setAttribute('aria-pressed', 'false'); // 초기 비활성 상태
초기화 동작 — 헤더 초기화 버튼 / 푸터 초기화 버튼
헤더(.sbf-head-reset)와 푸터(opts.reset) 두 버튼 모두 동일한 doReset()을 실행합니다. 모든 chip의 active 제거 + aria-pressed=false + updateCount.
function doReset() {
  getActiveChips().forEach(function (c) {
    c.classList.remove('active');
    c.setAttribute('aria-pressed', 'false');
  });
  updateCount();
}
headReset.addEventListener('click', doReset);
if (resetEl) resetEl.addEventListener('click', doReset);
적용 동작
적용 버튼 클릭 시 현재 선택 상태를 유지한 채 패널을 닫고 onApply(values)를 호출합니다. chip 상태는 초기화하지 않습니다.
if (applyEl) {
  applyEl.addEventListener('click', function () {
    var values = getValues();
    closePanel();
    if (typeof opts.onApply === 'function') opts.onApply(values);
  });
}

// values 예시: { status: ['작성중', '검토요청'], priority: ['높음'] }
패널 열기 — double rAF 애니메이션
hidden 제거 후 즉시 is-open을 추가하면 CSS transition이 동작하지 않습니다. 두 번의 requestAnimationFrame으로 레이아웃 계산을 보장한 뒤 클래스를 추가합니다.
function openPanel() {
  panelEl.hidden = false;
  requestAnimationFrame(function () {
    requestAnimationFrame(function () {
      panelEl.classList.add('is-open');
    });
  });
  triggerEl.setAttribute('aria-expanded', 'true');
  triggerEl.classList.add('is-open');
}

10. 금지 패턴

Toolbar에 세부 필터 chip 직접 노출 금지
상태·유형·우선순위 등 개별 chip을 Toolbar에 직접 배치하지 않습니다. 단일 필터 버튼으로 통합합니다.
화면별 필터 패널 HTML 복붙 금지
각 화면이 그룹 chip을 HTML에 직접 쓰면 유지보수가 어렵습니다. groups 데이터만 JS에 전달하세요.
화면별 필터 토글 JS 중복 구현 금지
열기/닫기·ESC·외부 클릭 등의 동작은 stam.board-filter.js 에서 공통 처리합니다. 화면 JS에 중복 구현하지 않습니다.
실제 필터링 로직과 UI 렌더링 결합 금지
stam.board-filter.js는 UI 상태만 관리합니다. 실제 데이터 필터링 로직은 onApply 콜백에서 화면별로 구현합니다.
