원칙 · 정책

stam.buttons.css는 stam.components.css의 .stam-btn 기반 위에 single-dash variant 별칭(.stam-btn-primary 등)과 신규 variant(warning, success), 크기 별칭(.stam-btn-sm, .stam-btn-lg), 액션 컨테이너를 추가 정의합니다.

적용 원칙
  • 기존 page-level class (rq-btn, msl-btn 등)는 유지하고 stam-btn 계열을 병행 추가
  • JS selector가 기존 class에 의존하므로 기존 class를 절대 삭제하지 않음
  • CSS 로드 순서: stam.components.css → stam.drawer.css → stam.table-selection.css → stam.buttons.css → 페이지 CSS
  • Drawer footer 버튼이 이미 stam-btn stam-btn--md stam-btn--primary 구문을 사용하는 경우 중복 추가 금지
  • 버튼 클릭 JS 로직 변경 금지

HTML 사용법

병행 적용 패턴 (기존 class 유지)
<!-- 기존 class 유지 + stam-btn 병행 --> <button class="rq-btn rq-btn-pri stam-btn stam-btn-primary" id="rq-reg-btn">등록</button> <button class="rq-btn rq-btn-out stam-btn stam-btn-outline">내보내기</button> <button class="rq-btn rq-btn-ghost stam-btn stam-btn-ghost">취소</button> <button class="rq-btn rq-btn-del stam-btn stam-btn-danger" id="rq-del-btn">삭제</button>
Drawer footer — 이미 stam-btn--* 구문 사용 시 (추가 불필요)
<!-- Drawer footer 버튼: 이미 stam-btn--md --primary 사용 → 추가 금지 --> <button class="stam-btn stam-btn--md stam-btn--ghost wbs-fv-trigger-btn">전체 보기</button> <button class="stam-btn stam-btn--md stam-btn--secondary wbs-drawer-cancel-btn">취소</button> <button class="stam-btn stam-btn--md stam-btn--primary wbs-btn-primary">저장</button>
신규 화면 — stam-btn 단독 사용
<button class="stam-btn stam-btn-primary" type="button">등록</button> <button class="stam-btn stam-btn-outline" type="button">내보내기</button> <button class="stam-btn stam-btn-danger stam-btn-sm" type="button">삭제</button>

CSS 로드 순서

<link rel="stylesheet" href="../../css/stam.tokens.css"> <link rel="stylesheet" href="../../css/stam.shell.css"> <link rel="stylesheet" href="../../css/stam.components.css"> <link rel="stylesheet" href="../../css/stam.form-controls.css"> <link rel="stylesheet" href="../../css/stam.drawer.css"> <link rel="stylesheet" href="../../css/stam.table-selection.css"> <link rel="stylesheet" href="../../css/stam.buttons.css"> <!-- ← 여기 --> <link rel="stylesheet" href="../../css/stam.{page}.css">
공통 CSS가 페이지 CSS보다 먼저 로드되므로 페이지 CSS가 동일 specificity에서 덮어씁니다. 페이지별 커스텀이 필요하면 페이지 CSS에서 overrride하면 됩니다.

08

다크 모드

버튼 토큰(--btn-primary-bg 등)은 stam.tokens.css의 [data-theme="dark"] 블록에서 자동으로 다크 모드 값으로 교체됩니다.
추가 다크 모드 재정의는 stam.buttons.css 내 [data-theme="dark"] 블록에서 처리됩니다.
