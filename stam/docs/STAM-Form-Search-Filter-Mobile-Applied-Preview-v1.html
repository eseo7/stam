01 Overview — 문서 목적 목적

본 문서는 STAM 모바일에서 Search · Filter · Form · Select · Date · Validation 공통 적용 기준을 시각화한다. Mobile Shell / Top / Navigation Guide v1 의 모바일 원칙(48px Topbar, Drawer, safe-area, page hscroll 금지)을 상속하고, WBS Mobile Board Applied Preview v1 의 카드 리스트 / 검색 / 필터 시안을 입력 UI 관점에서 보완한다. 본 문서는 디자인 / Preview 작업으로 실제 제품 코드는 한 줄도 수정하지 않는다.

03 Mobile Search Pattern — 진입 방식 3안 Search

STAM 모바일 검색은 현재 화면 검색이 기본 책임이다. 전역 검색은 별도 entry, 조건 검색은 Filter Bottom sheet 로 분리한다. 아래 3안 중 A안 (Inline Expand) 를 목록형 화면에 우선 적용한다.

A Inline Search Expand 목록형 우선
Topbar 검색 icon 탭 → topbar 아래 36px search bar 확장. 현재 화면 컨텍스트 유지하면서 즉시 결과 갱신.
Pros
  • 화면 전환 없음 · 결과 즉시 노출
  • topbar + 36px 만 차지 — content 영향 최소
  • chip 필터와 AND 결합이 자연스럽다
Cons
  • 최근 검색 / 자동완성을 한 화면에 모두 노출하기 어렵다
B Full Search Overlay 전역 검색
화면 전체를 검색 모드로 덮는다. 최근 검색 / 카테고리별 결과 노출 가능. STAM 전체(홈) 진입 검색에 적합.
Pros
  • 최근 검색 / 자동완성 / 그룹별 결과 모두 노출
  • history 분리 명확 (back 으로 이전 화면)
Cons
  • 현재 화면 컨텍스트가 일시적으로 사라짐
  • WBS 단독 검색에는 과중
C Search + Filter Bottom Sheet 복잡 조건 한정
Bottom sheet 안에 search input + 조건 chip 결합. 복잡한 조건(상태 + 기간 + 담당자 + 분류) 한해 A안과 병행.
Pros
  • 검색어 + 조건을 한 흐름에서 합성 가능
  • 적용 / 초기화 footer 와 자연스럽게 결합
Cons
  • 키보드 + sheet 동시 점유 — 화면이 크다
  • 1차 검색 진입으로는 비효율

07 Sort / View Option — 정렬 · 보기 Sort / View

Breakpoint 권장
  • max-width: 768px = 모바일 기준 (본 가이드 적용)
  • 768 ~ 1024px = 태블릿 · PC Drawer 축소 사용
  • ≥ 1024px = PC 기준 (기존 정의)
  • 모바일 분기는 컨테이너 단위로 분리 (단일 selector 덮기 금지)
