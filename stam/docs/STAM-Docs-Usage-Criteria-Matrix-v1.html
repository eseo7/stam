STAM Docs Usage Criteria Matrix v1 현재 STAM docs 문서를 핵심 / 보조 / 보류 3단계로 분류하고, 각 문서의 역할 · 참고 시점 · Claude Code(클코) 작업 시 필수 여부 · 사용자 확인 필요 여부 · 후속 작업 연결을 정리한 문서 사용 기준표다. 분류 대상은 STAM Docs Index 기준 핵심 세트이며, 토큰 매핑 · QA 리포트 · 화면설계서 템플릿 등 세부 패밀리는 05 패밀리 요약에 그룹 1행으로 묶어 정리한다. 문서 전용 inventory 작업이며 제품 화면 / JS / CSS / Firebase 를 변경하지 않는다.

01 Purpose & 분류 기준 — 표 읽는 법 Legend

이 표는 “어떤 문서를 언제 봐야 하는가”를 빠르게 판단하기 위한 기준표다. 분류 3단계는 다음과 같다.

핵심 Core 제품/화면 작업의 현재 기준. 작업 전 반드시 따라야 하는 baseline · 계약 · Gate · SSOT · 전략 문서.
보조 Supporting 특정 작업·컴포넌트·정책을 다룰 때 상황별로 참고하는 가이드 · 템플릿 · 정책 · Preview.
보류 Reference-only 현재 작업 기준이 아닌 이력 · QA 결과 · 감사 · 후속 확장 보류 문서. 비교·근거용으로만 본다.

컬럼 의미

  • 역할 — 문서가 책임지는 내용.
  • 언제 참고 — 이 문서를 펼쳐야 하는 작업 시점.
  • 클코 필수 — Claude Code 작업 시 준수 강도. 필수 반드시 적용 · 권장 해당 작업 시 적용 · 참고 필요 시 열람.
  • 사용자 확인 — 사람(PM/기획자) 승인·결정이 필요한가. 필요 승인/결정 대상 · 불필요 기준 그대로 적용.
  • 후속 연결 — 이 문서 다음에 이어지는 작업·문서.

06 사용 가이드 — How to Use Workflow

작업 유형별로 먼저 펼쳐야 할 문서 순서.

작업 시작→ 02 핵심 확인→ 03 보조에서 해당 영역→ 04 보류는 근거만→ Gate / 사용자 승인
제품 화면 변경
  • UI Baseline Guide → Component Guide → Implementation Gate (필수)
  • 해당 컴포넌트 Guide + Token Mapping 참고
  • Gate 통과 = 사용자 확인 필요
신규 게시판/보드
  • Common UI Control Center → Board List Template Contract (필수)
  • New Board List Screen Template + Toolbar/Filter Guide
  • customFields 전제 · Project Cycle Automation 연결
IA / 메뉴 변경
  • Sitemap & IA Overview (nav-data SSOT) 확인
  • IA 변경은 사용자 확인 필요
  • Full Menu Structure / IA Baseline 보조 참고
기능/백엔드 설계
  • Backend Architecture & API Boundary Guide 확인
  • 보안·정책은 Security/Policy 패밀리 참고
  • 아키텍처 결정은 사용자 확인 필요
유지 원칙 신규 문서를 추가하면 Docs Index 카드 등록과 함께 이 기준표(핵심/보조/보류)에도 분류를 반영한다. 분류 변경은 문서 폐기가 아니라 단계 재배치로 처리한다.
