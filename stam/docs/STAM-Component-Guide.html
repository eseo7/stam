역할: 사람용 컴포넌트 카탈로그 stub

This guide is a human-readable component catalog. The governance source for whether a component already exists is STAM Agent Component Inventory. Detailed component guides will be added in later phases.

컴포넌트가 어떻게 보이고, 어떤 상황에서 쓰이는지 설명하는 문서의 진입점입니다. 상세 컴포넌트 가이드는 후속 Phase에서 분리 생성합니다.

03 General Rules — 공통 규칙 기준 고정

스타일 / 폰트

  • 모든 컴포넌트는 --font (S-Core Dream) 사용
  • 외부 폰트·시스템 폰트 직접 기입 금지
  • 컴포넌트 내부 font-family 별도 선언 금지 — 상속 사용
  • 컬러는 반드시 stam.tokens.css 변수 참조

인터랙션 / 애니메이션

  • hover transition: 120ms ease-out
  • press/active transition: 80ms ease-out
  • 드로어·다이얼로그 슬라이드: 200ms ease-out
  • transform: scale() press 효과 금지
  • Bounce · Spring · Overshoot 금지

접근성

  • 모든 인터랙티브 요소 :focus-visible 링 필수 (--bd-f)
  • disabled 상태는 opacity·pointer-events: none 조합
  • 다이얼로그 포커스 트랩 필수
  • 아이콘 단독 버튼에 aria-label 필수

금지

  • 컴포넌트 임의 리디자인 금지
  • 토큰명 임의 신규 생성 금지
  • 컴포넌트별 인라인 스타일 남발 금지
  • emoji 사용 금지 — SVG 아이콘만 허용

05 Next Steps — 후속 작업 계획

후속 확장 작업 후보

  • 공통 CSS 컴포넌트 토큰 정규화 — Variant·Size·State 토큰 네이밍 정합화
  • Button / Form / Table / Drawer 실구현 클래스 정합화 — 가이드 ↔ stam.components.css 매핑
  • Icon Inventory 문서 작성 — 실제 사용 아이콘 세트와 명세 정리
  • Notification Drawer / Popover Guide 후보 검토 — v2 컴포넌트 후보 정의
  • 실제 제품 화면 적용 전 QA 체크리스트 작성 — 페이지 단위 점검 표
적용 순서 권장 상세 컴포넌트 가이드는 후속 Phase에서 분리 생성한다. Phase 1에서는 stub 인덱스와 cross-link만 유지한다.
