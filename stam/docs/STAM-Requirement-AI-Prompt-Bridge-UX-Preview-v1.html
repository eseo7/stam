01 Purpose — 이 문서의 목적 Preview Baseline

이 문서는 자유 텍스트 요구사항을 STAM 이 직접 AI 분석하지 않고, 사용자가 외부 AI 에 붙여넣어 STAM 표준 Import 포맷(Excel/CSV)으로 변환하도록 돕는 Prompt Bridge UX 를 정의한다.

  • STAM 내부에서 AI 분석을 직접 수행하지 않는다.
  • STAM 은 외부 AI 에 붙여넣을 프롬프트를 만들어 준다.
  • 사용자가 외부 AI 결과를 검토한 뒤 Excel/CSV 로 저장하거나 붙여넣어 Import Builder 에 업로드한다.
  • 외부 AI 결과는 확정 요구사항이 아니라 Draft / Review Needed 상태로 들어온다.
  • 원문에 없는 내용을 AI 가 만들지 않도록 프롬프트에 강한 제약을 둔다.
  • 개인정보/고객자료를 외부 AI 에 붙여넣을지는 사용자가 직접 판단하며, 주의문구를 명확히 표시한다.
  • 이 기능은 AI Requirement Extraction 의 직접 구현이 아니라 Prompt Bridge 다.
정적 UX Preview 이번 작업은 정적 UX Preview 문서다. 실제 AI API 호출 / 파일 변환 / Excel 생성 / CSV parsing / DB 저장은 구현하지 않는다.
02 Product Position — Prompt Bridge의 역할 Bridge

Prompt Bridge 는 “자유 텍스트”와 “STAM Requirement Import” 사이의 중간 다리다. STAM 이 AI 가 되는 것이 아니라, 사용자가 외부 AI 를 잘 쓰도록 프롬프트와 표준 포맷을 제공한다.

자유 텍스트→ Prompt Bridge (프롬프트 생성)→ 외부 AI (사용자가 사용)→ STAM 표준 Import 포맷
Prompt Bridge 가 하는 일
  • 외부 AI 용 프롬프트 생성
  • STAM 표준 컬럼 스키마 안내
  • 결과 업로드 흐름 안내
  • Import Builder 로 연결
Prompt Bridge 가 하지 않는 일
  • STAM 내부 AI 분석
  • AI API 자동 호출
  • 외부 AI 로 자동 전송
  • 결과 자동 확정/자동 등록

비용/운영STAM 이 AI API 를 직접 호출/과금하지 않는다
정확성 책임AI 결과는 사용자가 검토·승인. STAM 이 자동 확정하지 않음
유연성사용자가 선호하는 외부 AI(ChatGPT/Claude/Gemini 등)를 선택
단계적 구현실제 AI 연동은 후속 확장. 1차는 프롬프트 생성 + 표준 포맷 안내
책임 분리 STAM 이 책임 있게 할 수 있는 범위(프롬프트 생성·표준 포맷·업로드 흐름)와, 사용자가 판단해야 하는 범위(외부 AI 사용·민감정보·결과 검토)를 분리한다.
04 User Scenario — 실무 시나리오 Scenario

기획자가 고객 메일/회의록 텍스트를 요구사항으로 정리하고 싶지만 양이 많다. STAM 이 직접 분석하는 대신, 외부 AI 에 쓸 프롬프트를 만들어 주고, 사용자가 결과를 검토해 Import 한다.

원문을 STAM 에 붙여넣기→ STAM 이 변환 프롬프트 생성→ 사용자가 외부 AI 에 붙여넣기→ AI 가 표/TSV 반환

10 Generated Prompt Preview Core Output

외부 AI 에 붙여넣을 프롬프트 예시. 프롬프트는 “원문에 없는 내용 생성 금지”를 포함한 강한 제약을 반드시 담는다.

생성된 프롬프트 (복사용 예시)
  • 당신은 요구사항 정리 보조자다. 아래 원문을 STAM 요구사항 표로 변환하라.
  • 원문에 없는 내용을 만들지 말 것.
  • 확실하지 않으면 값은 미지정 으로 둘 것.
  • 한 행은 하나의 요구사항 후보로 만들 것.
  • 각 행의 sourceText 에 원문 근거를 반드시 남길 것.
  • 결과는 Excel 에 붙여넣기 쉬운 Markdown table 또는 TSV 로 출력할 것.
  • 모든 행의 reviewStatus 는 Review Needed 로 둘 것 (Imported Draft).
  • 우선순위/화면/담당자/권한은 원문에 명확할 때만 작성할 것.
  • 불확실한 점은 notes 에 적을 것.
  • 컬럼: externalRequirementId, title, description, requirementType, actor, priority, status, reviewStatus, acceptanceCriteria, relatedScreenCandidate, sourceText, confidence, notes
  • [원문 시작] … 사용자가 붙여넣은 원문 … [원문 끝]
제약이 핵심이다 프롬프트의 목적은 “많이 생성”이 아니라 “원문 근거 있는 후보만, 불확실하면 미지정”이다. 이 제약이 정확성 가드레일의 출발점이다.

14 Import Builder Handoff Handoff

AI 결과(Excel/CSV)는 Requirement Import Builder 의 입력으로 들어간다. Prompt Bridge 는 Import Builder 를 대체하지 않는다.

Excel/CSV→ Import Builder 업로드→ 컬럼 매핑→ 검증→ 미리보기→ 확정 가져오기 (Draft)
대체 아님 Prompt Bridge 는 “외부 AI 로 표준 포맷을 만들도록 돕는 단계”이고, 실제 가져오기·컬럼 매핑·검증·확정은 Import Builder 가 담당한다.
15 Draft / Review Needed Rules Human-in-loop
  • AI 가 만든 표는 확정 요구사항이 아니다.
  • 모든 행은 Imported Draft 다.
  • 모든 행은 Review Needed 다.
  • 사람이 검토/수정/승인한 뒤에만 Traceability Matrix 에서 연결 확정이 가능하다.
외부 AI 결과→ Imported Draft→ Review Needed→ 사람 승인→ 연결 확정 가능
16 Privacy and Security Notice Notice
외부 AI 사용 전 반드시 확인 고객사 자료 · 개인정보 · 계약정보 · 보안정보를 외부 AI 에 붙여넣기 전에 사용자가 직접 검토해야 한다. 민감정보는 마스킹 후 사용을 권장한다.
  • STAM 은 외부 AI 전송을 자동 수행하지 않는다.
  • 프롬프트/원문은 사용자가 직접 복사·붙여넣기 한다.
  • 외부 AI 서비스의 데이터 처리 정책은 사용자가 확인한다.
  • 민감정보는 마스킹/익명화 후 사용 권장.
  • 고객 계약상 외부 전송이 제한되는 자료는 사용하지 않는다.
판단 주체 무엇을 외부 AI 에 붙여넣을지는 사용자가 직접 판단한다. STAM 은 주의문구를 표시할 뿐 전송을 대신하지 않는다.
17 Accuracy Guardrails Guardrails
  • AI 결과는 틀릴 수 있다. 검토 없이 신뢰하지 않는다.
  • 없는 요구사항을 생성하면 안 된다 (프롬프트 제약).
  • 화면명/우선순위/담당자/권한은 원문 근거가 없으면 미지정.
  • sourceText 와 notes 로 근거/불확실성을 남긴다.
  • Import 후에도 사람 검토가 필요하다.
근거 우선 정확성의 핵심은 “원문 근거(sourceText)”와 “불확실성 표기(notes/confidence)”다. STAM 은 AI 결과의 의미를 자동 보증하지 않는다.

19 Relationship with Text Intake Helper Mapping

Text Intake Helper 는 AI 없이 수동으로 블록을 나누고 Draft 를 만드는 보조 도구다. Prompt Bridge 는 같은 자유 텍스트를 외부 AI 로 표준 Import 포맷에 맞게 변환하도록 돕는다. 둘은 대체 관계가 아니라 선택지다.

구분Text Intake HelperAI Prompt Bridge
분석 방식AI 없음 · Rule-based + 수동외부 AI · 사용자가 사용
STAM 역할블록 분리/선택 보조프롬프트 생성/포맷 안내
결과Draft / Review NeededDraft / Review Needed
의미 분석안 함STAM 안 함 (외부 AI 가)
충돌 없음 두 경로 모두 STAM 내부에서 자유 텍스트를 AI 로 의미 분석하지 않는다(PR #227 기준 유지). Prompt Bridge 의 AI 사용은 외부에서 사용자가 수행한다.
20 Relationship with Requirement Import Builder Mapping

Requirement Import Builder 는 Excel/CSV 결과를 실제로 가져오는 도구다. Prompt Bridge 결과(Excel/CSV)는 Import Builder 입력이 된다.

  • Prompt Bridge → 표준 포맷 Excel/CSV 생성을 돕는 단계
  • Import Builder → 업로드 · 컬럼 매핑 · 검증 · 미리보기 · 확정 가져오기
  • sourceType 은 AI 변환 출처를 함께 보존(예: AI Prompt Bridge)
  • 가져온 결과는 Draft / Review Needed 에서 시작
역할 분담 Prompt Bridge 는 “포맷 맞추기”, Import Builder 는 “실제 가져오기”. Prompt Bridge 는 Import Builder 를 대체하지 않는다.
21 Relationship with Traceability Matrix Mapping

가져온 요구사항이 기능정의·화면·WBS·화면설계서·테스트로 연결됐는지는 Traceability Matrix 에서 본다. Prompt Bridge 로 들어온 요구사항도 Draft / Review Needed 이므로, 승인 전까지 Matrix 에서 연결을 확정하지 않는다.

Prompt Bridge→ Import Builder (Draft)→ 사람 승인→ Traceability Matrix 연결 확정
일관된 기준 출처가 Excel Import / Text Intake / AI Prompt Bridge 중 무엇이든, 모든 요구사항은 Draft 에서 시작하고 사람 승인 후 연결이 확정된다.
