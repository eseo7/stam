01 Overview — 문서 목적 목적

02 Topbar Detail Scope — 다루는 세부 항목 범위

본 문서가 시각 기준을 확정하는 세부 항목은 다음 7가지이다. 각 항목은 STAM Topbar의 RIGHT 영역 또는 CENTER 영역에서 사용되며, 모두 light / dark 양쪽 컨텍스트에서 일관되게 동작해야 한다.

01 / Notification Indicator 알림 표시 방식 숫자 badge vs red dot · topbar 내 표시 방식 결정
02 / Theme Toggle 테마 전환 버튼 icon-only · 흰 사각형으로 보이지 않을 것
03 / Search Pill CENTER 검색 입력 텍스트가 아니라 pill / input 형태로 명확히 식별
04 / User Menu 사용자 메뉴 avatar + chevron · 28px · dropdown 진입점
05 / Quick Action 우측 빠른 액션 최대 1~2개 · 과밀하면 page toolbar로 이동
06 / Right Action Spacing 우측 액션 간격 gap 6~8px · 업무 / utility 그룹 separator
07 / Light / Dark Context 테마 컨텍스트 일관성 light · dark 둘 다 깨지지 않을 것
03 Notification Indicator — 결정 Decision
결정 — Topbar 알림 표시는 숫자 badge 대신 red dot 만 사용한다.
· Topbar 알림 아이콘에는 숫자를 표시하지 않는다.
· 알림 있음 상태는 red dot 만 표시한다.
· 알림 개수는 알림 패널 / 드롭다운 내부에서 표시한다.
· 2자리 · 99+ 숫자 badge는 topbar에서 사용하지 않는다.
R결정 사유 왜 숫자 badge 대신 red dot 인가
숫자 badge 의 문제
  • Topbar icon button 자체가 작아 숫자 가독성이 떨어진다
  • 2자리 이상이 되면 badge가 커져 아이콘을 가린다
  • 99+ 표기는 topbar에서 시각적 균형을 깨뜨린다
  • 모바일 36~40px 버튼에서는 badge가 잘리기 쉽다
  • light / dark 컨텍스트에서 badge 색이 어색해진다
red dot 의 장점
  • 매우 작은 dot 만으로 "알림 있음"이 명확히 보인다
  • 2자리 / 100건 이상 모든 상황에서 동일하게 안정적이다
  • 모바일 / 데스크탑 모두 동일 시각 규칙으로 동작한다
  • 아이콘 자체 가독성을 침해하지 않는다
  • 알림 수는 패널 내부에서 더 정확히 보여줄 수 있다
A시각 비교 close-up — bad badge vs good dot
Bad · 1자리 숫자 badge 28px 버튼 위에서 7px 글자는 가독성이 낮다. 금지
Bad · 2자리 / 99+ badge badge가 커져 아이콘을 가린다. 금지
Good · red dot only 알림 있음만 표시. 개수는 알림 패널에서 본다. 권장
04 Notification Dot — Visual Spec Spec

알림 dot 의 시각 기준은 아래와 같다. dot 은 알림 있음 상태에서만 노출되며, 아이콘을 과하게 가리지 않도록 우상단에 topbar background 색의 1~2px border 와 함께 표시된다. dot 내부에는 어떤 텍스트도 표시하지 않는다.

Size 7px 또는 8px 기본 8px · 작은 dense topbar 에서만 7px 허용
Border-radius 999px (완전 원) 정원이 아니면 dot 으로 인식되지 않는다
Color red 계열 · #FF5252 권장 danger token 과 일관 · light/dark 동일
Border 1~2px solid topbar bg dot 과 아이콘 사이에 분리 공간을 만든다
Position icon button 우상단 top 4px · right 4px (28px 버튼 기준)
Text 없음 숫자 / 글자 / 카운트 표시 금지
SState by 알림 상태
State Dot 노출 설명
no notification hidden 알림 없음. dot 자체를 렌더링하지 않는다.
has notification red dot visible 읽지 않은 알림이 1건 이상. 개수는 표시하지 않고 dot 만 표시한다.
unread high priority red dot · danger color 유지 긴급 알림이 있는 경우에도 색만 유지하며 숫자를 표시하지 않는다.
all read hidden 모두 읽음. dot 제거.
05 Theme Toggle — Visual Spec Spec

현재 staging 의 테마 토글은 dark topbar 위에서 흰 사각형처럼 보이는 문제가 있다. 테마 토글은 icon-only button 으로 정의되며, topbar 의 다른 우측 액션과 동일한 시각 규칙을 따른다. currentColor 기반 아이콘으로 그려지며 light / dark 양쪽에서 같은 형태로 보여야 한다.

07 Right Action — Spacing & Grouping Spec

09 Token / Class Mapping — Proposal Proposal

아래 class 이름은 향후 구현 후보이며, 본 문서에서는 실제 CSS 를 작성하지 않는다. Cursor 가 실제 반영 시 본 제안을 시작점으로 활용한다.

.stam-topbar-notification 알림 아이콘 버튼. icon-only 28px. dot 자식 요소만 별도 spec.
.stam-notification-dot 알림 있음 표시 dot. 8px · red · 2px topbar bg border. 있음 상태에서만 렌더링.
.stam-topbar-search CENTER 검색 pill. 28h · 240~320w · 1px border · 7px radius.
.stam-topbar-theme 테마 토글 icon-only 28px. moon / sun currentColor.
.stam-topbar-action 우측 icon-only 액션 공통. hover · focus 동일 규칙.
.stam-topbar-actions-util utility 그룹 컨테이너. 알림 · 테마 · 사용자 묶음.
.stam-topbar-actions-work 업무 액션 그룹. Quick Action 최대 1~2.
주의 class 이름은 제안 단계이며, 실제 정의는 Cursor 가 별도 commit 에서 CSS / JS 와 함께 결정한다. 본 문서는 .stam-topbar-* 어떤 class 도 실제로 정의하거나 적용하지 않는다.
10 Do / Don't 기준
Do
  • 알림은 red dot only 로 표시한다
  • icon-only 우측 액션 버튼은 28px 로 통일한다
  • CENTER 검색은 pill 형태로 visible 하게 표시한다
  • 아이콘은 currentColor 기준으로 그린다
  • dark context 에 맞게 background / border override 한다
  • 업무 / utility 그룹 사이에 1px vertical separator 둔다
  • 알림 개수는 알림 패널 내부에서 표시한다
  • 모바일에서는 검색을 icon → overlay 방식으로 둔다
Don't
  • topbar 에 알림 숫자 badge 표시 금지
  • 2자리 / 99+ badge 시도 금지
  • 테마 토글을 흰 사각형으로 만들지 말 것
  • CENTER 검색을 텍스트만으로 두지 말 것
  • topbar 에 업무 액션 과다 배치 금지 (Quick Action ≤ 2)
  • 전역 .tbtn 일괄 치환 금지
  • 다른 화면 일괄 적용 금지 — WBS 부터 적용
  • icon 을 고정 색상으로 박지 말 것 (currentColor)
