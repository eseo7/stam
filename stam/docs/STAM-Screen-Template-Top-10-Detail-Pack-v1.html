Top 10 Detail Pack v1

Priority 30 Korean Baseline(P2)에서 선정한 Top 10 후보 템플릿을 목적·레이아웃·필드·액션·권한·상태·모바일·QA 체크리스트 기준으로 상세화한 기준 문서다. 실제 UI 구현이나 제품 화면 수정이 아니라, 화면설계서 제품에서 템플릿 선택 → 초안 생성 → 검토 흐름을 수행할 때 각 템플릿이 제공해야 할 기본 구조를 정의하는 기준이다. 한국형 업무/관리자/그룹웨어/전자결재/프로젝트 산출물 화면을 기준으로 작성했다. 상위 기준은 Master Guide(P1), 30종 목록 기준은 Priority 30(P2)를 따른다.

Top 10 Selection Rationale — 상세화 우선 선정 이유

산출물 관리 핵심 흐름목록 조회(T01·T02·T03·T05) → 상세/승인(T07·T08) → 작성/수정(T11·T12) → 설정(T26·T27) 흐름이 완성된다. 이 10개만으로 산출물 관리의 80% 시나리오를 커버한다.
반복 사용 가능성T01(기본 목록)·T07(기본 상세)·T11(기본 작성폼)은 요구사항·화면설계·WBS·정책·공지 등 모든 산출물 유형에 반복 적용된다. 한 번 기준을 잡으면 전체에 적용 가능하다.
Board Builder 연계T01·T05·T11·T12·T26·T27은 Board Builder 필드/게시판 구조 정의와 직접 또는 강하게 연결된다. 설정 화면(T26·T27)은 Board Builder 자체가 화면설계서 템플릿으로 표현된 형태다.
권한/검토/승인/이력 연결T07·T08·T11·T12는 검토요청·승인·변경이력(T09 Change History 패턴)과 직접 연결된다. 특히 T08 Approval Detail View는 전자결재 핵심 화면으로 권한 분기 기준이 복잡하다.
모바일 기준 필요성T01·T02·T03·T05는 PC 테이블 기반이라 모바일/좁은 화면 전환 기준이 반드시 필요하다. T08·T11은 모바일에서 핵심 액션(승인/저장) 우선 노출이 UX 핵심이다.
화면설계서 초안 가속이 10개 기준이 확립되면 화면설계서 제품에서 템플릿 선택 → 필드/액션/권한 초안 자동 생성 흐름의 기준으로 직접 사용 가능하다. 설계자가 처음부터 만들지 않아도 된다.

T01

Basic Board List

List / BoardBoard Builder 직접 연계

목적

모든 산출물 유형의 기본 목록 조회 화면. 레코드 목록을 테이블로 표시하고, 기본 검색·정렬·페이지네이션·신규 등록 흐름을 제공한다.

적용 화면 예시

공지사항 목록 · 요구사항 목록 · 정책 목록 · 화면설계서 목록 · 일반 산출물 목록

기본 레이아웃

상단 검색바 + 툴바(신규/내보내기/컬럼설정) → 본문 테이블 → 하단 페이지네이션. 툴바는 목록 상단에 고정.

기본 필드

  • ID (자동)
  • 제목 (필수, 클릭 시 상세)
  • 상태 (배지)
  • 담당자
  • 작성자
  • 수정일
  • 조회수 (선택)

기본 액션

  • 키워드 검색 / 초기화
  • 컬럼 정렬(오름/내림)
  • 페이지네이션 / 페이지크기
  • 행 선택(단일·다중)
  • 신규 등록
  • 상세 진입
  • 일괄 삭제(권한)
  • 엑셀·문서 내보내기

상태값 / 배지

작성중 검토중 완료 보류 반려

상태 배지는 Cross-template 공통 기준(섹션 13) 준수.

권한 포인트

  • 목록 조회 — 최소 권한
  • 행 열람 — 문서별 공개범위
  • 신규 등록
  • 수정 / 삭제
  • 일괄 삭제
  • 내보내기

연결 산출물

  • 요구사항 · 화면설계서 · WBS · 정책 · 공지
  • 상세: T07 Basic Detail View
  • 작성: T11 Basic Create Form
  • 승인: T08 Approval Detail View

오류 상태

모바일 / 좁은 화면

T02

Advanced Search List

List / BoardBoard Builder 부분 연계

목적

키워드·기간·상태·담당자·분류·태그 등 다중 조건 조합으로 산출물을 통합 검색하는 화면. 조건 저장·재사용·결과 내보내기를 지원한다.

적용 화면 예시

산출물 통합 검색 · 요구사항 검색 · 화면설계서 검색 · WBS 검색 · 고급 필터 목록

기본 레이아웃

상단 검색 조건 패널(접힘/펼침) + 검색/초기화/조건저장 버튼 → 적용된 조건 칩 → 결과 목록 테이블 → 페이지네이션.

기본 필드(검색 조건)

  • 키워드 (전문 검색)
  • 기간 (시작~종료)
  • 상태 (다중 선택)
  • 담당자
  • 분류 / 유형
  • 태그 (복수)

기본 액션

  • 검색 실행
  • 조건 초기화
  • 조건 저장(프리셋) / 불러오기
  • 결과 건수 표시
  • 결과 내보내기(엑셀·문서)
  • 상세 진입

상태값 / 배지

적용된 조건 칩 N건 검색됨 조건 없음

검색 결과 건수는 항상 노출. 적용 조건은 칩으로 표시·개별 제거 가능.

권한 포인트

  • 검색 범위 (전체·프로젝트·본인)
  • 필드 노출 (역할별)
  • 결과 내보내기
  • 조건 저장 (개인·공유)

연결 산출물

  • 요구사항 · 화면설계서 · WBS · 테스트 케이스
  • 상세: T07 Basic Detail View
  • 저장 프리셋: T17 Saved Filter Preset

Board Builder 연계

게시판별 검색 가능 필드를 Board Builder 구조에서 읽어 조건 패널을 동적으로 구성. 일부 필드 구조만 연계(부분 연계).
→ Board Builder

빈 상태

"검색 결과가 없습니다."
"조건을 변경하거나 초기화 후 재시도하세요." + [초기화] 버튼

오류 상태

"검색 중 오류가 발생했습니다. 잠시 후 다시 시도하세요."
잘못된 조건 입력 시 인라인 유효성 메시지.

모바일 / 좁은 화면

• 검색 조건 패널 기본 접힘 → 토글 펼침
• 적용된 조건은 칩 요약으로 표시
• 결과 목록 카드 전환
• 내보내기 버튼 툴바 최우선

#QA 항목확인 기준
1조건 조합 검색2개 이상 조건 동시 적용 검색 동작
2초기화전체 조건 초기화 후 전체 결과 반환
3조건 저장/불러오기프리셋 저장, 이름 입력, 불러오기 적용
4결과 없음 상태0건일 때 빈 상태 메시지 노출
5모바일 접힘 토글조건 패널 접힘/펼침 동작
6조건 칩 제거개별 조건 칩 X 클릭 시 해당 조건 제거 후 재검색

List / BoardBoard Builder 부분 연계

목적

업무 항목을 그룹(업무영역·기능·메뉴·카테고리)으로 묶어 표시하는 목록 화면. 그룹 접기/펼치기, 그룹별 집계, 항목 상태 관리를 제공한다.

적용 화면 예시

기능정의서 그룹 목록 · 메뉴/화면 목록 · WBS 업무 목록 · 산출물 그룹 목록 · 테스트 케이스 그룹

기본 레이아웃

전체 접기/펼치기 툴바 → 그룹 헤더(이름·집계·상태) + 그룹 내 항목 행 반복. 그룹 헤더 sticky 고정.

기본 필드

  • 그룹명 (헤더)
  • 그룹 내 항목 수 / 완료 수
  • 항목명
  • 항목 상태
  • 담당자
  • 진척률 (선택)
  • 마감일 (선택)

기본 액션

  • 전체 접기 / 전체 펼치기
  • 그룹별 개별 접기/펼치기
  • 그룹 내 항목 정렬
  • 신규 항목 등록
  • 항목 상태 변경
  • 상세 진입

상태값 / 배지

작성중 검토중 완료 지연 미시작

그룹 헤더에 완료율(완료/전체) 표시.

권한 포인트

  • 그룹 열람 (그룹별 공개 범위)
  • 그룹 내 항목 등록/수정/삭제
  • 그룹 자체 추가/삭제 (관리자)
  • 상태 변경 (담당자·권한자)

연결 산출물

  • WBS · 기능정의서 · 메뉴/화면목록
  • 상세: T07 Basic Detail View
  • 작성: T11 Basic Create Form

Board Builder 연계

→ Board Builder

목적

컬럼 수가 많은 대용량 데이터(테이블 정의서·API 명세 등)를 효율적으로 조회하는 화면. 핵심 컬럼 고정 + 가로 스크롤 + 전체보기(Focus View) 지원.

적용 화면 예시

테이블 정의서 · API 명세 목록 · DB 정보 · 서버 정보 · 접근보안 정보 · ERD 필드 목록

기본 레이아웃

좌측 핵심 컬럼 고정 + 우측 가로 스크롤 영역. 상단 컬럼 설정 · 포커스 뷰 전환 버튼. 헤더 행 sticky.

기본 필드

  • 키 / ID (고정 컬럼)
  • 이름 / 컬럼명 (고정 컬럼)
  • 타입
  • 설명
  • 제약조건 / NULL 여부
  • 기본값
  • 비고
  • (추가 컬럼 동적 구성)

기본 액션

  • 컬럼 고정 / 고정 해제
  • 가로 스크롤
  • 전체보기 / 포커스 뷰 전환
  • 컬럼 표시/숨김 설정
  • 정렬
  • 내보내기 (Excel·CSV)

상태값 / 배지

활성 비활성 검토중 고정 컬럼

고정 컬럼은 시각적으로 구분(배경·경계선). 행별 상태 배지.

권한 포인트

  • 컬럼별 노출 권한 (역할별)
  • 행 편집 권한
  • 전체 내보내기 권한
  • 컬럼 설정 저장 (개인·공유)

연결 산출물

  • 테이블 정의서 · API 명세 · DB ERD

  • 작성: T11 Basic Create Form

Basic Detail View

DetailBoard Builder 미연계

목적

단일 레코드의 상세 내용을 조회하는 기본 상세 화면. 헤더·메타정보·본문·첨부·변경이력·하단 액션바 구조를 제공한다.

적용 화면 예시

요구사항 상세 · 화면설계서 상세 · 정책 상세 · 공지 상세 · 기능정의서 항목 상세

기본 레이아웃

상단 헤더(제목+상태+액션) → 메타정보(작성자/담당자/수정일) → 본문(리치텍스트) → 첨부 → 변경이력 → 하단 액션바.

기본 필드

  • 제목 (헤더)
  • 상태 (배지)
  • 작성자
  • 담당자
  • 수정일 / 작성일
  • 본문 (리치텍스트)
  • 연결 항목
  • 첨부 파일
  • 변경이력 (접힘/펼침)

기본 액션

  • 수정 (권한)
  • 삭제 / 숨김 (권한)
  • 검토 요청
  • 인쇄 / 내보내기
  • 목록으로
  • 복사 (선택)
  • 승인 요청 → T08 연결

상태값 / 배지

작성중 검토중 승인요청 완료 보류 반려

권한 포인트

  • 문서 열람
  • 편집
  • 삭제 / 숨김
  • 검토 요청
  • 내보내기
  • 변경이력 열람

연결 산출물

  • 요구사항 · 화면설계서 · 정책
  • 승인: T08 Approval Detail View
  • 이력: T09 Change History Detail
  • 수정: T12 Multi-section Edit Form

Board Builder 연계

상세 보기는 목록 구조(T01·T05)에 종속. Board Builder 필드 정의가 메타정보 필드 세트에 간접 반영되나 직접 연계는 없음.

빈 상태

상세 화면은 항상 데이터 로드 전제. 로딩 상태 스켈레톤 표시.

오류 상태

"문서를 불러올 수 없습니다. 권한 또는 삭제 여부를 확인하세요."
404(삭제됨) / 403(권한없음) 메시지 구분.

모바일 / 좁은 화면

• 섹션 아코디언(본문·첨부·이력 순)
• 단일 컬럼 레이아웃
• 하단 액션바 fixed 여부 판단(긴 본문 시 고정 권장)
• 메타정보 2열 → 1열 전환

#QA 항목확인 기준
1메타정보 표시작성자·담당자·수정일 정상 표시
2본문 리치텍스트리치텍스트 렌더링 (이미지·링크·목록)
3변경이력 표시이력 섹션 접힘/펼침, 변경 항목 표시
4검토요청 액션검토요청 버튼 → 검토자 선택 → 상태 변경
5권한 분기편집/삭제 버튼 권한 없을 때 미노출
6모바일 아코디언섹션별 접힘/펼침 동작
T08

Approval Detail View

DetailBoard Builder 미연계

목적

승인 대상 문서의 결재선·의견·이력을 포함한 상세 화면. 승인자·요청자·참조자별 액션이 분기되며 전자결재 흐름의 핵심 화면이다.

적용 화면 예시

전자결재 상세 · 승인 요청 상세 · 산출물 승인 상세 · 검토 요청 상세

기본 레이아웃

상단 제목+승인 상태 → 대상 문서/본문 → 결재선(단계별 세로 표시) → 의견 입력 → 하단 액션바(승인/반려/보류) → 이력.

기본 필드

  • 제목 (대상 문서)
  • 승인 상태 (헤더 배지)
  • 요청자 / 요청일
  • 결재선 (단계·승인자·상태·의견)
  • 처리일
  • 반려 사유 (반려 시)

기본 액션 (역할별 분기)

  • 승인 (결재 권한자)
  • 반려 + 사유 입력 (결재 권한자)
  • 보류 (결재 권한자)
  • 의견 등록 (참조자·권한자)
  • 회수 (요청자, 미처리 단계)
  • 이력 펼치기

상태값 / 배지

승인대기 진행중 승인완료 반려 보류 회수됨

권한 포인트

  • 결재 단계별 승인 권한
  • 반려 권한 (단계 권한자)
  • 보류 권한
  • 회수 권한 (요청자, 미처리 시)
  • 의견 열람/등록 (참조자)
  • 이력 열람

연결 산출물

  • 승인 대상 산출물 (요구사항·화면설계 등)
  • 목록: T22 Approval Queue
  • 상세 원문: T07 Basic Detail View

Board Builder 연계

승인 흐름은 게시판 구조보다 결재선 정의에 의존. Board Builder 직접 연계 없음. 승인 완료 시 대상 문서 상태가 T07 상세로 반영됨.

빈 상태

해당 없음. 승인 상세는 항상 결재 요청 데이터 존재 전제.

오류 상태

"승인 정보를 불러올 수 없습니다. 결재선 또는 권한을 확인하세요."
이미 처리된 건 재처리 시 "이미 처리된 결재입니다." 안내.

모바일 / 좁은 화면

• 결재선 세로 스택 (단계 카드)
• 승인/반려 버튼 하단 고정바 최우선 노출
• 본문 아코디언(긴 경우)
• 의견 입력 영역 하단 고정 or 인라인

#QA 항목확인 기준
1결재선 단계 표시각 단계 승인자·상태·의견 정상 표시
2권한별 액션 분기승인자/요청자/참조자별 버튼 분기
3승인 처리승인 후 상태 변경·이력 기록
4반려 사유반려 시 사유 입력 필수 확인
5이력 표시처리 이력 시간순 표시
6모바일 고정바승인/반려 버튼 하단 고정 동작
T11

Basic Create Form

Create / Edit FormBoard Builder 직접 연계

목적

신규 레코드를 작성·저장하는 기본 폼 화면. 필수 필드 유효성 검증, 임시저장, 저장 후 상세 이동 흐름을 제공한다.

적용 화면 예시

요구사항 등록 · 정책 등록 · 일반 게시글 작성 · 공지 작성 · 기능 항목 등록

기본 레이아웃

단일 컬럼 폼 영역(제목+필드+본문+첨부) → 하단 액션바(저장/임시저장/취소). 폼 영역은 스크롤, 하단 액션바 fixed.

기본 필드

  • 제목 (필수)
  • 분류 / 유형
  • 상태 (기본값 '작성중')
  • 담당자
  • 본문 (리치텍스트)
  • 첨부 파일
  • (Board Builder 커스텀 필드)

기본 액션

  • 저장 → 상세 이동
  • 임시저장 (로컬 또는 서버)
  • 취소 → 확인 후 목록 이동
  • 검토 요청 (저장 완료 후)
  • 첨부 파일 추가/삭제

상태값 / 배지

필수(*) 유효성 오류 임시저장됨 저장 성공

필수 필드는 레이블에 * 표시. 저장 실패 시 인라인 오류 메시지.

권한 포인트

  • 작성 진입 권한
  • 첨부 파일 업로드 권한
  • 저장 권한
  • 검토 요청 권한

연결 산출물

  • 신규 요구사항 · 화면설계서 · 정책
  • 저장 후: T07 Basic Detail View
  • 복합 폼: T12 Multi-section Edit Form

Board Builder 연계

Board Builder 필드 정의 → 폼 필드 세트로 직접 반영. 커스텀 필드 추가 시 폼에 동적 추가됨.
→ Board Builder

빈 상태

폼 초기 상태: 빈 입력 필드(기본값 설정 필드는 채워진 상태). 임시저장 데이터 있을 경우 "임시저장된 내용을 불러올까요?" 팝업.

오류 상태

필수 미입력: "제목을 입력하세요."(인라인)
저장 실패: "저장 중 오류가 발생했습니다. 다시 시도하세요."
임시저장 실패: "임시저장에 실패했습니다."

모바일 / 좁은 화면

• 1열 폼 레이아웃 (2열 → 1열)
• 필수 필드 우선 배치
• 하단 액션바 fixed
• 리치텍스트 에디터 툴바 컴팩트 모드
• 첨부 버튼 전체 너비

#QA 항목확인 기준
1필수 필드 유효성제목 빈 상태 저장 시 인라인 오류 표시
2임시저장임시저장 후 새로고침 시 복구 팝업
3취소 확인내용 입력 후 취소 시 "변경사항을 저장하지 않겠습니까?" 팝업
4저장 후 이동저장 성공 시 생성된 레코드 상세로 이동
5첨부 파일파일 추가·미리보기·삭제 동작
6모바일 1열980px 이하 1열 폼, 하단 액션바 fixed
T12

Multi-section Edit Form

Create / Edit FormBoard Builder 부분 연계

목적

기본정보·상세내용·연결정보·권한·첨부 등 다중 섹션으로 구성된 복합 문서를 수정하는 폼 화면. 섹션별 저장과 전체 저장을 모두 지원한다.

적용 화면 예시

화면설계서 수정 · 기능정의서 수정 · 복합 산출물 수정 · 정책 정의서 수정

기본 레이아웃

섹션 탭 또는 아코디언(기본정보/상세/연결/권한/첨부) → 각 섹션 내 폼 필드 → 섹션 저장 버튼 → 하단 전체 저장/취소.

기본 섹션 / 필드

  • 기본정보: 제목·유형·상태·담당자
  • 상세내용: 본문·연결 항목·태그
  • 연결정보: 관련 문서·참조 링크
  • 권한/검토: 검토자·승인자 지정
  • 첨부: 파일 목록·버전 관리

기본 액션

  • 섹션 저장 (섹션별 독립)
  • 전체 저장
  • 취소 → 변경사항 확인 팝업
  • 미리보기 (읽기 모드 전환)
  • 변경사항 확인 (미저장 이탈 경고)

상태값 / 배지

섹션 변경됨 섹션 저장됨 저장 실패 미저장 이탈

섹션 탭/헤더에 변경됨(·) 표시. 미저장 이탈 시 경고 팝업.

권한 포인트

  • 섹션별 편집 권한 (섹션별 분리)
  • 저장 권한
  • 검토자·승인자 지정 권한
  • 미리보기 (열람 권한)

연결 산출물

  • 화면설계서 · 기능정의서 · 정책
  • 저장 후: T07 Basic Detail View
  • 신규 작성: T11 Basic Create Form

Board Builder 연계

기본정보 섹션의 필드 일부가 Board Builder 필드 구조와 연결. 연결정보·권한/검토 섹션은 Board Builder 범위 외 독립 구성.

빈 상태

Cross-template Standards — 10개 공통 기준

상태 배지 공통 기준

  • 완료 — 모든 작업/검토/승인 완료
  • 검토중 / 승인요청 — 진행중 상태
  • 작성중 — 편집 가능 상태
  • 반려 — 반려/오류 상태
  • 보류 / 미시작 — 비활성 상태

상태 배지 색상 기준은 모든 10개 템플릿에 동일 적용.

권한 단위 공통 기준

  • 화면 진입 권한 (최소 단위)
  • 행/문서 열람 권한
  • 등록 / 작성 권한
  • 수정 / 편집 권한
  • 삭제 / 숨김 권한
  • 내보내기 권한
  • 관리자 전용 (설정·권한 관리)

검토 / 승인 / 댓글 연결

작성형(T11·T12)·상세형(T07) 템플릿은 검토 요청·승인 요청·댓글/멘션(D그룹 공통 레이어) 액션을 기본 액션에 포함한다. 승인 흐름은 T08 Approval Detail View로 연결된다.
→ STAM Sitemap & IA Overview

변경이력 연결

수정 가능한 템플릿(T07·T11·T12)은 변경자·변경일·변경 항목 이력(T09 Change History Detail 패턴)과 연결한다. 이력은 접힘/펼침 섹션으로 제공하며 롤백은 권한자 요청 방식으로 처리한다.

첨부 / 파일 공통 기준

파일 첨부 지원 템플릿(T07·T11·T12): 파일명·형식·크기·업로더·업로드일 표시. 다운로드·미리보기·삭제 권한 분리. 버전 관리가 필요한 경우 T10 Attachment Detail View 연결.

빈 상태 공통 기준

• 데이터 없음: 아이콘 + 설명 텍스트 + 액션 버튼
• 검색 결과 없음: 조건 초기화 버튼 포함
• 권한 없음: "접근 권한이 없습니다." 별도 상태
• 로딩 중: 스켈레톤 UI 표시

오류 상태 공통 기준

• 네트워크 오류 / 서버 오류: 재시도 버튼 포함
• 404(삭제됨): 삭제 안내 메시지
• 403(권한없음): 권한 안내 메시지
• 인라인 유효성 오류: 필드별 개별 표시

내보내기 공통 기준

목록형(T01·T02·T03·T05): Excel·CSV 기준. 상세형(T07·T08): PDF·인쇄. 내보내기 권한은 독립적으로 관리. 내보내기 범위는 현재 필터/조건 기준.

모바일 공통 기준

• 980px 이하: 테이블 → 카드 전환(T01·T02·T03)
• 고정 컬럼 + 가로 스크롤(T05)
• 하단 액션바 fixed(T11·T12·T26·T27)
• 승인/저장 등 핵심 액션 최우선 노출(T08·T11)
• 드래그 → 버튼 대체(T27)

로딩 / 스켈레톤 공통 기준

데이터 로딩 중에는 스켈레톤 UI 표시. 목록 로딩: 행 스켈레톤. 상세/폼 로딩: 섹션 스켈레톤. 저장 중: 버튼 비활성화 + 로딩 스피너.

부분 연계 템플릿 (2종)

  • T02 Advanced Search List — 게시판별 검색 조건 필드
  • T12 Multi-section Edit Form — 기본정보 섹션 일부 필드

T03 Grouped Work List도 그룹 카테고리 필드가 부분 연계되나, Top 10 상세화 기준에서는 독립 운영 가능.

역할 분담

Board Builder는 필드/게시판 구조를 정의하고, Top 10 템플릿은 그 구조를 화면 UX 산출물로 표현한다. 게시판 구조 정의 → 템플릿 선택 → 화면설계서 작성 흐름으로 보완 관계.

주의 사항

현재 Board Builder는 preview / local 기준이며 실제 저장·배포 기능이 아니다. 입력 결과는 localStorage 기준 Preview이고 Firestore/API/fetch 호출이 없는 상태를 유지한다. 이 문서는 Board Builder 제품 파일을 수정하지 않으며 참조 링크만 제공한다.
