00

목적 · 원칙

stam.board-toolbar.css는 보드 화면 목록 상단 Toolbar의 구조(레이아웃) 기준을 정의합니다.

Toolbar 기본 노출 — 딱 3가지만
  • 검색바 (stam-board-search)
  • 필터 패널 열기 버튼 (stam-board-filter-trigger)
  • 삭제 버튼 (stam-board-delete)

구조 원칙
  • 상태/유형/우선순위/담당자 등 세부 조건 chip은 Toolbar에 직접 배치하지 않는다 — Board Filter Panel 내부에 배치
  • stam-board-toolbar-extra는 Special Board 전용 (Gantt 접기, Focus View 등 화면 조작 버튼) — 일반 보드에서 사용 금지
  • CSS 로드 순서: stam.components.css → stam.buttons.css → stam.board-toolbar.css → 페이지 CSS
  • 페이지 CSS의 toolbar flex 레이아웃은 stam-board-toolbar가 담당 — 페이지 CSS에서 display:flex 제거
  • 필터 조건 UI는 STAM Board Filter 컴포넌트가 담당 — Toolbar는 trigger만 제공
01

표준 구조: 검색 + 필터 Trigger + 삭제

일반 목록형 보드의 Toolbar는 아래 구조가 전부입니다.
stam-board-toolbar-base 안에 검색 → 필터 버튼 → 삭제 버튼 순서로 배치합니다.
필터 조건 chip은 Toolbar에 두지 않습니다 — Board Filter Panel에서 처리합니다.

02

Board Filter와의 역할 분리

Board Toolbar와 Board Filter는 역할이 명확히 분리됩니다.
컴포넌트역할필터 조건 UI 소유
Board Toolbar 검색 · 필터 패널 열기 trigger · 삭제 ❌ 직접 보여주지 않음
Board Filter 필터 조건 Panel UI (chip toggle / 초기화 / 적용) ✅ 담당
Board Toolbar의 역할 (Toolbar에 있어야 할 것)
· 검색바 — 키워드 실시간 필터링
· 필터 버튼 (stam-board-filter-trigger) — Board Filter Panel을 여는 trigger
· 삭제 버튼 — 선택 항목 삭제 액션
· Special Board 전용 모드/뷰 전환 버튼 (extra에만)
Board Toolbar에 있으면 안 되는 것
· 상태 chip / 유형 chip / 우선순위 chip / 담당자 chip / LV1 chip / FO/BO chip
· 어떤 형태의 필터 조건 popover나 dropdown
· 화면별 자체 filter IIFE 또는 filter popover 신규 구현
→ 이 모든 것은 Board Filter Panel(stam.board-filter.css + stam.board-filter.js)이 담당합니다.
Board Filter 상세 사용법 → Board Filter Guide v1.1
03

extra 영역 — Special Board 전용

⚠️ 일반 목록형 보드에서 사용 금지
stam-board-toolbar-extra는 일반 목록형 보드(requirements, menu-screen-list 등)에서 사용하지 않습니다.
일반 보드의 필터 조건은 Board Filter Panel을 사용합니다.
허용되는 extra 사용 예 (Special Board 한정)
  • WBS: Gantt 접기 / 그룹 접기 / 컬럼 설정 / Focus View 진입
  • 특수 보드: 뷰 모드 전환 (리스트/칸반 등)
  • 특수 보드: 특수 표시 모드 토글

extra에 절대 넣으면 안 되는 것
  • 상태 chip, 유형 chip, 담당자 chip, 우선순위 chip
  • 일반 필터 조건 버튼 어떤 형태든
WBS 참고 예 (Special Board)
WBS는 Legacy/Special Board 예외입니다. 신규 일반 보드에서 이 패턴을 따라 만들지 마세요.
04

우측 right 구조

stam-board-toolbar-right는 오른쪽 끝 액션 영역입니다.
등록, 내보내기 등 화면 수준 액션을 배치합니다. 우측 액션이 없으면 빈 div로 둡니다.
필터 조건 chip은 right에도 배치하지 않습니다.
05

삭제 버튼 위치 기준

기준내용
위치stam-board-toolbar-base 안, 필터 버튼 바로 우측
기본 상태선택 항목이 없을 때 disabled 상태 유지
기존 id/classid="rq-del-btn" 등 기존 JS selector 유지
공통 class 추가stam-btn stam-btn-danger stam-board-delete 병행 추가
로직삭제 로직 신규 구현 금지 — 기존 핸들러만 유지
금지삭제 버튼을 chip 필터보다 왼쪽에 두거나 right에 배치하지 말 것

금지 패턴

① Toolbar에 세부 조건 chip 직접 배치
상태 chip / 유형 chip / 담당자 chip / 우선순위 chip / LV1 chip / FO/BO chip 등
어떤 형태의 필터 조건 버튼도 Toolbar(extra 포함)에 직접 배치하지 않습니다.
→ 모든 필터 조건은 STAM Board Filter Panel 내부에 배치합니다.
② 화면별 자체 filter popover / IIFE 신규 구현
window.STAM.boardFilter.init()을 사용하지 않고 화면별로 자체 필터 패널을 새로 만드는 것은 금지입니다.
기존 WBS / screen-specification의 자체 필터는 Legacy 예외이며, 신규 보드에서 이 패턴을 따라 만들지 않습니다.
③ 일반 보드에서 extra 사용
일반 목록형 보드에서 stam-board-toolbar-extra를 사용하지 않습니다.
extra가 필요하다면 Special Board 여부를 먼저 검토합니다.
④ 검색 바로 옆에 chip이 먼저 오는 구조
검색바 → 상태 → 유형 → 우선순위 → 삭제 구조는 금지입니다.
올바른 순서: 검색바 → 필터 버튼 → 삭제
⑤ 삭제 버튼 누락
삭제 기능이 있는 모든 보드의 Toolbar에는 stam-board-delete 마커가 있는 삭제 버튼이 base 안에 있어야 합니다.
삭제 로직이 아직 없으면 disabled 상태로 두어도 됩니다.
⑥ 우측 액션을 좌측 base / extra에 섞기
등록 버튼 등 우측 전용 액션을 stam-board-toolbar-base나 stam-board-toolbar-extra에 넣지 않습니다.
우측 액션은 반드시 stam-board-toolbar-right 안에 위치합니다.
⑦ stam-board-filter-trigger에 stam-btn 적용
필터 trigger 버튼은 stam-board-filter-trigger class만 사용합니다.
stam-btn, stam-btn-ghost 등 버튼 variant를 중복 적용하지 않습니다 — 스타일 충돌 발생.
09

신규 보드 Toolbar 체크리스트

신규 일반 목록형 보드 생성 시 Toolbar 관련 체크 항목입니다.
  • stam.board-toolbar.css 로드 확인
  • stam.board-filter.css 로드 확인
  • stam.board-filter.js 로드 확인
  • Toolbar base 안에 검색 + stam-board-filter-trigger + 삭제만 배치
  • 상태/유형/담당자 chip이 Toolbar base/extra에 없음 확인
  • stam-board-toolbar-extra 미사용 (특수 보드가 아닌 경우)
  • 필터 조건은 window.STAM.boardFilter.init({ groups })로 구성
  • 화면별 filter IIFE 신규 작성 없음 확인
  • 페이지 CSS에서 toolbar display:flex 제거 확인
  • stam-board-filter-trigger에 stam-btn 중복 미적용 확인
