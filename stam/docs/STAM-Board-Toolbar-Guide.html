00

목적 · 원칙

stam.board-toolbar.css는 보드 화면 목록 상단 Toolbar의 구조(레이아웃) 기준을 정의합니다.
검색 / 삭제 / 화면별 필터 / 우측 액션을 일관된 위치에 배치합니다.

적용 원칙
  • 기존 page-level class(rq-toolbar, msl-toolbar 등)는 유지하고 stam-board-toolbar를 병행 추가
  • JS selector가 기존 class에 의존하므로 기존 class를 절대 삭제하지 않음
  • CSS 로드 순서: stam.components.css → stam.buttons.css → stam.board-toolbar.css → 페이지 CSS
  • 페이지 CSS의 toolbar flex 레이아웃은 stam-board-toolbar가 담당하므로 제거
  • 버튼 클릭 JS 로직 변경 금지 — 기존 id/class 유지
  • Toolbar 기본 노출은 검색 + 필터 + 삭제만 — 화면별 세부 필터(상태/유형/담당자 등)는 Toolbar에 직접 나열하지 않고 필터 패널 내부로 이동
  • stam-board-toolbar-extra는 Gantt 접기 / 컬럼 설정 / Focus View처럼 화면 조작용 버튼에만 사용
  • 상태/유형/담당자 등 조건 필터는 extra에 직접 노출하지 않고 stam-filter-panel에 배치
01

좌측 base 구조: 검색 + 필터 + 삭제

stam-board-toolbar-base는 항상 검색바 → 구분선(선택) → 삭제 버튼 순서를 유지합니다.
WBS처럼 필터 패널 토글 버튼이 있는 경우 검색바와 삭제 사이에 추가합니다.
02

좌측 extra 구조: 화면별 추가 필터/버튼

stam-board-toolbar-extra에 화면별 필터 chip 버튼(상태, 유형, 담당자 등)을 배치합니다.
base와 extra는 stam-board-toolbar-left 안에서 좌→우 순서로 배열됩니다.
03

우측 right 구조: 초기화 / 내 담당 / 리스크 / 등록

stam-board-toolbar-right는 오른쪽 끝 액션 영역입니다.
초기화, 내 담당, 리스크 토글, 등록 버튼 등 우측 전용 액션을 배치합니다.
화면에 우측 액션이 없으면 빈 div로 두면 됩니다.
04

삭제 버튼 위치 기준

기준내용

CSS 로드 순서
<link rel="stylesheet" href="../../css/stam.components.css"> <link rel="stylesheet" href="../../css/stam.buttons.css"> <link rel="stylesheet" href="../../css/stam.board-toolbar.css"> <!-- ← 여기 --> <link rel="stylesheet" href="../../css/stam.{page}.css">
페이지 CSS 주의사항
stam-board-toolbar가 flex 레이아웃을 담당하므로
페이지 CSS의 .{page}-toolbar에서 display:flex, align-items, gap 속성을 제거하고
margin-bottom만 남깁니다.
/* 변경 전 */ .rq-toolbar { display:flex;align-items:center;gap:7px;margin-bottom:8px;flex-wrap:wrap } /* 변경 후 — flex 레이아웃은 stam-board-toolbar가 담당 */ .rq-toolbar { margin-bottom:8px }

07

금지 패턴

① 검색 바로 옆에 상태/유형 등 필드 필터가 먼저 오는 구조
검색바 → 상태 → 유형 → 우선순위 → ... → 삭제 구조는 금지.
올바른 구조: 검색바 → 삭제 (base), 그 다음에 상태 → 유형 → ... (extra)
② 삭제 버튼 누락
삭제 기능이 있는 모든 보드 화면의 Toolbar에는 stam-board-delete 마커가 있는 삭제 버튼이 base 안에 있어야 함.
현재 삭제 로직이 없으면 disabled 상태로 표시만 해도 됨.
③ 우측 액션을 좌측 base에 섞기
초기화, 내 담당, 리스크 등 우측 전용 액션을 stam-board-toolbar-base나 stam-board-toolbar-extra에 넣으면 안 됨.
우측 액션은 반드시 stam-board-toolbar-right 안에 위치.
④ 페이지 Toolbar에서 stam-board-toolbar 없이 임의 layout 구현
신규 보드 화면의 Toolbar는 반드시 stam-board-toolbar를 루트에 추가하고 base/extra/right 구조를 따름.
페이지 CSS만으로 임의 Toolbar layout 구현 금지.
