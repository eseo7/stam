Left Nav 구성 요소

  • 상단: 프로젝트명/브랜드 영역 (.po-proj-badge) — flex-shrink: 0 고정
  • 검색: 메뉴명·그룹명·메뉴코드·상태 badge 기준 실시간 필터링
  • 그룹: 7개 (A–G) 아코디언 접기/펼치기, 그룹 count badge
  • 메뉴 아이템: active 상태 (.gitem.on), dot indicator, status badge
  • 검색 결과 없음: .po-nav-empty 표시 — 메뉴·그룹명 검색 안내
  • 하단 footer: 사용자/프로젝트 정보 고정 — 메뉴 overflow 시에도 덮히지 않음

08 Content Layout Standard 기준 고정
토큰값용도비고
--content-max1560px프로젝트 내부 본문 최대 폭1440·1600·1920 모두 대응
--content-pad-x32px (한쪽)본문 좌우 여백합산 64px
--grid-gap10px카드 그리드 gap--content-gutter 대체
--content-focus-max1760pxFocus View 모드 최대 폭Wide 모니터 대응
--content-gutter미사용삭제됨--grid-gap 으로 대체
  • 페이지 레벨 (.po-main) 가로 스크롤 금지 — overflow-x: hidden
  • 컬럼이 많은 화면은 테이블 내부 래퍼에만 overflow-x: auto 허용
  • 본문 폭 패턴: width: min(calc(100% - var(--content-pad-x) * 2), var(--content-max))
  • 1440 / 1600 / 1920 폭 모두에서 본문이 답답하지 않도록 설계

10 Focus View / Full View Standard 가이드
좌측 메뉴를 숨기고 본문을 최대화하는 모드. 테이블·보드·간트차트 등 넓게 봐야 하는 업무, 빔프로젝터 공유·회의·고객 리뷰 시 활용.

Focus View 필수 적용 우선 화면

  • B3 WBS 작업
  • C8 오픈 시나리오
  • B8 테이블 정의서
  • B9 DB 정보
  • B10 서버 정보
  • B11 접근/보안 정보
  • C5 테스트 결과 대시보드
  • C6 결함 대시보드

Focus View 필수 포함 사항

  • 진입 버튼 — Page Header 우측
  • 종료 버튼 — Focus 상태에서
  • 좌측 메뉴 숨김 (.po-sidebar display: none)
  • 본문 최대화 — --content-focus-max (1760px)
  • 페이지 레벨 가로 스크롤 금지 유지
  • 내부 테이블 영역 overflow 처리
  • 현재 필터/검색 조건 유지

21 Page Layout Contract 기준 고정
STAM 화면 공통 width 기준을 outer page wrapper 한 곳에서만 관리한다. 화면별로 width / max-width / padding을 감으로 직접 넣지 않는다. 이 계약을 어기는 경우 반드시 주석 또는 이 문서에 예외 사유를 남겨야 한다.

Width Ownership Rule

width는 outer page wrapper가 단독 소유한다. inner wrapper · 섹션 · 카드 · 테이블 래퍼 등 내부 요소는 width를 재지정하지 않는다. --content-max · --content-pad-x 두 토큰이 유일한 width 결정 소스이다.
소유자역할width 권한
outer page wrapper
(예: .screen-spec-page)		 전체 본문의 최대 폭·여백·가운데 정렬 담당 ✔ 소유 — 유일한 width 결정 지점
inner wrapper
(섹션 묶음 등)		 레이아웃 내부 그룹핑 전용 ✖ 소유 금지
카드 · 테이블 · 폼 등 콘텐츠 자체 너비 ✖ max-width / 고정 px 직접 기입 금지

Outer Wrapper / Inner Wrapper Rule

Outer Wrapper — 필수 기준

width: 100%; max-width: var(--content-max, 1560px); margin: 0 auto; padding: 20px var(--content-pad-x, 32px) 64px; box-sizing: border-box;
  • 토큰이 없는 환경에서도 fallback 값으로 동작 보장
  • margin: 0 auto 로 가운데 정렬 — 개별 화면에서 재선언 금지
  • box-sizing: border-box 필수 — padding이 width 밖으로 넘치지 않도록

Inner Wrapper — 허용 범위

width: 100%; max-width: none; margin: 0;
  • 섹션 묶음·카드 그룹·필터 영역 등 내부 구분 목적으로만 사용
  • max-width 재설정 금지 — outer wrapper가 이미 제한함
  • 개별 좌우 padding 추가 금지 — outer wrapper의 --content-pad-x가 담당

Do / Don't

✔ Do✖ Don't
max-width: var(--content-max) — outer wrapper에서 max-width: 1360px · 1440px · 1560px 직접 기입
padding: 20px var(--content-pad-x, 32px) 64px padding: 20px 40px · padding: 20px 32px 직접 기입
inner wrapper: width: 100%; max-width: none inner wrapper에 max-width: 1200px 등 재설정
예외 화면: 주석 또는 이 문서에 사유 명시 후 사용 예외 사유 없이 임의 width 값 사용
Focus View 최대 폭: var(--content-focus-max, 1760px) Focus View에서 일반 본문과 같은 --content-max 재사용
토큰 fallback 형식: var(--content-max, 1560px) 토큰 없이 1560px 단독 기입

예외 화면 처리 원칙

예외가 필요한 경우 아래 형식으로 CSS 주석을 남기고 이 문서 예외 목록에도 추가한다. /* [Layout Exception] 화면명: {이유} width: {값} — 기준 outer wrapper와 다른 이유를 1줄로 기재 승인: PR #{번호} */
