STAM Custom Select Contract v1

STAM 의 모든 커스텀 select / dropdown / multiselect 가 화면별 전용 select 스타일을 새로 만들지 않고 공통 .stam-cs-* 구조를 사용하도록 강제하는 기준 문서.

공통 CSS: stam.custom-select.css 기준 PR: #97 (공통 패턴) · #98 / #99 / #100 (적용) 버전: v1
Related: Custom Select — Bare Preview Board List Contract v1 Form Controls Guide Component Guide Docs Index
Section 00 · Custom Select 핵심 원칙
  1. 새 select 는 화면별 .wbs-sel-* · .ss-sel-* · .rq-sel-* · .fn-cs-* 같은 전용 시각 class 를 새로 만들지 않는다.
  2. 공통 class .stam-cs-* 를 기본으로 사용한다. 시각 스타일(배경 · 테두리 · hover · selected · icon rotate · placeholder · dark)은 모두 공통 CSS stam.custom-select.css 가 담당한다.
  3. 화면별 CSS 는 위치 · z-index · portal mount 특성 · max-height · 옵션 개수에 따른 menu height · 화면 고유 width 같은 구조 보정만 허용한다.
  4. legacy namespace class(.wbs-sel-* 등)는 구조/스크립트 호환용으로만 유지하고, 시각 스타일을 다시 정의하지 않는다.
Section 01 · 필수 공통 class
Class역할적용 위치
.stam-cscustom select 루트 (open/disabled 상태 보유)래퍼 <div>
.stam-cs-trigger현재 값 + caret 을 보여주는 클릭 트리거<button type="button">
.stam-cs-value현재 선택 값 텍스트트리거 내 <span>
.stam-cs-value.is-placeholder값 없음(placeholder) 상태 — t3 색트리거 내 <span>
.stam-cs-iconcaret 아이콘 (open 시 회전)트리거 내 SVG
.stam-cs-menu옵션 패널 (listbox)<div role="listbox">
.stam-cs-opt개별 옵션<div role="option">
.stam-cs-opt.is-sel선택된 옵션 — stam-soft 배경 + check 표시<div role="option">
.stam-cs-checkcheck icon 슬롯 (선택 시에만 visible · 항상 고정폭)옵션 내 <span>
.stam-cs-otext옵션 텍스트 (긴 텍스트 ellipsis)옵션 내 <span>

* 화면별 namespace class(.wbs-sel-box, .wbs-sel-menu 등)를 부가적으로 병행하는 것은 허용. 단 위 공통 class 는 반드시 포함하고, 시각 스타일은 공통 CSS 가 담당한다. CSS 파일: stam/css/stam.custom-select.css. 시각 기준은 Custom Select Bare Preview 참조.

Section 02 · 상태 class
대상Class의미
root .stam-cs.is-openpanel 열림 + caret 회전 + ring
root .stam-cs.is-uppanel 을 위로 펼침 (clipping 보정, 선택적)
root .stam-cs.is-disabled전체 비활성 (opacity .5 + pointer-events none) — 필요 시
option .stam-cs-opt.is-sel선택됨 — stam-soft 배경 + stam 텍스트 + 700 + check 표시
option .stam-cs-opt.is-active키보드 강조 — hover 와 동일 톤 (bg-sur2 / t1)
option .stam-cs-opt.is-placeholderplaceholder 옵션 — 강조색 금지 (t3 유지)
option .stam-cs-opt.is-disabled개별 옵션 비활성 (opacity .4) — 필요 시
value .stam-cs-value.is-placeholder트리거 값이 placeholder 상태

* 핵심 3종: root open = .is-open, selected option = .is-sel, placeholder = .is-placeholder. 이 3종은 모든 select 에서 반드시 적용한다. disabled 는 필요 시 .is-disabled.

Section 03 · 표준 HTML 구조 예시

1) single select — 값 있음 / placeholder

<!-- 값 있음 -->
<div class="stam-cs">
  <button type="button" class="stam-cs-trigger" data-stam-cs-toggle
          aria-haspopup="listbox" aria-expanded="false">
    <span class="stam-cs-value">박PM</span>
    <svg class="stam-cs-icon" ...>caret</svg>
  </button>
  <div class="stam-cs-menu" role="listbox">
    <div class="stam-cs-opt is-sel" role="option" aria-selected="true">
      <span class="stam-cs-check"><svg ...>check</svg></span>
      <span class="stam-cs-otext">박PM</span>
    </div>
    <div class="stam-cs-opt" role="option">
      <span class="stam-cs-check"><svg ...>check</svg></span>
      <span class="stam-cs-otext">김기획</span>
    </div>
  </div>
</div>

<!-- placeholder (값 없음) -->
<div class="stam-cs">
  <button type="button" class="stam-cs-trigger" aria-haspopup="listbox" aria-expanded="false">
    <span class="stam-cs-value is-placeholder">— 담당자 선택 —</span>
    <svg class="stam-cs-icon" ...>caret</svg>
  </button>
  <div class="stam-cs-menu" role="listbox"> ... </div>
</div>

2) option selected 상태 — 선택 옵션에 .is-sel + aria-selected="true", check 슬롯은 모든 옵션에 동일하게 두고 .is-sel 일 때만 보인다.

<div class="stam-cs-opt is-sel" role="option" aria-selected="true">
  <span class="stam-cs-check"><svg ...>check</svg></span>
  <span class="stam-cs-otext">설계</span>
</div>

3) multiselect — 동일 구조. 여러 옵션에 .is-sel 부여, 트리거 값은 요약 텍스트("3개 선택" 등)로 표시. 시각 스타일은 single 과 동일하게 공통 class 사용.

<div class="stam-cs" data-multiselect>
  <button type="button" class="stam-cs-trigger" aria-haspopup="listbox" aria-expanded="false">
    <span class="stam-cs-value">김기획, 박PM (+1)</span>
    <svg class="stam-cs-icon" ...>caret</svg>
  </button>
  <div class="stam-cs-menu" role="listbox" aria-multiselectable="true">
    <div class="stam-cs-opt is-sel" role="option" aria-selected="true">
      <span class="stam-cs-check"><svg ...>check</svg></span>
      <span class="stam-cs-otext">김기획</span>
    </div>
    <div class="stam-cs-opt is-sel" role="option" aria-selected="true">
      <span class="stam-cs-check"><svg ...>check</svg></span>
      <span class="stam-cs-otext">박PM</span>
    </div>
    <div class="stam-cs-opt" role="option">
      <span class="stam-cs-check"><svg ...>check</svg></span>
      <span class="stam-cs-otext">최개발</span>
    </div>
  </div>
</div>

4) portal-mount dropdown — menu 를 body/portal 로 이동해 띄우는 경우. 공통 class 는 그대로 유지하고, 위치 좌표 · z-index · display:block override 만 화면별 inline/CSS 로 보정한다 (WBS 사례 = Section 06).

<!-- 트리거는 화면 안에, menu 는 portal 로 이동 -->
<div class="stam-cs wbs-sel-box">
  <button type="button" class="stam-cs-trigger" aria-haspopup="listbox" aria-expanded="false">
    <span class="stam-cs-value is-placeholder">— 단계 선택 —</span>
    <svg class="stam-cs-icon" ...>caret</svg>
  </button>
</div>

<!-- portal mount 위치로 이동된 menu (좌표/z-index 만 화면별 보정) -->
<div class="stam-cs-menu wbs-sel-menu" role="listbox"
     style="display:block; top:...px; left:...px; width:...px;">
  <div class="stam-cs-opt" role="option">
    <span class="stam-cs-check"><svg ...>check</svg></span>
    <span class="stam-cs-otext">착수</span>
  </div>
</div>

* caret SVG / check SVG 의 구체 path 는 Bare Preview 의 마크업을 그대로 복사해 사용한다.

Section 04 · 화면별 CSS 허용/금지

✅ 화면별 CSS 에서 허용 (구조 보정)

  • portal mount 위치 보정 (top / left / transform 등 좌표)
  • z-index (다른 UI 위로 띄우기)
  • max-height (긴 옵션 목록 스크롤)
  • 옵션 개수에 따른 menu height
  • 화면 고유 width (트리거 / menu 폭)

❌ 화면별 CSS 에서 금지 (시각 재정의)

  • trigger 배경 / 테두리 / hover 재정의
  • option hover 색상 재정의
  • selected option 색상 재정의
  • icon rotate 스타일 재정의
  • dark mode 색상 재정의
  • placeholder 색상 재정의
Section 05 · WBS 적용 사례 — PR #100

WBS custom dropdown 은 PR #100 에서 공통 .stam-cs-* 패턴을 병행 적용한 기준 사례이다.

  • WBS 는 기존 .wbs-sel-* legacy class 와 공통 .stam-cs-* class 를 병행 적용한다. legacy class 는 구조/스크립트 호환용으로만 유지한다.
  • 루트 .wbs-selectbox 에 .stam-cs, 트리거 <button> 에 .stam-cs-trigger, 값 span 에 .stam-cs-value(+ 값 없을 때 .is-placeholder), caret 에 .stam-cs-icon, 옵션에 .stam-cs-opt(+ 선택 시 .is-sel), check 슬롯에 .stam-cs-check, 텍스트에 .stam-cs-otext, menu 에 .stam-cs-menu 병행.
  • open 상태는 박스에 .is-open 토글 → 공통 .stam-cs.is-open selector 매칭으로 ring / caret 회전이 공통 CSS 로 적용.
  • WBS 는 portal-mount 특성 때문에 .wbs-sel-menu 의 z-index / padding / max-height 일부만 유지한다. 공통 .stam-cs-menu 가 기본 display:none + right:0 이므로, portal 마운트 시 inline override(display:block + 절대 좌표 top/left/width)로 위치만 보정.
  • WBS 전용 시각 CSS(.wbs-sel base / hover / open / placeholder / .wbs-sel-opt hover / selected / dark 변형 등 약 38줄)는 제거되고, 시각 스타일은 전부 .stam-cs-* 공통 CSS 가 담당한다.
  • WBS portal-mount select smoke PASS — 키보드 / click outside / portal 위치 / 선택 동작 확인 완료.

* 즉, WBS 의 잔여 화면별 CSS 는 Section 04 의 ✅ 허용 항목(z-index · max-height · portal 위치 · padding)뿐이며, ❌ 금지 항목(시각 재정의)은 없다. WBS 는 PR #100 기준을 유지하고 추가 변경은 별도 QA 후 진행한다.

Section 06 · 새 select 생성 체크리스트
  1. stam.custom-select.css 로드 여부 (페이지 <head> · 페이지 CSS 보다 앞)
  2. 공통 .stam-cs-* class(루트 / trigger / value / icon / menu / opt / check / otext) 적용 여부
  3. 상태 적용 여부 — root .is-open / option .is-sel / placeholder .is-placeholder (필요 시 .is-disabled)
  4. keyboard 조작 (위/아래 · Enter · Esc) / click outside 닫힘 / portal 마운트 여부 확인
  5. Bare Preview(STAM-Common-Custom-Select-Preview.html) 또는 실제 화면과 육안 비교 여부
  6. 화면별 CSS 에 select 시각 스타일 재정의가 없는지 확인 (Section 04 ❌ 항목)
Section 07 · Do / Don't

✅ Do

  • 공통 .stam-cs-* class 병행 적용
  • legacy class 는 구조 / 스크립트 호환용으로만 유지
  • 시각 스타일은 공통 CSS(stam.custom-select.css)에서 관리
  • 핵심 상태 3종(.is-open / .is-sel / .is-placeholder) 적용
  • Bare Preview 기준과 육안 비교

❌ Don't

  • 새 화면에서 전용 select 디자인 재구현 (.wbs-sel-* · .ss-sel-* · .rq-sel-* 시각 class 신설)
  • 화면별 hover / selected / dark style 생성
  • trigger / option / icon / placeholder 색상 화면별 재정의
  • computed style 만 보고 PASS 처리 (실 인터랙션 미검증)
