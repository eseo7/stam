01 Overview — 문서 목적 목적

STAM WBS 는 평소에는 작업자가 자기 화면에서 사용하지만, 주간 회의 · 진척 점검 · 리스크 점검 · 외부 보고에서는 회의실 빔 프로젝터 · 대형 TV · 화면 공유로 띄워 공유된다. 이때 작업자용 정보 밀도와 편집 도구가 그대로 노출되면 글자가 작고 컬럼이 많아 회의가 산만해진다. 본 문서는 이 발표 시나리오를 위한 별도 모드의 기준을 확정한다.

02 Presentation Mode Purpose Purpose

발표 모드의 목표는 한눈에 현재 진행 상황을 파악하고, 지연 / 위험 업무를 즉시 식별하며, 담당자별 이슈를 빠르게 확인해 회의 중 다음 액션을 결정하는 데 있다. 편집 기능은 우선순위에서 뒤로 빠진다.

발표 모드가 달성해야 하는 것
  • 0.5초 안에 전체 진행 / 지연 / 위험 수치 인지
  • 3초 안에 가장 급한 1 ~ 2 건 확인
  • 한 화면 안에서 회의 진행자가 다음 액션 결정 가능
  • 참가자가 회의실 뒤에서도 글자가 읽힘 (3 ~ 6m)
  • 회의 중 편집 충동을 의도적으로 차단 — 메모는 별도 후보
발표 모드에서 일어나면 안 되는 것
  • 회의 중 누가 실수로 row 를 편집하거나 삭제
  • 작은 icon-only action 버튼이 행마다 노출돼 시선 분산
  • 10개 이상의 컬럼이 한 화면에 압축되어 글자가 8 ~ 10px
  • 지연 / 위험이 색상만으로 표현되어 빔 프로젝터에서 묻힘
  • 필터 적용 후에도 dim 처리된 행이 같이 보여 시선 산만
03 Entry / Exit Pattern UX

발표 모드는 자동 진입 없음 · 사용자 명시적 전환만 허용한다. 일반 사용자가 평소 작업 중 의도치 않게 진입하지 않도록 진입 지점을 분리하고, 종료는 ESC · 상단 발표 종료 버튼 · view mode 기본 선택 3가지로 보장한다.

AWBS Toolbar · 발표 모드 버튼recommend

툴바 우측에 발표 모드 버튼을 두고 클릭 시 view mode 가 presentation 으로 전환된다. 가장 직관적이며 사용자가 클릭하므로 사고 진입이 없다.

  • 발견 쉬움 — 누구나 한 번에 찾음
  • WBS 안에서 진입 — 컨텍스트 유지
  • 회의 시작 시 한 번만 누르면 됨
  • 툴바 영역 1슬롯 점유
BView Mode Segmented · 기본 / 압축 / 발표

WBS 상단에 view 전환 segmented control 을 두고 기본 · 압축 · 발표 3택. 작업자가 평소 압축 모드를 자주 쓰는 경우 효과적.

  • 여러 view 를 한 곳에서 관리
  • 토글이 한 줄에 정리
  • 실수로 발표 모드 진입 가능 — 클릭 거리 짧음
  • compact / presentation 의 시각 차이가 작으면 혼란
CFullscreen 진입 = 발표 모드

Fullscreen API 진입과 발표 모드를 자동 동기화. F11 / 발표 모드 버튼이 같은 행위.

  • 한 행위로 두 효과 — 빔 환경 친화
  • Fullscreen 종료 = 발표 종료
  • Fullscreen API 별도 Gate 필요
  • 전체화면 없이 발표 모드만 원하는 경우 대응 불가
  • 일부 브라우저 권한 이슈

04 Presentation Layout Layout

발표 모드의 레이아웃은 콘텐츠 영역을 최대화해야 한다. Topbar 는 최소화 또는 compact, Left Navigation 은 접거나 숨기고, 테이블 가로 스크롤을 피하기 위해 핵심 컬럼만 노출한다. 페이지 전체 가로 스크롤은 어떤 경우에도 금지한다.

ACompact Topbar + Collapsed Left Nav

기본 Shell 구조를 유지하되 Topbar 를 compact 로 줄이고 Left Nav 는 collapsed (icon-only) 로 접는다. 1366 ~ 1920 모두 동일 구조.

  • 구조 변동 최소 · 구현 부담 적음
  • 발표 종료 후 복귀 부드러움
  • 가로 56px 가 그대로 점유 — 1366 에서는 콘텐츠 폭 부족
BFullscreen Table

Topbar · Left Nav 모두 숨김. 화면 전체가 테이블 + 작은 종료 버튼만.

  • 최대 폭 확보 — 빔 친화적
  • 참가자가 표 외 요소에 시선이 가지 않음
  • 회의 중 다른 메뉴 이동이 어려움
  • 현재 위치 / breadcrumb 부재
CMeeting Dashboard + Table Hybridrecommend

상단에 Summary Bar (전체 · 진행 · 완료 · 지연 · 위험 · 오늘 · 이번 주) 를 큰 숫자로 두고, 그 아래 P0 컬럼 테이블. Topbar 는 compact, Left Nav 는 1440 이상 collapsed / 1366 이하 숨김.

  • 회의 시작 0.5초 안에 상황 파악
  • 핵심 컬럼 + 상태 chip 시인성 최고
  • 회의 진행자 시선이 표 위 ~ 표로 자연 이동
  • Summary Bar 컴포넌트 신규 정의 필요
1차 추천 — C 안 상단 Summary Bar + 핵심 WBS 테이블 hybrid 구조. 발표 모드 진입 시 자동 적용되며, Topbar 는 compact, Left Nav 는 viewport 폭에 따라 collapsed / hidden 자동 전환한다.

  • 작은 폰트 — 회의실 뒤에서 안 보임
  • 가로 스크롤 — 정보가 화면 밖으로 나감
  • row 마다 ··· / 편집 / 삭제 action — 시선 분산
  • 지연 / 위험이 chip 하나로만 표시 — 행 강조 없음 → 묻힘
  • edit UI 노출 → 회의 중 사고 편집 위험
  • 발표 모드 진입점 없음 → 작업자용 화면 그대로 띄움

18 Implementation Guardrails Rule
!기본 WBS 테이블을 발표 모드로 강제 변경 금지 — view mode 분기로만 적용.
!자동 진입 금지 — 사용자가 명시적으로 진입 (버튼 · segmented · URL flag).
!페이지 전체 가로 스크롤 금지 — Data Table Guide 정책 그대로 적용.
!발표 모드에서 작은 action 버튼 과다 노출 금지 — row 단위 ··· / 편집 / 삭제 비노출.
!edit / delete action 기본 노출 금지 — 읽기 중심.
✓필터 결과는 행 제거 방식 — dim 은 비교 / 검토 모드에서만.
✓빔 모드에서도 S-Core Dream 유지 — 외부 폰트 / Pretendard CDN / Google Fonts 금지.
!기능 로직 변경은 별도 Gate — Sort / Filter 동작 정의 변경은 본 가이드 범위 밖.
!전역 .tbtn 일괄 치환 금지 — 도큐 nav 전용 클래스. 제품 버튼과 분리.
iFullscreen API 는 별도 Gate — 본 문서는 UI 기준만 제안.
i발표 모드 진입 후 URL 에 flag 노출 — 새로고침 / 공유 시 모드 보존 (?present=1).
i색 단독 표시 금지 — 상태 / 위험 모두 색 + 텍스트 + 아이콘 병행.
✓발표 모드 종료 시 working mode 의 스크롤 / 필터 / 정렬 유지 — 복귀 후 자연스러움.
!read-only panel 의 primary 액션 금지 — ghost / 메모 추가만 후보.
