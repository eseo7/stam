01Executive Summary — 시각 주석형 화면설계서 기준 개요기준
  • 기존 문서는 구조와 흐름을 설명했지만, 실제 화면처럼 보이는 시각 주석 구조가 부족했다.
  • STAM 화면설계서 템플릿은 앞으로 "왼쪽 UI 목업 + 오른쪽 번호별 Description" 구조를 기본으로 한다.
  • 번호는 화면 위 요소에 직접 배치한다. Description에는 목적·조건·입력·검증·권한·연결 화면을 적는다.
  • 이 구조로 개발자·기획자·디자이너·검토자가 같은 화면을 기준으로 대화할 수 있다.
  • 첨부파일은 번호 주석 데이터를 대체하지 않는다. 번호 주석이 화면 구조의 근거다.

번호 마커 규칙

번호는 123 형태로 화면 요소와 인접 배치한다.
번호는 화면 흐름 순서대로 위→아래, 좌→우로 배치한다.
같은 성격 항목은 그룹 번호 사용 가능하다.
화면 이동/액션 버튼은 별도 번호로 분리한다.
단순 장식 요소에는 번호를 부여하지 않는다.

08Authoring Rules — 번호 주석 작성 규칙8 규칙
모든 주요 입력 필드에는 번호를 부여한다.
모든 주요 버튼에는 번호를 부여한다.
상태 chip, 오류 메시지, 빈 결과 문구도 설명이 필요하면 번호를 부여한다.
단순 장식 요소에는 번호를 부여하지 않는다.
번호는 화면 흐름 순서대로 위→아래, 좌→우로 배치한다.
Description은 개발자가 구현 기준으로 이해할 수 있을 정도로 구체적으로 작성한다.
첨부파일은 참고자료이며, 번호 주석을 대체하지 않는다. 번호 주석이 화면 구조의 근거다.
조건부 표시 요소(빈 상태, 오류, 권한 없음)도 반드시 Description에 포함한다.
09Review Rules — 검토자 기준7 기준
번호가 화면 요소와 정확히 연결되어 있는가
Description이 기능/데이터/검증/권한을 충분히 설명하는가
누락된 버튼/필드/상태가 없는가
연결 화면이 명확한가
오류/빈 상태/권한 없음 상태가 정의되어 있는가
개발자가 이 문서만 보고 구현 범위를 이해할 수 있는가
첨부파일에 의존하지 않고 화면 구조가 정의되어 있는가
10Accessibility Notes — 번호 주석형 화면설계서 접근성 메모7 항목
번호 마커는 색상만으로 구분하지 않는다 — 텍스트 숫자와 함께 표시
번호와 Description은 텍스트로 연결되어야 한다 (숫자 일치)
Drawer 초점 이동/복귀 기준을 Description에 명시한다

상태 chip은 텍스트를 함께 표시한다 (색상 단독 금지)
버튼명은 기능을 명확히 설명해야 한다 (아이콘만 금지)
키보드 접근 순서는 번호 순서와 크게 어긋나지 않게 한다
11Cursor Implementation Notes — 구현 기준 메모후속 참고
중요: 이 문서는 정적 Preview다. 실제 번호 마커 편집 기능은 후속 작업이다.
우선은 템플릿 상세/결과 화면에서 번호 주석형 Preview를 표시하는 것부터 구현한다.
번호 마커 위치는 초기에는 템플릿별 고정 좌표/영역 매핑으로 시작할 수 있다.
자유 배치/드래그 편집은 후속 기능으로 분리한다.
Description 패널 데이터는 화면설계서 본문 데이터로 저장해야 한다.
첨부파일은 번호 주석 데이터를 대체하지 않는다.
Register Drawer에서는 번호 주석을 자동 생성하고, 작성자가 설명을 수정하는 방식이 적합하다.
번호 주석은 화면설계서 버전과 함께 관리되어야 한다.
번호는 1부터 시작하며, 템플릿 구조가 바뀌면 번호도 재할당할 수 있어야 한다.
