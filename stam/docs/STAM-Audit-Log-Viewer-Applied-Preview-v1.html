01Overview — 문서 목적목적

STAM 감사로그 조회 화면은 단순한 "로그 목록"이 아니라 보안 운영 / 사고 추적 화면이다. 누가 언제 누구의 개인정보를 unmask 했는지, 다운로드 / 권한 변경 / 로그인 실패 사이의 인과관계를 추적하는 곳이며, 조회 자체도 audit 대상이다.

이 문서가 다루는 것
  • Security Guide v1 의 7 로그 소스 통합 조회 UI
  • Admin Permission Manager v1 의 변경 이력 → audit log linkage
  • Dashboard / Tabs (7) / Basic + Advanced Filter / Search
  • Unified Audit Log Table + Detail Panel + Related Timeline
  • Security Event Highlight (9 위험 패턴)
  • Export Control + Reason Modal + Watermark
  • Retention / Archive Notice (4 보존 정책)
  • Viewer Permission 정책 + Empty / Denied / Failed 상태
  • Mobile — 카드 리스트 · 긴급 조회 · export 금지
  • Mock A ~ J (Bad Example 포함)
이 문서가 다루지 않는 것
  • stam/pages/**, stam/js/**, stam/css/** 직접 수정
  • 실제 DB 스키마 생성 / Firebase rules 수정 / API 구현
  • security event detector 알고리즘 구체 구현 (v2)
  • SIEM / 외부 보안 시스템 연동 (v2)
  • archive / cold storage 마이그레이션 (v2)
  • 본인 셀프 조회 — v2 후보
  • 전역 .tbtn 일괄 치환 · Firebase / package.json 변경
02Audit Log Viewer PrinciplesPrinciple
!감사로그는 append-only — 수정 / 삭제 기능 없음. 보존 기간까지 보관.
!일반 사용자 조회 불가 — SECURITY_ADMIN / SUPER_ADMIN 만. PM 이하 차단.
!SECURITY_ADMIN 중심 — 업무 데이터 read 와 분리. 감사 전용.
!조회 행위도 audit 대상 — admin_action_logs 에 AUDIT_VIEW 로 기록.
✓실패 / 거부 로그도 표시 — DENIED / FAILED 가 가장 중요한 단서.
!개인정보 원문 비저장 · 비노출 — targetMaskedLabel · targetId 만.
!로그 다운로드 = 강한 통제 — 별도 사유 + 워터마크 + 범위 제한.
i보존 / 파기 정책 표시 — 각 로그의 retentionUntil 명시.
i필터 / 검색은 강력하되 남용 방지 — 광범위 쿼리는 경고 + 사유.
✓SUCCESS / DENIED / FAILED 시각 위계 — chip 컬러로 즉시 식별.
!실시간 동기화 X — 30초 polling 또는 manual refresh.
✓모든 detail = 동일 logId 식별 — URL 공유 시 다른 사람도 권한 검증 후 조회.
